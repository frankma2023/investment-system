欧奈尔 A 股量化投资系统

从数据采集到交易决策的完整工程实践 · 2026 年 5 月

一、缘起与理念

每一笔亏损的交易背后，都有一个没有被系统捕捉到的信号。

威廉·欧奈尔的 CAN SLIM 投资体系在美股市场被验证了数十年。这不是一套模糊的"投资哲学"，而是一套高度量化的规则系统——每一个决策节点都有明确的数字标准：RPS 达到多少才算领涨，成交量放大多少倍才算有效突破，抛盘日累积多少个应该清仓。

但规则越多，人工执行的难度越大。一个投资者需要同时关注大盘方向、行业轮动、个股基本面、技术形态、机构动向——这不是一个人脑能稳定处理的信息量，更不用说在连续亏损的焦虑或连续盈利的亢奋中保持纪律。

这套系统把欧奈尔的每一条规则翻译成了可执行的代码，跑在真实的 A 股数据上。但它不替代你做决策——引擎负责检测和打分，你负责最终判断。引擎不说"你应该买"，它说"这里出现了基部突破信号，RS 评分 91，CANSLIM 总分 72，基本面恶化 0 项红灯，大盘处于上升趋势，建议仓位 80%。"

把纪律交给机器，把判断留给自己。

二、快速导览：投资者的一天

假设你是一个使用这套系统的 A 股投资者，每个交易日下午 4 点开始你的盘后流程：

16:00 — 数据自动更新完成

daily_update.py 已在后台跑完 14 个步骤：拉取最新 K 线、计算全市场 RS 排名、运行形态扫描引擎、更新 CANSLIM 评分、生成观察池、扫描持仓告警、输出欧奈尔每日精选。

16:05 — 查看大盘环境

打开 大盘扫描 页面：确认当前处于什么阶段（上升/回调/震荡）、抛盘日数量是否在安全线内、追盘日是否已确认。如果大盘健康度显示 85 分且建议仓位 80%，你会更有信心加仓。

16:10 — 找到最强方向

打开 指数扫描 页面：看看今天二级行业、行业主题、策略指数三个池子里最强的是哪些。比如今天"科创芯片"和"消费电子"排名前二——你知道资金在往半导体和消费电子方向流。这决定了你今晚重点看什么板块。

16:15 — 浏览每日精选

打开 欧奈尔每日精选 页面：20 只按综合得分排序的候选标的已经生成。每只股票你都能看到它的 O'Neil 得分、信号摘要、买点和止损价、以及它和哪个最强指数共振。点击"精选理由"可以看到一份六层评估报告。

16:20 — 深入复核

点击精选列表中的股票代码，进入 复核详情 页面：K 线图上标注了所有形态信号（基部突破、口袋支点、顶部形态等），下方是 CANSLIM 七维体检表、量价结构、风险收益比计算、凯利仓位建议。你花 2 分钟浏览一遍，判断这只票是否值得加入自选池。

16:30 — 买入决策

在 交易记录 页面录入买入：系统自动跑买入前检查清单（大盘环境/CANSLIM/RS/仓位上限），逐条给你看通过还是警告。你看了后果预览——如果触发止损，你会亏多少钱——然后确认买入。

次日 — 持仓监控

每天打开 持仓监控 页面，每笔持仓显示当前市值、浮盈、仓位占比和告警灯（红/黄/绿）。止损触发或大盘恶化会自动生成告警。你可以标记"已知晓"，记录自己看到了、有没有行动。

三、功能全景

3.1 大盘环境：第一道闸门

欧奈尔统计：75% 的个股走势与大盘方向一致。系统的大盘环境模块回答一个核心问题：当前适不适合买卖？

模块功能
抛盘日检测追踪沪深 300 等关键指数在 25 日内的抛盘日数量，≥ 5 个警告，≥ 6 个触发清仓
追盘日确认暴跌后第 4~7 天检测放量反弹，确认新一轮上升趋势
大盘健康度综合 MA50 占比、新高新低比、融资余额等多项指标，输出 0~100 评分
市场全景全市场涨跌分布、成交量趋势、行业轮动热力图
逆势操作是个人投资者最大的亏损来源。大盘方向不是用来预测的，是用来遵守的。

3.2 指数扫描：找到资金流向

A 股有超过 400 个指数。在 5000 只个股中选股之前，先看资金往哪个方向流。

系统对 5 个指数池（市场指数 29 个、一级行业 11 个、二级行业 45 个、行业主题 229 个、策略指数 93 个）分别计算 RS_20/60/120/250 四个周期的相对强度，并自动筛选出最强指数：

全部指数→RS 排序→条件过滤→最强 TOP N

在此基础上，还有指数拥挤度（当所有人都在买的时候反而是风险）和机构吸筹/出货评级（A~E，基于 A/D 线）两个辅助指标，帮助判断指数的安全边际。

3.3 个股基本面：不只看数字，更看趋势

在 个股扫描 页面，输入任意 A 股代码，可以一站式看到：

模块内容
估值仪表PE/PB/PS/股息率/市值的十年历史分位
盈利质量ROE、EPS、净利润增速、营收增速、毛利率、自由现金流的年度趋势
财务健康资产负债率、有息负债率、负债结构、流动/速动比率
市场强度RS 评级、双强股判定、机构持股变化
财务评估DCF 估值 + 可比公司分析 + 三表联动模型
基本面恶化6 项检测，3 项以上红灯建议卖出

CANSLIM 七维评分（C/A/N/S/L/I）每天自动计算全市场约 5000 只股票的分数，在看板上按分排序。

3.4 买入形态：统一引擎下的八种基部

欧奈尔定义了八种经典基部形态——杯柄、碟形、双重底、扁平基部、高紧旗、一底比一底高、递升底部、箱体。最初系统为每种形态写了独立引擎，但参数越来越精细，信号却越来越少，最终几乎归零。

重新审视后发现，所有八种形态共享同一个底层结构：上涨 → 前高 → 回调/整理 → 谷 → 回升 → 突破。区别只在于回调的形状和深度——这些不改变"是否突破"的判断。

K 线数据→Layer 1 核心突破引擎→Layer 2 形态标注器 (9个)→Layer 3 用户过滤→突破信号

在 形态识别 页面，输入代码和时间范围，系统同时运行所有引擎——包括口袋支点、量价背离、铁轨线、头部形态、高潮见顶等——在 K 线图上标注所有信号点，不同引擎用不同颜色和形状区分，一目了然。

3.5 卖出信号：比买入更重要的能力

"会买的是徒弟，会卖的才是师傅。"系统提供三类卖出信号：

类别引擎说明
见顶信号高潮见顶连续急涨后放量长上影或高位滞涨
见顶信号铁轨线两根几乎等高的放量 K 线，多空博弈后空方胜
见顶信号头部形态头肩顶、双重顶、三重顶识别
突破失败突破失败检测7 条规则独立触发，10 日监控每个突破信号
基本面恶化基本面恶化检测EPS 减速、ROE 下滑等 6 项财务检查

3.6 回测验证：用历史数据验证你的参数

每个检测引擎都配有独立的回测看板。你可以在左侧滑块调整参数，点击"回测"后右侧显示历史触发的所有信号明细，参数保存到 YAML 配置文件，下次打开自动恢复。目前有 19 个回测看板，覆盖几乎所有检测引擎。

四、知行系统：投资纪律的执行层

如果说前三章的分析引擎是系统的"眼睛"和"大脑"，那知行系统就是系统的"手脚"——它负责把分析结果转化为可执行的交易动作，并持续监控执行质量。

分析师能告诉你买什么，但只有纪律系统能让你在应该卖的时候真的卖掉。

4.1 观察池与自选池

观察池是系统每天自动生成的候选股票池。它以 RS 动量为核心维度，将全市场正常上市的股票按 RPS 250 和 RPS 20 两个周期筛选，分类为双强、龙头、加速、短爆、稳健五个等级，再叠加 CANSLIM 评分和基础财务指标。用户不能手动添加——这保证了观察池的客观性。

自选池是你从观察池中确认关注的标的。也支持手动添加（需标注理由），支持筛选、排序、一键跳转交易录入。

4.2 交易记录与买入前检查

每一笔交易都是一条完整的买卖闭环：买入日期/价格/数量/理由/情绪/止损价 → 卖出日期/价格/理由/情绪 → 自动计算盈亏和持股天数。

买入前检查清单是知行系统的核心模块之一。在录入买入之前，系统自动运行 6 条规则检查：

规则类型说明
大盘环境软警告当前市场阶段是否适合买入
CANSLIM 评分软警告基本面是否达标
RS 强度软警告动量是否处于领涨水平
单票仓位上限硬拦截买入后该股占总资产不超过 20%
总仓位上限硬拦截买入后总仓位不超过 80%
行业集中度软警告同行业持仓是否过度集中

检查结果直接展示在买入弹窗中——绿色通过、黄色警告、红色拦截。硬拦截触发时买入按钮灰色不可用，软警告触发时用户勾选"已知风险"后可继续。你还必须输入 20 字以上的买入理由，系统会弹出后果预览——如果触发止损你会亏多少钱。

4.3 持仓监控与告警

买入后系统不会沉默。持仓监控引擎每天自动扫描所有持仓：

检查项告警级别
止损触发（当前价 ≤ 止损价）🔴 红色
大盘风险（风险级别=高/危险）🔴 红色
个股走弱（跌破 50 日均线 / 连跌 5 日）🟡 黄色
见顶信号（检测到头部形态）🟡 黄色

每张持仓卡片显示买入日/买入价/现价/浮盈/持股天数/市值/仓位占比和告警灯。有告警的排在前面，点击可展开告警详情，标记"已知晓"后记录确认时间——下次复盘时你知道自己看到了，有没有行动。

对于在理杏仁中没有 K 线数据的指数持仓，系统还支持手工录入当日现价，写入快照表后在市值计算和盈亏统计中自动生效。

4.4 仪表盘

知行系统首页是一个总览仪表盘，显示：观察池标的数、自选池标的数、持仓中、累计盈亏、胜率、盈亏比、最大回撤、总资产（可点击编辑）、总市值、现金、仓位百分比。一目了然的投资快照。

五、欧奈尔每日精选：从数据到决策

这是系统最新完成的核心功能——它将前面所有分析模块的能力汇聚到一个六层漏斗筛选引擎中，每天盘后自动输出 20 只最值得关注的股票。

5.1 六层漏斗

层名称类型条件
1大盘闸门提醒非上升趋势时显示强烈警告，仍输出结果供参考
2RS 精英过滤RPS250 ≥ 90、RPS20 ≥ 85，仅保留双强和龙头
3CANSLIM 质量过滤总分 ≥ 48、ROE ≥ 5%、营收增速 ≥ 5%、负债率 < 70%
4技术形态过滤+否决近 20 日必须有基部突破信号，一票否决顶部/高潮/失败
5行业共振加分命中最强指数（二级行业/主题/策略 TOP 5）加权 ×1.15~1.25
6综合排序输出O'Neil = CANSLIM×40% + RPS250×25% + 信号×20% + 共振×15%

第四层的信号检测不是从单日快照读数据——系统对每只 L3 幸存者加载 400 天 K 线，实时运行全部形态扫描引擎，检出 K 线范围内所有技术信号，再用 20 日窗口筛选买入信号和否决信号。这保证了精选不会因为某一天没有扫描数据而漏掉机会。

5.2 精选理由与推荐信

每只精选股票都有一个"精选理由"按钮。点击后会弹出一篇结构化推荐信，以欧奈尔的评估框架逐一分析：

  • 第一层 · 大盘环境：当前阶段、建议仓位、风险提示
  • 第二层 · RS 动量：RPS 各周期值、在全市场的位置、RS 线趋势
  • 第三层 · CANSLIM 基本面：七维评分逐一展示、财务健康度
  • 第四层 · 技术形态：检测到的买入/否决信号、理想买点、止损价
  • 第五层 · 机构与行业：覆盖基金数、机构持股比、行业共振情况
  • 总结与建议：该股在欧奈尔体系中的定位、建仓建议

推荐信底部附有雪球和东方财富的快捷搜索链接，方便查看外部观点。

六、技术内幕

6.1 架构选型

系统采用 Python Flask + SQLite + 原生 JavaScript + ECharts。这个技术选型很"朴素"——但它基于一个现实约束：这是一个个人投资者在 Windows 笔记本上运行的本地系统，不需要分布式，不需要微服务，不需要云数据库。

理杏仁 API→采集脚本 (20+)→SQLite (~41GB)→Flask API (8788)→前端 (8772)

前后端分离部署：Flask 做 API 服务（74 个端点），Python http.server 挂载静态文件，`nav.js` 内置 API 代理自动将前端请求转发到 Flask。

6.2 数据管线

daily_update.py 是每日盘后的一键更新脚本，14 个步骤串行执行：

股票状态→指数K线→个股K线→基本面→拥挤度→融资融券→大盘健康→RS计算→形态扫描→CANSLIM→观察池→监控扫描→精选

指数 K 线拉取采用增量模式——脚本自动查询每只指数在数据库中的最新日期，只拉取缺失部分，不会重复拉取。

6.3 引擎体系

系统的检测引擎实现了自动发现机制（engine_registry.py）：任何放在 src/scanners/ 目录下、实现了 detect() 函数且声明了 ENGINE_META 字典的 Python 文件，都会被自动识别并纳入形态扫描体系，无需手动注册。

这形成了一个开放式的引擎生态——目前包含 10+ 个主动引擎（基部突破、口袋支点、双重底、扁平基部、碟形基部、杯柄、高潮见顶、铁轨线、头部形态、突破失败、量价背离、抛盘日）和 1 个基本面恶化检测引擎。

6.4 数据体系

核心数据库约 41GB，50+ 张表：

类别代表表规模
K 线daily_kline, index_daily_kline~1846 万 + ~142 万
基本面fundamental_indicator~1.25 亿
