## 第一阶段：寻找开源参考

### 用户初始需求
我想开发一个仓位管理系统，对我的投资提出深刻和专业意见，控制我冲动的买入或卖出操作，导致亏损。但没有成熟的工具可借鉴。

功能设想：
- 录入股票代码、买卖价格、数量、时间等
- 统计时间区间盈亏、总市值、总仓位、持股时长、风格判断和操作建议
- 录入买入原因、目标、卖出理由，方便事后复盘
- 结合已开发的大盘/指数/个股扫描引擎、CAN SLIM评分、财务评估、形态扫描、买卖信号识别引擎，对持仓进行监控，给出专业判断和仓位建议
- 自选池功能，可添加/剔除
- 对自选池标的给出专业意见

### 开源项目全景分析

市场上有这些参考，但没有一个开箱即用的纯仓位管理系统：

**量化框架中的仓位管理模块**：
| 项目 | Stars | 仓位/风控能力 | 适用性 |
|------|-------|-------------|--------|
| VnPy | 25k+ | PortfolioManager + RiskManager 双模块，多账户多策略统一风控，支持事前风控 | A股原生支持 |
| Backtrader | 14k+ | Sizer 抽象类，内置 FixedSize/PercentSize/AllInSizer | 架构优雅，无A股数据源 |
| Freqtrade | 30k+ | PositionHandler + StoplossManager，动态止损 | 加密货币 |
| Zipline-Reloaded | 3k+ | order_percent / order_target_percent | 已停止维护 |
| QuantConnect Lean | 10k+ | PortfolioConstruction + RiskManagement | 太重 |

**纯投资组合优化库**：PyPortfolioOpt、Riskfolio-Lib、Empyrial

**结论**：VnPy 的仓位/风控模块最值得参考。

## 第二阶段：深入分析 VnPy 源码

### 获取源码的方式
用户通过 Watt Toolkit（证书代理，非端口代理）上网。绕过方式：
```python
ctx = ssl.create_default_context()
ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE
resp = urllib.request.urlopen(url, context=ctx)
```

### 获取到的核心源码分析

**RiskEngine（风控引擎）** — `vnpy_riskmanager/engine.py`：
- 通过 monkey-patch `main_engine.send_order` 拦截所有下单请求
- 动态加载 `rules/` 目录下的风控规则（自动发现 `.py` 文件）
- 每条规则继承 `RuleTemplate`，实现 `check_allowed()` 方法
- 任何规则返回 False → 阻止下单

**RuleTemplate（规则模板）**：
```python
class RuleTemplate:
    name: str                     # 规则名称（UI显示）
    parameters: dict[str, str]    # 用户可配置的参数
    variables: dict[str, str]     # 运行时计算的状态变量

    def check_allowed(self, req, gateway_name) -> bool:
        """检查是否允许下单 ← 核心方法"""
        return True

    # 四种生命周期回调：
    def on_init(self): pass       # 初始化默认参数
    def on_tick(self, tick): pass # 收到行情时
    def on_order(self, order): pass # 订单状态变化时
    def on_trade(self, trade): pass # 成交时
    def on_timer(self): pass      # 每秒触发
    def put_event(self): ...      # 推送状态到UI
```

**现有规则示例**：
- `ActiveOrderRule`：活跃订单数量限制（默认50笔）
- `DailyLimitRule`：日内交易频率限制（总委托≤20000，单合约委托≤2000 等6个维度）

**PortfolioEngine（组合引擎）**：
- 按 strategy reference 分组追踪每个合约维度（ContractResult）+ 组合维度（PortfolioResult）
- 每 5 秒计算一次浮动盈亏（trading_pnl + holding_pnl → total_pnl），推送到 UI

### 核心洞察
> 好的仓位系统不是事后记账，而是事前拦截。
> VnPy 的设计：拦截器 + 规则组合 + 自动发现 → 完全可以映射到 Flask + SQLite 架构。

## 第三阶段：从"仓位管理"到"知行系统"

### 用户真正需要什么

> 炒股是孤独的——没有人商量、没有人监督、没有人复盘、没有绩效归因。
> 我需要的不只是一个记录软件，而是一个 AI 投资纪律官。

### 四角色合一

机构交易员有四个独立角色约束决策：
| 角色 | 职责 | 对应模块 |
|------|------|---------|
| **风控总监** | 超限单子根本发不出去 | 硬拦截规则 |
| **投资委员会** | 审视每笔持仓的理由是否仍然成立 | 软警告规则 |
| **合规部门** | 确保不违反预先注册的策略边界 | 合规检查 |
| **绩效分析师** | 把每笔盈亏拆成"市场给的"和"自己作的" | 归因复盘 |

### 三阶段闭环

```
事前拦截 ──→ 事中监控 ──→ 事后复盘
   │              │              │
   │ 规则拦截     │ 引擎扫描     │ 决策归因
   │ 冷静期       │ 信号预警     │ 情绪标记
   │ 后果预览     │ 止损触发     │ 心力评分
   │              │              │
   └──────────────┴──────────────┘
                 ↑
            迭代优化规则
```

### 超越预期的功能设想

1. **交易冷静期**：候选标的必须在候选池中停留 N 天才能买入
2. **后悔模拟器**：买入确认前弹出"如果明天跌 5%，这还是好交易吗？"
3. **心力曲线**：心力评分叠加大盘 K 线图，发现心力与市场顶底的关联
4. **规则回溯**：对比"实际收益"与"如果 100% 遵守规则的收益"

### 命名

"知行" — 知行合一。知：已有扫描引擎（CAN SLIM、RS、形态、见顶信号）。行：纪律执行（检查清单、止损、情绪管理、归因）。

英文：O'Neil Discipline Engine

## 第四阶段：V1/V2/V3 功能规划

### V1：交易记录 + 买入前检查清单

买入前强制检查清单（不可跳过）：
```
🛑 买入前强制检查
□ 1. 大盘环境允许买入吗？    [自动读取 market_direction]
□ 2. CAN SLIM 评分 ≥ 70？    [自动读取 canslim_scores]
□ 3. RS 强度在市场前 20%？    [自动读取 rs_daily]
□ 4. 买入后单票仓位占比？    [自动计算，对比上限]
□ 5. 你的止损价是多少？      [必填]
□ 6. 你的目标周期是什么？    ○ 短线 ○ 波段 ○ 中线
□ 7. 买入理由（≥20字）       [必填]
[全部勾选并填写后才能确认买入]
```

心理学机制：
- 自动读取引擎数据，不给自我欺骗的空间
- 必须填写止损价和买入理由，**写下 = 承诺**
- 后果预览："如止损，将亏损 X 元"

### V2：持仓监控 + 信号预警

持仓后持续监控：
- 大盘环境变化
- RS 强度下滑
- 见顶信号出现（抛盘日/高潮顶/量价背离）
- 止损线逼近
- 7 周法则触发

综合判断：持有 / 减仓 / 清仓 + 理由（标注来源引擎）

### V3：绩效归因 + 心力追踪 + 规则回溯

四维归因：
1. **α/β 分解**：超额收益来自选股还是市场？
2. **规则遵守追踪**：检查清单通过率、止损执行率、信号响应率 → "纪律曲线"
3. **情绪-结果关联**：FOMO 买入 vs 冷静买入的胜率差异
4. **规则回溯**（最狠）：如果 100% 遵守规则，收益率曲线什么样？

输出：「我的投资人格」雷达图 + 年度体检报告

## 第五阶段：Grill-with-Docs 13 轮追问

### 决策 1：规则分级机制

8 条检查规则按拦截力度分三级：
- **硬拦截（风控）**：`PositionLimitRule`、`TotalPositionLimitRule`、`StopLossMustSetRule` → 按钮灰色，不可交易
- **软警告（投委会）**：`MarketPhaseRule`、`CANSLIMRule`、`RSRule`、`IndustryConcentrationRule` → 弹出警告，用户可勾选"已知风险"继续
- **合规检查**：`ReasonRule`（买入理由 ≥20 字） → 不满足不可提交

### 决策 2：API 硬拦截返回 422

`POST /api/discipline/trades` 在硬拦截规则未通过时返回 422 + 具体原因，前端抛错而不是静默拒绝。

### 决策 3：已有数据表处理

| 已有表 | 处理 |
|--------|------|
| `portfolio_holdings` (12条) | 迁移到 `discipline_trades` 后废弃 |
| `portfolio_signals` (44条) | 保留，作为 V2 告警逻辑验证参考 |
| `buy_signals_daily` (空表) | 废弃 |
| `watchlist` | 保留并扩展字段（source/review_status） |

### 决策 4：选股链 — 观察池 + 自选池

```
全网 A 股 → RS 分类筛选 → 观察池（自动，每日定时）
   → 多维度扫描（财务+CANSLIM+形态）→ 用户浏览复核
      → 确认加入自选池（手动确认）
```

**观察池**：每日计划任务自动生成。用户不可手动添加。
**自选池**：用户从观察池确认选入，或手动添加（走异步评估通道）。

### 决策 5：观察池生命周期 — 每日全量替换

每天 truncate + insert 新一批，用户操作状态（复核/确认）独立存储在另一张状态表。

### 决策 6：宽表策略

所有扫描维度结果写入单表 `discipline_observation_pool`：
- RS 分类（稳健龙头/短期爆发/双强股/rps各周期）
- CANSLIM 七维评分
- 财务健康（ROE/EPS增速/营收增速/负债率/毛利率）
- 估值仪表（PE/PB/PE分位/市值）
- 形态信号（买入信号 JSON + 卖出信号 JSON）
- 综合建议

前端一次查询完成页面渲染。

### 决策 7：统一复核详情页

不再跳转多个回测看板。一只股票 = 一个详情页 = 一个 K 线图 + 所有引擎信号标记。

### 决策 8：复核页展示全部信号（买入 + 卖出）

买入前看不到卖出信号，才是最大的风险。K 线图上买/卖信号全部显示，颜色分层：
- 绿色：杯柄突破、碟形基部、VCP 突破等
- 红色/橙色：高潮顶、铁轨线、量价背离

信息卡片分左右：左边买入信号汇总，右边风险提示。如检测到卖出信号，综合建议自动提示谨慎。

### 决策 9：复核专用 API

新增 `GET /api/discipline/review/<stock_code>`，后端一次性返回该股全历史 K 线 + 所有引擎信号，前端不用调 N 个 API 聚合。

### 决策 10：分批买卖按批次独立管理

每批买入 = 一条独立交易记录。盈亏按批次独立计算。不采用 FIFO 聚合。更利于复盘："这笔加仓的决策是对是错"一目了然。

### 决策 11：手动添加异步评估

手动添加的股票触发异步评估任务（后台跑完整扫描），评估完成前标记 `pending_review`，不可进入买入流程。强制弹窗："该股未通过观察池自动筛选，你的添加理由是什么？"

### 决策 12：买入确认完整链路

```
自选池（已评估通过）
  → 点击「计划买入」
    → 弹出买入表单：股数、价格
    → 系统实时执行买入前检查清单
    → 展示硬拦截/软警告结果
    → 填写止损价、目标周期、买入理由
    → 展示后果预览
    → 勾选"已知风险"（如有软警告）
    → 点击「确认买入」
      → 写入 discipline_trades
```

### 决策 13：成交价手动填写

盘后日 K 线数据源，没有实时行情，成交价手动填入。
