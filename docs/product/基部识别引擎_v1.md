# 基部识别引擎 v1.0

## 核心理念

不识别杯柄、双底等具体几何形状，只判断股价是否处于"合格基部"的右侧末端。
用三个定量规则描述"大涨 → 休息够了 → 准备再起"这个完整状态。

## 三条规则

### 规则一：前置上涨

在最近 L 个交易日（默认 120）内，存在一个**低点-高点对**：
- 低点在前，高点在后
- 涨幅 = (高点 - 低点) / 低点 ≥ 30%
- 高点距今 ≥ M 个交易日（默认 20，即整理至少持续 20 天）

### 规则二：价格紧凑整理

从规则一找到的高点开始到今天：
- 最大回撤 ≤ 前涨幅的 R% （默认 60%）
- 持续时间 ≥ M 天（默认 20）
- 振幅收窄：最近 10 日平均振幅 < 前 20 日平均振幅（可选，开关控制）

### 规则三：右侧蓄力信号

最近 5 个交易日内：
- 成交量 MA5 ≥ 成交量 MA50 × V 倍（默认 1.2）
- 收盘价不再创 20 日新低

## 输出

对每只股票，每个交易日返回：

| 字段 | 含义 |
|------|------|
| `is_setup` | 当日是否处于基部右侧 |
| `prior_low_date` / `prior_high_date` | 规则一找到的低点/高点日期 |
| `prior_gain` | 前涨幅(%) |
| `drawdown` | 从高点至今最大回撤(%) |
| `consolidation_days` | 已整理天数 |
| `amp_trend` | 振幅收窄比例（<1=收窄） |
| `vol_ratio` | 近5日均量/MA50 |

## 参数配置

| 参数 | 默认值 | 含义 |
|------|:--:|------|
| `lookback` | 120 | 规则一搜索窗口 |
| `min_advance` | 30 | 最小前置涨幅(%) |
| `min_consolidation` | 20 | 最少整理天数 |
| `max_drawdown_ratio` | 60 | 最大回撤/前涨幅(%) |
| `amp_narrow_enabled` | true | 是否要求振幅收窄 |
| `vol_ratio` | 1.2 | 量比阈值 |
| `new_low_days` | 20 | 不创新低检查窗口 |
