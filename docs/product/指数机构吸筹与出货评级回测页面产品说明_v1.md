# 指数机构吸筹/出货评级回测页面产品说明 v1

> 最后更新：2026-05-06  
> 对应引擎：`src/detectors/index_ad.py`  
> 配置文件：`config/index_ad.yaml`

## 1. 回测页面总体需求

- 使用**小红书卡片风格**，与已有回测页面保持一致
- 页面布局：**左侧参数配置面板 + 右侧结果展示区**
- 日期选择：日历选择器选单个交易日，默认最近一个交易日
- 用户调整参数后点击「计算」按钮，调用API展示结果
- 点击「保存配置」，将当前参数写入 `config/index_ad.yaml`
- **不持久化计算结果**，每次实时计算

## 2. 左侧参数可配置卡片

### 2.0 通用选择器
- 指数池选择：下拉菜单（market / sector_l1 / sector_l2 / thematic / strategy）
- 日期选择：日历组件，默认最新交易日

### 2.1 卡片1：评级阈值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| AD窗口 | 65日 | 滚动窗口交易日数（40~130日可调） |

### 2.2 卡片2：评级映射

展示A+到E-的百分位映射表，不可手动调整（只读），供用户参考。

| 评级 | 百分位区间 | 含义 |
|------|-----------|------|
| A+ | ≥ 90% | 机构强烈吸筹 |
| A | 80%~89% | 机构强烈吸筹 |
| B+ | 70%~79% | 机构较强烈吸筹 |
| B | 60%~69% | 机构温和吸筹 |
| C+ | 50%~59% | 机构买入稍大于卖出 |
| C | 40%~49% | 机构买卖大致平衡 |
| D+ | 30%~39% | 机构温和出货 |
| D | 20%~29% | 机构大力出货 |
| E | 10%~19% | 机构强烈出货 |
| E- | < 10% | 机构强烈出货（最差） |

## 3. 右侧结果展示区

### 3.1 统计卡片

| 指标 | 说明 |
|------|------|
| 指数总数 | 池内指数数量 |
| 净吸筹数 | A+ ~ C+ 的指数数量 |
| 净出货数 | C ~ E- 的指数数量 |
| 最强吸筹 | 评级最高的指数名称 |

### 3.2 池内评级表格

按 AD_Score 降序排列：

| 排名 | 代码 | 名称 | AD_Line(万元) | 百分位 | 评级 | AD_Score | 含义 |
|------|------|------|-------------|--------|------|----------|------|
| 1 | ... | ... | ... | ... | **A+** | ... | ... |

### 3.3 评级分布柱状图（ECharts）

横轴：A+ ~ E- 十个评级
纵轴：该评级内的指数数量
柱状颜色：吸筹侧（A+~C+）暖色，出货侧（C~E-）冷色

### 3.4 各池子差异化展示

| 池子 | 展示方式 |
|------|---------|
| 全市场 (market) | 默认表格，AD_Score ≥ 0 高亮 |
| L1行业 (sector_l1) | 表格 + 评级分布热力图（横向行业，纵向评级） |
| L2行业 (sector_l2) | 按一级行业分组折叠 |
| 主题 (thematic) | 搜索框 + 评级筛选（仅显示A~C+或全部） |
| 策略 (strategy) | 按策略类型分组折叠 |

## 4. 技术实现

### 4.1 API端点

```
GET /api/index-ad?pool=market&date=2026-04-30&window=65
```

响应结构：
```json
{
  "as_of_date": "2026-04-30",
  "window": 65,
  "pools": {
    "market": {
      "rankings": [
        {
          "code": "000985",
          "name": "中证全指",
          "ad_line": 12345000,
          "percentile": 95.0,
          "rating": "A+",
          "ad_score": 90
        }
      ]
    }
  }
}
```

## 5. 验证用例

- 同一池子内百分位最高99，最低0
- 评级分布柱状图总和等于池子指数总数
- A+~C+ 数量 + C~E- 数量 = 统计卡片的净吸筹数 + 净出货数
- AD_Line 值大的指数排在前面，AD_Score 为正
