# 扁平基部(Flat Base)形态识别引擎 — 产品需求书

## 1. 产品定位

**扁平基部（Flat Base）** 是 William J. O'Neil 定义的七种经典基部形态中最紧凑的一种。它形似一个水平的窄箱子——股价在一个极窄的区间内横盘整理，波动极小，成交量持续萎缩，最终放量突破箱体上沿。

在欧奈尔体系中，扁平基部常用于**第二阶段或第三阶段的基部**——即股价已经经历过一轮杯柄或双重底突破后，在上升途中短暂休整。因为震荡区间极窄，浮筹被清洗得极为彻底，突破后的爆发力往往强于其他形态。

**"清洗最彻底，爆发最猛烈"** 是扁平基部区别于其他形态的核心特征。

**在整体系统中的位置**：个股扫描模块 > 基部识别引擎的子模块。与杯柄、双重底并列，共享 K 线数据管道和回测看板。与口袋支点引擎联动——扁平基部内部出现口袋支点信号时，是极佳的抢先入场时机。

---

## 2. 核心理念

扁平基部的本质是：**在上升趋势中，机构资金通过时间（而非深度回调）来消化获利盘。** 股票不跌（或只微跌），而是在一个窄幅区间内横盘——不愿卖的人已经锁仓，想买的人还在观望。当横盘时间足够长（5 周以上），所有短线浮筹都被"磨"走了，剩下的全是坚定持有者。一旦有利好催化，股价突破箱体上沿，买方蜂拥而入，推动股价快速拉升。

类比：弹簧被水平压缩——压缩得越久、越窄，释放时的动能越大。

### 2.1 与其他基部形态的区别

| 维度 | 杯柄形态 | 双重底 | 扁平基部 |
|---|---|---|---|
| 回调深度 | 12%~33% | 15%~35% | **8%~15%（最浅）** |
| 波动特征 | U 形杯底+柄部 | W 形双底 | **水平窄箱体** |
| 形成时间 | 7~65 周 | 7~8 周 | **5~7 周（最短）** |
| 成交量 | 底部萎缩 | 右底量 < 左底量 | **全程持续萎缩** |
| 突破点 | 柄部上沿 | 中间峰最高价 | **基部最高点** |
| 常见位置 | 第一个基部 | 第一个基部 | **第二/第三个基部（上升途中）** |
| 突破爆发力 | 中-强 | 强 | **极强（最猛烈）** |

## 3. 形态结构定义

```
        ┌────────────────── 突破买点（基部最高价 + 0.01）
        │    ┌─ 窄幅横盘区间（振幅 < 12%）
        │    │
  ──────┼────┼─────── 箱体上沿
        │    │
        │    │  成交量持续萎缩
  ──────┼────┼─────── 箱体下沿
       /      \
      /        \  前置上涨 ≥ 30%
     /
```

### 3.1 关键特征（按重要性排序）

| 优先级 | 特征 | 量化描述 |
|---|---|---|
| **1** | 极窄振幅 | 基部内最高价到最低价的幅度 ≤ 12%（比杯柄的 12-33% 窄得多） |
| **2** | 水平横盘 | 价格无明显趋势，箱体上沿和下沿近似水平 |
| **3** | 成交量萎缩 | 基部后半段的日均成交量 < 前半段的 60%（持续的缩量是核心信号） |
| **4** | 足够时长 | 至少 25 个交易日（5 周），横盘太短不足以清洗浮筹 |
| **5** | 突破放量 | 突破日成交量 ≥ 50 日均量的 1.4 倍 |

## 4. 量化计算规则

### 4.1 前置条件

```
步骤 0.1: 在考察日之前, 确认存在一段 ≥ 30% 的前置上涨
步骤 0.2: 当前价格处于上升趋势中（收盘价 > SMA50，SMA50 斜率 ≥ 0）
步骤 0.3: 不是第一波上涨后的第一个回调（优先寻找第二/第三个基部）
         判断方法: 在最近 120 个交易日内，已出现过其他基部形态突破
```

### 4.2 扁平基部识别（核心算法）

#### 步骤 1：定位基部区间

从当前日期向前回溯，找到一个满足以下条件的区间：

```
条件 1.1: 区间内最高收盘价与最低收盘价的振幅 ≤ amp_max（默认 12%）
条件 1.2: 区间长度 ≥ min_duration 个交易日（默认 25，即约 5 周）
条件 1.3: 区间内的线性回归斜率 ≈ 0（|slope| ≤ 0.001/天，即几乎水平）
条件 1.4: 当前日期距区间结束 ≤ 5 个交易日（基部刚刚或接近结束）
条件 1.5: 区间起点距前高 ≥ 10 个交易日（给予足够的回调/盘整空间）
```

**区间发现的实现建议**：使用滑动窗口法。从当前日期向过去滑动一个 25 天的窗口，计算窗口内的振幅。如果振幅 < 12%，逐步扩大窗口直到振幅超过 12%，取最大的满足条件的窗口作为基部区间。

#### 步骤 2：成交量验证（核心差异化规则）

扁平基部的成交量特征与杯柄和双重底不同——它要求**全程持续萎缩**，而非只在底部的某几天萎缩：

```
条件 2.1: 将基部区间等分为前半段和后半段
条件 2.2: 后半段日均成交量 < 前半段日均成交量 × vol_contraction_ratio（默认 0.60）
         → 即后半段的量萎缩到前半段的 60% 以下
条件 2.3: （可选增强）后半段的成交量变异系数（CV）< 前半段
         → 量不仅小，而且稳定（无异常放量日）
```

**设计理由**：扁平基部是"磨人"的——横盘初期还有人试图抄底或逃顶，越往后交易越清淡。后半段成交量萎缩到前半段的 60% 以下，说明短线资金已经彻底离场，只剩下锁仓的机构。

#### 步骤 3：布林带带宽收缩（可选增强）

扁平基部的一个显著特征是布林带带宽（Bollinger Band Width）收缩至极低水平：

```
条件 3.1: 计算过去 20 日布林带带宽 = (上轨 - 下轨) / 中轨
条件 3.2: 过去 250 日的布林带带宽历史分位 ≤ bb_pct_max（默认 20%）
         → 即当前波动率处于过去一年最低的 20% 区间
```

**注意**：此条件为可选（配置项 `bb_enabled` 默认 true，可关闭）。如果历史 K 线数据不足以计算 250 日均值和分位，可退化为仅比较当前带宽与 250 日均值。

#### 步骤 4：突破信号

```
定义: 今日收盘价 ≥ 基部区间的最高收盘价（突破箱体上沿）
约束:
  - 突破日成交量 ≥ 50 日均量 × 1.4（放量突破）
  - 突破日为阳线（收盘 > 开盘）
  - 收盘位置 ≥ 日内振幅的 50%（收盘在高位）
  - （可选）突破日收盘价 > 昨日收盘价 × 1.02（排除假突破，要求当日涨幅 ≥ 2%）
```

#### 步骤 5：RS 确认（输出而非过滤）

```
输出: RPS_20, RPS_250 附加到每个识别的扁平基部信号中
建议: RPS ≥ 80 时信号有效性更高（人工综合判断，不硬过滤）
```

## 5. 信号输出格式

### 5.1 输出字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `date` | date | 突破信号日期 |
| `stock_code` | str | 股票代码 |
| `stock_name` | str | 股票名称（可选） |
| `base_start_date` | date | 基部起始日期 |
| `base_end_date` | date | 基部结束日期 |
| `base_duration` | int | 基部持续交易日数 |
| `base_high` | float | 基部内最高收盘价（突破买点参考） |
| `base_low` | float | 基部内最低收盘价 |
| `base_amplitude` | float | 基部振幅 % = (high - low) / high |
| `price_slope` | float | 基部内价格的日线性回归斜率 |
| `vol_first_half_avg` | float | 前半段日均成交量 |
| `vol_second_half_avg` | float | 后半段日均成交量 |
| `vol_contraction` | float | 后半段量 / 前半段量（越小越好） |
| `bb_width` | float | 突破前 20 日布林带带宽 |
| `bb_width_pct` | float | 布林带带宽历史分位（0-1，越低越紧） |
| `breakout_vol_ratio` | float | 突破日成交量 / 50 日均量 |
| `prior_advance_date` | date | 前置上涨起点日期 |
| `prior_advance_pct` | float | 前置涨幅 % |
| `is_second_stage_base` | bool | 是否为第二/第三阶段基部 |
| `rps_20` | int | RS 强度 20 日 |
| `rps_250` | int | RS 强度 250 日 |

### 5.2 输出接口

- **API 端点**: `GET /api/flat-base?stock=600519&date=2026-05-12`
- **批量扫描**: `GET /api/flat-base/scan?date=2026-05-12&pool=watchlist`
- **回测看板**: `/flat-base-backtest/` 页面

## 6. 与口袋支点、标准突破的配合

扁平基部是口袋支点的**最佳拍档**。两者的结合可以产生极高的胜率：

```
场景: 扁平基部形成 → 基部内出现口袋支点（量 > 过去 10 日最大阴线量）
      → 抢先入场（比突破买点早 5-15 天）
      → 确认突破时加仓
```

建议在回测看板中标注：
- 🔵 蓝色三角：扁平基部内的口袋支点（抢先信号）
- 🔴 红色三角：扁平基部突破（确认信号）

## 7. 配置项

```yaml
# config/market/flat_base.yaml
flat_base:
  # 前置条件
  min_prior_advance: 0.30        # 前置最小涨幅
  sma50_check: true              # 是否要求收盘价 > SMA50

  # 基部识别
  min_duration: 25               # 基部最少交易日数（约 5 周）
  max_duration: 100              # 基部最多交易日数（约 5 个月）
  amp_max: 0.12                  # 基部最大振幅（12% = 极窄）
  amp_ideal: 0.08                # 理想振幅（8%，超出标黄警告）
  slope_abs_max: 0.001           # 价格斜率绝对值上限（≈ 0 为水平）
  max_days_from_base_end: 5      # 距基部结束最大天数

  # 成交量
  vol_contraction_ratio: 0.60    # 后半段量 / 前半段量最大值
  vol_cv_check: false            # 是否检查成交量变异系数（可选增强）

  # 布林带（可选）
  bb_enabled: true               # 是否启用布林带带宽检查
  bb_lookback: 250               # 布林带历史回看天数
  bb_width_window: 20            # 布林带计算窗口
  bb_pct_max: 0.20               # 带宽历史分位最大值

  # 突破
  breakout_vol_ratio: 1.4        # 突破日量 / 50 日均量
  require_green: true            # 突破日必须阳线
  close_position_min: 0.50       # 突破日收盘位置最低
  breakout_min_gain: 0.02        # 突破日最小涨幅（可选，默认 2%）

  # 过滤
  exclude_st: true               # 排除 ST
  min_listing_days: 250          # 最少上市天数（需要布林带历史数据）
```

## 8. 数据处理规则

| 场景 | 处理方式 |
|---|---|
| **数据不足** | 上市不满 250 交易日不识别（布林带需要历史数据） |
| **停牌** | 停牌日不计入基部持续天数 |
| **ST/\*ST** | 默认剔除 |
| **振幅略超 12%** | 标黄警告但不直接拒绝（15% 以内仍输出但标记为"非标准振幅"） |
| **成交量未萎缩** | 标黄警告但不直接拒绝（可能因为大盘放量带动的伪萎缩） |
| **布林带带宽数据不足** | 跳过此条件（`bb_enabled` 自动降级为 false） |
| **基部后有快速拉升 > 5%** | 视为突破已发生，不再重新识别 |

## 9. 与其他引擎的对比

| 维度 | 杯柄 | 双重底 | 扁平基部 |
|---|---|---|---|
| 识别难度 | 中等（需要定位杯体+柄部） | 较高（需要四节点定位） | **较低（滑动窗口+振幅判断）** |
| 假阳性风险 | 中 | 中-高 | **低**（极窄振幅是强约束） |
| 信号频率 | 较多 | 中等 | **较少（条件严格）** |
| 突破后胜率 | 中等 | 中等 | **较高（信号质量好）** |
| 计算复杂度 | O(N²) | O(N) | **O(N)（滑动窗口单次扫描）** |
