# 指数强度回测页面产品说明 v2.1

> 最后更新：2026-05-06  
> 对应引擎：`src/detectors/index_rs.py`  
> 配置文件：`config/index_rs.yaml`

## 1. 回测页面总体需求

- 使用**小红书卡片风格**（与抛盘日回测、追盘日回测页面保持一致），共享 `web/shared/css/xhs-cards.css` 等样式文件
- 导航栏与抛盘日/追盘日回测页面互链，保持顶部banner统一
- 页面布局：**左侧参数配置面板 + 右侧结果展示区**
- 日期选择：使用日历选择器（`<input type="date">`），**选择单个交易日**，默认取最近一个交易日。RS计算为时间点快照，非时间区间。
- 用户调整参数后点击「计算」按钮，触发 `/api/index-rs` 请求，右侧展示结果
- 点击「保存配置」，将当前参数写入 `config/index_rs.yaml`，RS强度引擎从该文件读取参数
- **不持久化计算结果**，每次请求实时计算（与抛盘日/追盘日引擎设计一致）

## 4. 技术实现要点

### 4.1 样式
- 使用小红书卡片风格（`xhs-cards.css`），与抛盘日/追盘日回测页面一致
- 共享 `web/shared/css/base.css`、`web/shared/css/components.css`、`web/shared/css/theme.css`
- 图表使用 ECharts 5.5（`web/shared/js/echarts.min.js`）

### 4.2 数据流
```
前端参数 → GET /api/index-rs?pool=xxx&date=yyyy-mm-dd
         → server.py 调用 index_rs.detect()
         → 返回 JSON { pools: { pool_name: { rankings, tiers, top10 } } }
         → 前端渲染表格和图表
```

参数保存：
```
前端参数 → POST /api/config?signal_type=index_rs
         → 写入 config/index_rs.yaml
```

### 4.3 引擎改动
- L3 增加 MA 均线判定：`close > MA(ma_days)`，`ma_days` 默认20，范围10~60
- 配置 `config/index_rs.yaml` 需增加 `ma_days` 字段
- 共振验证字段保留但 v1 不计算

### 4.4 文件结构
```
web/index-rs-backtest/
├── index.html       ← 看板页面
└── js/
    ├── main.js      ← 主逻辑（参数面板交互、API调用、渲染）
    └── table.js     ← 表格渲染（可选拆分）
```

## 5. 验证用例

- market 池 29 个指数，最高 RS=99，最低 RS=0，排名无矛盾
- sector_l1 池 11 个指数，RS 梯度合理不重复
- L1 绝对强势池筛选正确（同时满足 RS_120≥90、RS_250≥85、RS_60≥80）
- 三层池优先级互斥：满足 L1 的不再进入 L2/L3
- 保存配置后刷新页面，参数从 `config/index_rs.yaml` 正确回读
- 卡片4控件始终置灰不可操作
- L3 均线条件：收盘价低于 MA 的指数不进入 L3
