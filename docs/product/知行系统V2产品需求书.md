# 知行系统 · V2 产品需求书

> 版本：v2.0
> 日期：2026-05-20
> 状态：设计完成，待开发

---

## Problem Statement

V1 实现了观察池筛选 → 复核浏览 → 自选池管理 → 交易记录的基础闭环。但两个关键链路断裂：

1. **观察池没有形态信号**：`signals_json` 字段全空，用户在观察池和复核页中看不到 K 线图上的技术面标记
2. **持仓后完全没有监控**：买入后系统沉默——没有告警、没有止损提醒、没有引擎持续扫描。用户不知道何时该减仓、何时该清仓
3. **总览仪表盘不完整**：index.html 只有盈亏和持仓数，缺总资产、总仓位、现金等关键快照信息

V2 补齐这三个缺口，让知行系统从"记录工具"升级为"纪律工具"。

---

## Solution

V2 包含三个模块，按依赖顺序排列：

| 模块 | 做什么 | 依赖 |
|------|--------|------|
| **A. 形态信号扫描** | `observation.py` 从 `pattern_scan_signals` 表读取信号，写入观察池 `signals_json` | 已有的 `daily_pattern_scan.py` |
| **B. 复核详情页增强** | K 线图信号标记 + RS 副图 + 风险收益比卡片 + 凯利仓位建议 | 依赖 A 的信号数据 |
| **C. 持仓监控 + 告警** | 每日扫描持仓标的，聚合引擎信号，输出告警灯和建议 | 依赖 A 的引擎调用能力 |
| **D. 总览仪表盘增强** | index.html 补齐总资产/仓位/现金/最大回撤/盈亏比 | 独立 |

## User Stories

### A. 形态信号扫描

1. 作为一个观察池浏览者，我希望在观察池表格中看到每只股票的形态信号数量，这样我可以快速判断哪些股票有技术面异动。
2. 作为一个投资者，我希望观察池日更时自动从 `pattern_scan_signals` 表读取当天信号，不需要手动触发。
3. 作为一个系统运维者，我希望信号数据持久化在数据库中（而非仅存 JSON 文件），多日历史可追溯。

### B. 复核详情页增强

4. 作为一个投资者，我希望复核页 K 线图上标注所有形态信号标记点，不同引擎用不同颜色区分（参考 pattern-scan 的图例方案）。
5. 作为一个投资者，我希望在 K 线图下方看到 RS 线副图，直观判断个股相对大盘的强度趋势。
6. 作为一个投资者，我希望看到完整的 CAN SLIM 七维详情——不仅是总分，还有每维具体数值。
7. 作为一个投资者，我希望看到 L（领涨）维度信息：该股所属行业 RS 排名。
8. 作为一个投资者，我希望看到 I（机构）维度信息：机构持股变化。
9. 作为一个投资者，我希望看到 M（大盘）维度信息：大盘阶段、风险级别、建议仓位。
10. 作为一个投资者，我希望看到量价结构摘要——过去 25 日吸筹/出货日数量。
11. 作为一个投资者，我希望系统自动计算理想买点（形态引擎的 breakout_price，未检出时回退 200 天最高价）。
12. 作为一个投资者，我希望系统自动计算止损价（买入价 × 0.92）。
13. 作为一个投资者，我希望看到潜在收益空间（前高距离）和风险收益比。
14. 作为一个投资者，如果风险收益比 < 2，系统应明确提示警告。
15. 作为一个投资者，我希望看到基于凯利公式的仓位建议——根据历史胜率和盈亏比动态计算。
16. 作为一个投资者，如果输入单笔最大可接受亏损，系统告诉我最多可买多少股。
17. 作为一个投资者，V1 阶段（信号数据不全）复核页应标注"⚠ 基于简单前高，未结合形态分析"，直到 A 模块完成。

### C. 持仓监控 + 告警

18. 作为一个持仓者，我希望系统每天自动扫描我的持仓标的，告诉我哪些有风险、哪些可以安心持有。
19. 作为一个持仓者，我希望告警按优先级排列：止损触发 > 大盘恶化 > 个股走弱 > 见顶信号。
20. 作为一个持仓者，我希望止损触发或大盘风险=危险时显示红色告警灯，其他警告显示黄色，正常显示绿色。
21. 作为一个持仓者，我希望每条告警可以标记"已知晓"，确认时间被记录——下次复盘时我知道自己看到了但有没有行动。
22. 作为一个持仓者，我希望在持仓监控页面上点击"立即扫描"手动触发一次检查，不等待每日定时任务。
23. 作为一个持仓者，我希望持仓列表每行显示：代码、名称、买入日、买入价、现价、浮盈%、持股天数、告警灯、操作入口。
24. 作为一个持仓者，我希望有告警的持仓排在前面（红 > 黄 > 绿），点击某条进入详情看到完整告警清单。

### D. 总览仪表盘增强

25. 作为一个投资者，我希望在知行总览页（index.html）看到完整的投资快照：总资产、总市值、现金、总仓位百分比。
26. 作为一个投资者，我希望在总览页看到累计盈亏、胜率、最大回撤、盈亏比。
27. 作为一个投资者，我需要手动输入总资产金额（因为系统不连接券商），系统自动计算现金 = 总资产 - 总市值。

## Implementation Decisions

### 架构决策

1. **信号来源**：不独立跑引擎。消费 `pattern_scan_signals` 表（由 `daily_pattern_scan.py` 每天写入）。
2. **信号不区分买卖**：`signals_json` 存完整信号数组（带 `type` / `source` / `date` / `details`），前端和 K 线图按需分层。
3. **持仓监控**：每日自动（挂 `daily_update.py`）+ 手动按需（前端按钮），双模式并存。
4. **告警确认**（方案 C）：确认写入 `discipline_alerts.acknowledged`，条件仍在时下次生成新告警。

### 数据架构决策

5. **表结构合并**（V2）：`discipline_observation_pool` 的 `buy_signals_json` + `sell_signals_json` → `signals_json`（一个 TEXT 列）。
6. **总览页数据**：总资产由用户手动输入，存 `discipline_rules_config` 表。总市值 = SUM(持仓 buy_qty × 最新收盘价)。
7. **现价获取**：从 `daily_kline` 取每只持仓股的最新 close。

### 前端决策

8. **复核详情页**：三屏自上而下——K 线图（40%，含信号标记+RS副图+图例）→ 体检表（30%，CANSLIM七维+L/I/M+量价结构）→ 风险收益卡（30%，买点/止损/收益比/凯利）。
9. **K 线图信号标记**：复用 pattern-scan 的图例方案——不同引擎用不同颜色标记点，K 线图下方列出图例。
10. **RS 副图**：K 线图下方独立副图（复用 kline-chart.js 成交量副图的 layout 模式），数据来源 `stock_rs_daily.rs_line`。
11. **持仓监控页**：卡片式布局，每笔持仓一张卡。左侧持仓信息，右侧告警灯。红→黄→绿排序。告警卡展开显示告警消息，绿卡收起。
12. **总览仪表盘**：在 index.html 底部统计卡片区增加：总资产（可编辑）、总市值、现金、总仓位%、最大回撤、盈亏比。点击数字跳转到对应页面。
13. **自选池页**：维持 watchlist.html 现有设计（V1 已有买入/移除按钮），V2 增加按评估状态筛选。

### 理想买点与风险收益

14. **理想买点**（Grill 决策 2）：优先从形态引擎取 `breakout_price`；未检出时回退到 200 天最高价。
15. **风险收益计算**：纯前端 JS 计算，不依赖外部引擎。止损价 = 买入价 × 0.92，潜在收益 = (前高 - 当前价) / 当前价。
16. **凯利公式**：`f* = (bp - q) / b`，b = 平均盈亏比，p = 历史胜率（从 `discipline_trades` 已平仓记录汇总）。
17. **量价结构**：从 `daily_kline` 计算过去 25 日的吸筹/出货日数量（价涨量增 = 吸筹日，价跌量增 = 出货日）。

## Testing Decisions

### 测试策略

只测试外部行为，不测试实现细节。测试应模拟用户的真实操作流程。

### 测试重点

- **形态信号采集**（模块 A）：验证 observation.py 读取 pattern_scan_signals → 写入 signals_json 的正确性。测试边界：当天无信号数据时 signals_json 为 NULL
- **复核 API**：验证 review API 返回新增的 L/I/M 维度和 RS 线数据
- **持仓监控**（模块 C）：验证告警优先级排序正确（止损 > 大盘 > 个股 > 信号），验证告警灯颜色规则
- **凯利公式**：验证前端计算逻辑（和手算结果对比）

### 已有测试参考

项目已有回测看板开发标准（`docs/dev/回测看板开发标准.md`），新模块测试风格保持一致。

## Grill Decisions

1. **RS 线展示**（方案 A）：K 线图下方独立副图
2. **理想买点**（方案 A）：形态引擎优先，回退 200 天最高价
3. **V1 缺失信号处理**：展示但标注"基于简单前高"
4. **版本归属**：复核详情页全部功能划入 V2
5. **信号持久化**：存 `pattern_scan_signals` 表，多日可追溯
6. **K 线图信号标记**：复用 pattern-scan 的图例方案
7. **告警优先级**：止损 > 大盘 > 个股 > 信号
8. **监控频率**：每日自动 + 手动按需
9. **告警确认**：方案 C（确认写入 acknowledged 字段）
10. **持仓监控页布局**：卡片式，红→黄→绿排序
11. **总览仪表盘**：index.html 增强，总资产手动输入
12. **自选池**：维持 V1 设计，V2 增加评估状态筛选
