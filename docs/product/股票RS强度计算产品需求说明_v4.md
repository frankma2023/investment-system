# 个股RS强度计算引擎 产品说明文档

## 1. 产品定位

本引擎用于每日计算全市场A股的**相对强度（Relative Strength, RS）**，是欧奈尔 CAN SLIM 体系中 **"L – 领军股"** 维度的核心量化工具。输出结果可直接用于双强选股、口袋支点、卖出预警等策略。

**在整体系统中的位置**：个股扫描模块的第一个功能。上游依赖指数扫描模块（已完成：指数 RS、拥挤度、机构吸筹出货、指数背离）；下游消费者包括：`stock_candidates_daily` 候选管线、`index_rs.yaml` resonance 指数共振判断。

## 2. 核心理念（欧奈尔原教旨）

- **RS 评级（RPS）**：将个股过去 N 日的涨幅在全市场内排序，转化为 1～99 的百分位分数。**RPS ≥ 80 才值得关注，90 以上为超级强势股**。
- **多周期**：长期（250日）确认趋势稳定性，中期（120日和60日）确认趋势变化，短期（20日）捕捉爆发动能。若长期（250日）和短期（20日）两者共振（双强）是最佳买点，中期（120日和60日）强度也计算用于辅助判断。
- **RS 线**：个股收盘价 / 大盘收盘价，观察相对强度趋势，背离是重要卖出信号。
- **全市场基准**：采用中证全指（000985.CSI），避免风格偏好。

## 3. 输入数据

| 数据类型 | 字段 | 来源 | 更新频率 |
| :--- | :--- | :--- | :--- |
| 个股日线 | 前复权收盘价、成交量 | 理杏仁 API | 每日收盘后 |
| 指数日线 | 中证全指（000985）close | index_daily_kline 表 | 每日收盘后 |
| 基础信息 | 上市日期、上市状态 | stock_basic 表 | 按需 |

**关于前复权价格的使用策略**（重要）：

本项目使用的理杏仁 API 返回的价格为**前复权价格**（adj_close = close / cf × latest_cf）。前复权会改写历史——当某只股票除权除息后，其全部历史收盘价都会被比例缩放。这意味着：

- 同一天、同一只股票的 RPS，用"今天的复权基准"算和用"去年的复权基准"算，数值会不同（point-in-time 漂移）。
- **但这不影响每日选股的准确性**——因为 RPS 是横截面排名：某一天 5000 只股票用同一个复权基准计算，排名自洽。

**核心策略：每次全量重算，不保存历史 RPS 值。** 每日从 `daily_kline.adj_close` 重新计算 ret_20/ret_250 → RPS 排名 → 选股。不使用历史缓存 RPS 做跨日策略决策。如需回测 panel 展示"历史上某天的 RPS"，应在回测看板标注"当前复权基准计算，非 point-in-time"。

## 4. 量化计算规则

### 4.1 涨跌幅计算

对每只股票及基准指数，分别计算两个时间窗口的区间涨跌幅：

\[
\text{Ret}_{N} = \left( \frac{P_t}{P_{t-N}} - 1 \right) \times 100\% \quad (N=20,\ 250)
\]

- 使用**前复权收盘价**（`daily_kline.adj_close`），消除分红、送股、配股的断层。
- N=20/60/120/250 均指**交易日**，非日历日。回溯时按交易日记数（停牌日不在 K 线序列中）。
- 若有效交易日不足 N/2（即 ret_20 需要 ≥10 条有效 K 线，ret_250 需要 ≥125 条），该周期不参与排名（标记为 NaN）。
- 若股票在 T-N 位置因停牌无数据，依次回退至最近的有效交易日。

### 4.2 RS 评级（RPS）计算

在每个交易日，分别对 **250 日涨跌幅** 和 **20 日涨跌幅** 在全市场有效股票内排序，计算百分位排名：

\[
\text{RPS} = \left(1 - \frac{\text{rank} - 1}{N_{\text{valid}} - 1}\right) \times 99
\]

- 结果四舍五入取整，范围 **0～99**。
- RPS=90 表示该股票表现优于市场中 90% 的股票。
- 新股（有效交易日不足）、停牌股票不参与排名，也不出现在结果中。

**边界情况**：

- **N_valid = 1**（只有一只股票有效参与排名）：分母为 0，公式失效。此时该股票 RPS 设为 50（中性值）。
- **并列排名（tie-breaking）**：多只股票 ret 完全相同时，使用 min rank（即并列者共享同一 rank 值），保证公平。
- **ST/\*ST 股票**：从 `stock_basic.listing_status` 读取，`special_treatment` 和 `delisting_risk_warning` 状态的股票默认剔除（可通过配置项 `exclude_st` 关闭）。
- **可选流动性过滤**：近 20 日均成交额 < 5000 万元的股票剔除（可通过配置项 `min_avg_amount_20d` 调整）。此项在 RPS 排名之前执行——被过滤的股票既不参与排名，也不计入有效样本数。

### 4.3 RS 线

\[
\text{RS\_Line} = \frac{\text{个股前复权收盘价}}{\text{中证全指收盘价}}
\]

- 原始值量级不直观（个股几十元 / 指数几千点）。前端展示时应做归一化：取基日 RS 线 = 100，`RS_Line_Normalized_t = (RS_Line_t / RS_Line_base) × 100`。基日建议使用上市首日或引擎上线首日。
- 使用场景：股价新高但 RS 线未新高 → 顶背离，警惕转弱。RS 线持续走低 → 个股持续跑输大盘。

## 5. 双强选股标准（核心输出）

每日筛选满足以下**任一模式**的股票：

| 模式 | RPS_250 | RPS_20 | 含义 | 操作参考 |
| :--- | :--- | :--- | :--- | :--- |
| **稳健龙头** | ≥ 90 | ≥ 85 | 长期优异+近期持续强势 | 均线附近低吸 |
| **加速爆发** | ≥ 80 | ≥ 95 | 长期良好+短期急剧爆发 | 突破放量时追入 |

输出字段：日期、股票代码、名称、RPS_250、RPS_20、收盘价、成交额、所属行业（可选）。

## 6. 数据处理规则

| 场景 | 处理方式 |
| :--- | :--- |
| **新股** | 上市不满 **250 交易日**不计算 RPS_250；不满 20 日不计算 RPS_20 |
| **停牌** | 停牌期间不更新，复牌后按实际交易日回溯计算 |
| **ST/\*ST** | 默认剔除（从 `stock_basic.listing_status` 过滤。可配置关闭） |
| **流动性过滤** | 可选剔除近 **20 日均成交额 < 5000 万** 的股票（可配置） |
| **退市** | `listing_status='delisted'` 的股票不参与 |
| **前复权价格策略** | **每次全量重算，不保存历史结果。** 避免跨日缓存导致的复权基准漂移 |

## 7. 和现有系统的衔接

### 7.1 行业 RS（已存在）

行业 RS 排名在 `sector_rs_daily` 表中已完成。个股 RS 引擎上线后，可进一步计算"行业内个股 RPS 排名"（同一中证一级行业/二级行业/主题指数/策略指数内独立排序），提供更精细的选股信号。

### 7.2 指数共振（待启用）

`config/index_rs.yaml` 中的 resonance 配置当前 `enabled: false`。其逻辑为：

- 检查某指数的成分股中，RPS ≥ `strong_stock_rs`（默认 80）的个股占比
- 若强共振（≥ 40% 强势股 + 中位数 RS ≥ 75），则该指数的强势是"普涨"而非"权重股拉指数"

个股 RS 引擎上线后，将 `enabled` 改为 `true` 以激活此功能。

## 8. 欧奈尔理念验证清单（开发后必做）

| 验证项 | 通过标准 |
| :--- | :--- |
| RPS 分布 | 每日全市场中位数约为 50，最高 99，最低 1 |
| 双强股票数量 | 牛市通常 30～100 只，熊市可能仅 0～10 只 |
| 滞后性 | 回测确认 RPS 向上突破后的股票未来 1~3 个月超额收益显著 |
| 前复权一致性 | 同一日期用不同复权基准日计算，排名相关系数 > 0.99 |

## 9. 数据库设计决策

- **不建持久化 RPS 结果表。** 前复权价格每天变化，历史 RPS 值的准确含义随复权基准漂移。每日选股使用当日新鲜计算结果。
- **如需用于前端展示历史 RPS 曲线**：在回测看板中标注"当前复权基准计算，非 point-in-time"。如未来有严格的 point-in-time 回测需求，需额外存储后复权价格或复权因子历史快照。
- **旧表 `rs_daily`**：历史遗留表，可删除。新引擎不再写入该表。

## 11. 技术实现设计（2026-05-07 更新）

### 11.1 技术栈

- **Polars**：高表结构（tall），stock_code × date × adj_close，利用 lazy API 窗口函数
- **高表优于宽表**：股票是分组维度（面板数据），`.over("stock_code")` shift 和 `.over("date")` rank 是 Polars 原生高效操作
- **内存路径**：5000 只股票 × 2600 交易日 ≈ 1300 万行，高表 3 列约 200MB 内存，Polars 多线程自动并行

### 11.2 执行计划

```
数据加载 → 预排序(stock_code, date) → 去重防御 → 预计算 avg_amount_20d
  → 过滤 ST/退市 → 流动性过滤 → 计算 _bar_index
  → shift(20/60/120/250).over(stock_code) 计算 ret_N
  → 标记 valid_N（_bar_index >= N/2）
  → rank("min").over(date) 计算 rank_N
  → RPS 公式（分母用 max_horizontal 防御除零）
  → 双强筛选 → 输出
```

### 11.3 关键防御

| 问题 | 方案 |
| :--- | :--- |
| 数据乱序/重复 | 排序后 `unique(subset=["stock_code","date"], keep="last")` |
| 除零（单只有效股票） | `pl.max_horizontal(rank_max - 1, 1)` 保证分母 ≥ 1 |
| 并列排名 | `rank("min")` 并列共享同一 rank |
| N_valid=1 时 RPS | 设为 50（中性值），特殊处理 |

### 11.4 RPS_60 和 RPS_120

与 RPS_250 和 RPS_20 在同一趟 lazy 扫描中计算（无额外开销），用于辅助判断趋势拐点，在看板上展示但不参与双强选股。

### 11.5 RS 线归一化

统一取引擎上线首日（2026-05-07）为基日，所有股票 RS 线统一起点 = 100：
```
RS_Line_Normalized_t = (个股 adj_close_t / 中证全指 close_t) / (个股 adj_close_base / 中证全指 close_base) × 100
```

### 11.6 数据库策略

- **不建持久化 RPS 结果表**：每次全量重算，避免前复权基准漂移
- **旧表 `rs_daily`**：暂时保留，新引擎上线稳定后删除
- **API 输出**：`GET /api/stock-rs?date=YYYY-MM-DD` 返回当日全市场 RPS，`GET /api/stock-rs/double-strong` 返回双强股列表
