# 基部突破检测引擎 — 设计思路与推导过程

> 记录从"10 个引擎零信号"到"1 个 Layer 1 核心引擎 + N 个 Layer 2 标注器"的完整推导过程。

---

## 1. 起点：越多引擎，越少信号

我们在 2025 年陆续开发了以下形态识别引擎：

| 引擎 | 参数量 | 串联条件数 |
|---|---|---|
| 杯柄形态 | 27 | ~15 个 |
| 碟形基部 | 31 | ~18 个 |
| VCP 形态 | 24 | ~12 个 |
| 标准突破日 | 21 | ~12 个 |
| 高紧旗 | 16 | ~10 个 |
| 双重底 | ~15 | ~10 个 |
| 一底更比一底高 | 8 | ~8 个 |
| 递升底部 | 9 | ~8 个 |
| 箱体 | 8 | ~8 个 |
| 扁平基部 | ~10 | ~8 个 |

每个引擎各自有几十个参数，各自独立检测"突破日"并判断"形态类型"。结果是：

- **杯柄**：27 参数串联通过率 ≈ 0.24%，实测零信号
- **碟形**：切割法 + 双层窗口 + 弧线圆润 + 对称性 + 成交量，太紧
- **高紧旗**：前置暴涨 ≥ 100% 几乎排除所有 A 股标的
- **所有引擎**：参数越精细，信号越少，最终归零

### 根本问题

每个引擎都把两件事混在一起做：

1. **检测是否是突破日**（择时）
2. **判断是什么形态**（分类）

形态判断的严格参数把突破日信号也过滤掉了。引擎越多，信号越少。

## 2. 倒退一步：8 种形态的共性拆解

我们逐一拆解了 O'Neil 体系中的 8 种基部形态，寻找它们共享的底层结构。

### 杯柄形态 (Cup with Handle)

```
上涨 → 前高 → 回调 12%~33% → 杯底 → 回升到杯口附近 →
横盘 1~2 周（柄）→ 突破前高
```

核心元素：前高、回调、杯底、回升、突破。

### 碟形基部 (Saucer Base)

```
上涨 → 前高 → 缓慢圆弧下行 8%~20% → 平坦碟底 3~5 周 →
温和回升到前高附近 → 突破
```

核心元素：前高、回调（圆弧）、碟底（平坦）、回升、突破。

### 双重底 (Double Bottom)

```
上涨 → 前高 → 第一波回调 → 反弹 → 第二波回调（不创新低或略低）
→ 回升 → 突破前高
```

核心元素：前高、两次回调、谷、回升、突破。

### VCP (Volatility Contraction Pattern)

```
上涨 → 前高 → 回调 → 反弹 → 回调 → 反弹 → 回调（每次幅度递减）
→ 量缩到极致 → 突破
```

核心元素：前高、多次回调（逐次收缩）、谷、回升、突破。

### 高紧旗 (High Tight Flag)

```
暴涨 ≥ 100% → 前高 → 极窄旗形整理 1~3 周（振幅 < 5%）→ 突破
```

核心元素：前置暴涨、前高、极浅回调、短暂整理、突破。

### 箱体 (Box / Rectangle)

```
上涨 → 前高 → 矩形横盘 8%~15% 振幅 → 突破箱顶
```

核心元素：前高、横向整理、回调（有限）、突破。

### 扁平基部 (Flat Base)

```
上涨 → 前高 → 极浅回调 < 15% → 横向整理 6~8 周 → 突破
```

核心元素：前高、浅回调、横向整理、突破。

### 一底更比一底高 / 递升底部

```
上涨 → 前高 → 第一次回调 → 反弹 →
第二次回调（底高于第一次底）→ 突破前高
```

核心元素：前高、多次回调、谷底抬高、突破。

---

## 3. 发现：8 种形态的最小公分母

把 8 种形态的"核心元素"叠加在一起，发现它们共享完全相同的底层骨架：

```
前高 ───────────────────────────────── 突破买点
   \                                   /
    \    (任何形态的回调/整理区间)      /
     \                                /
      谷（基部最低点）────────────────┘
```

用一句话描述这个共同结构：

> **一段上涨之后，价格见顶回落/整理，形成一个谷（或整理区间），然后从谷回升并重新突破前高。**

这就是所有 O'Neil 基部形态的**充要底层结构**。8 种形态的区别仅仅在于：

- 回调的形状（直线下跌 vs 圆弧下行 vs 横盘）
- 回调的深度（浅 vs 深）
- 谷的形态（V 形尖底 vs 平坦碟底 vs W 双底）
- 整理的时间（短 vs 长）
- 回升的路径（直线上涨 vs 逐级反弹）

但这些差异**不影响"是否突破"的判断**。一个杯柄和一个碟形，在突破前高那一刻，价格和成交量行为是完全一样的。

---

## 4. 新架构：Layer 1 / 2 / 3 分离

基于这个发现，我们把原来的"检测 + 分类混合"架构拆成三层：

```
旧架构：K线 → 杯柄引擎(27参数) → 杯柄信号(几乎为0)
             → 碟形引擎(31参数) → 碟形信号(几乎为0)
             → ...
        每个引擎独立输出，互相不知道对方的存在

新架构：K线 → Layer1 核心突破引擎(10参数) → 全部突破信号（~1.4% 通过率）
                  │
                  ├→ Layer2 杯柄标注器  → 杯柄得分 0.85
                  ├→ Layer2 碟形标注器  → 碟形得分 0.30
                  ├→ Layer2 VCP 标注器  → VCP 得分 0.65
                  └→ ...
             所有标注器并行运行，互不干扰
```

- **Layer 1** 负责**择时**——只回答"今天是不是一个基部突破日"。参数少（10 个），通过率可控。
- **Layer 2** 负责**标注**——回答"这个突破日有什么形态特征"。不参与过滤，只提供信息。
- **Layer 3** 交给用户自主过滤。

## 5. Layer 1 为什么只需要 10 个参数

原来的每个形态引擎把"突破条件"和"形态条件"混在一起。我们把"形态条件"全部剥离到 Layer 2，Layer 1 只保留所有形态共享的突破结构验证：

| # | 参数 | 作用 | 所有 8 种形态都满足？ |
|---|---|---|---|
| 1 | `lookback` | 回溯窗口，找前高 | ✅ |
| 2 | `min_prior_advance` | 前高之前有过上涨（≥30%） | ✅ |
| 3 | `drawdown_min` | 最小回调深度（5%）| ✅ 排除噪音波动 |
| 4 | `drawdown_max` | 最大回调深度（40%）| ✅ 排除崩盘 |
| 5 | `min_base_days` | 前高距今最少天数（20天） | ✅ 基部至少 4 周 |
| 6 | `min_descent_days` | 前高到谷最少天数（10天） | ✅ |
| 7 | `min_recovery` | 回升到前高比例（75%） | ✅ |
| 8 | `breakout_vol_ratio` | 突破日量比（≥1.3） | ✅ |
| 9 | `require_green` | 是否要求阳线 | ✅ |
| 10 | `close_position_min` | 收盘位置分位（≥50%） | ✅ |
| — | `sma50_check` | 收盘 > SMA50 | ✅（开关型） |

**不在 Layer 1 的参数**（这些都是 Layer 2 的事情）：
- 杯柄：柄部时间/回撤/位置/缩量、杯底平坦性、杯口回升到 95% 以上
- 碟形：弧线圆润性、双层窗口平坦性、对称性、碟底量萎缩
- VCP：逐次收缩幅度、每次反弹高度递减
- 高紧旗：前置暴涨 ≥ 100%、旗形振幅 < 5%
- 箱体：矩形振幅 8%~15%、横盘天数
- 双重底：两底高度差、反弹高度
- ...

这些条件是用于**区分形态**的，不是用于**判断突破**的。把它们从 Layer 1 剥离后，信号量从几乎为零恢复到 ~1.4%（每只股票每年约 3.5 个信号）。

## 6. 谷底驱动法：为什么新引擎能准确定位前高

Layer 1 的核心算法创新是**谷底驱动法**。

### 旧方法（简单 max/min）

```
前高 = lookback 内最高收盘价
谷   = 前高之后到检测日之间的最低收盘价
```

问题：如果 lookback 内含有一个更老的旧周期高点（比如 2015 年牛市的顶部），它会被错误地当作"前高"，导致回调深度和回升比例全部计算错误。

### 新方法（谷底驱动法）

```
1. 先找谷：检测日之前 ≥20 天内最低收盘价
2. 从谷逐日向前回溯，每步记录路上碰到的最高收盘价
3. 在回溯路径上找第一个"局部高点"（前后 ±10 天内最高）
4. 验证局部高点到谷的回调 ≥ 5%
5. 该局部高点即为前高
```

**为什么正确**：

- 谷是最新形成的，必然属于当前基部周期
- 从谷向前回溯是**确定性**的——每次只走一步，碰到第一个局部高点就停
- 旧周期的高点即使存在，只要它与谷之间有 ≥ 5% 的下跌，就会在回溯过程中被自然定位到；如果没有 ≥ 5% 的下跌，说明它和谷在同一个周期内，作为前高是正确的
- 无需碟形基部的"切割 A/B/C"（因为我们只找"一个"前高，不要求左侧有多段下跌）
- 回调范围 5%~40% 本身已经兜底——如果回溯碰到的局部高点到谷的回调 < 5%，跳过，继续往前找

### 与其他引擎方法对比

| 方法 | 前高定位 | 适用性 |
|---|---|---|
| 杯柄切割法 | 从杯底逆向爬坡，遇 ≥ 20% 大波动切割 | 需要明确的杯形 |
| 碟形切割法 | 三段切分（A/B/C），验证弧线圆润 | 仅适用于圆弧下行 |
| 简单 max/min | lookback 内最高价 | 旧周期干扰严重 |
| **谷底驱动法** | 从谷向前回溯找第一个局部高点 | **通用，无形态依赖** |

谷底驱动法是所有 8 种形态的最小公分母——它不需要任何关于"回调形状"的先验假设，只要求"存在一个谷、谷之前有一个高点"。

## 7. 信号量估算与校验

以 10 个参数的宽松默认值，串联通过率约 1.4%：

```
单条件通过率:
  - 前高在谷前:           95%
  - 前高距今 ≥ 20 天:     70%
  - 回调 ∈ [5%, 40%]:    85%
  - 回升 ≥ 75%:          60%
  - 前置上涨 ≥ 30%:      40%
  - 突破量比 ≥ 1.3:      50%
  - 阳线:                 55%
  - 收盘高位 ≥ 50%:      65%
  - 收盘 > SMA50:         55%

串联通过率: 0.95 × 0.70 × 0.85 × 0.60 × 0.40 × 0.50 × 0.55 × 0.65 × 0.55
          ≈ 1.4%
```

每只股票每年期望信号数 ≈ 250 × 0.014 ≈ **3.5 个/年**。

A 股约 5000 只 → 每年约 **17,500 个信号**。

对比：
- 旧杯柄引擎：0.24% 通过率 → 0.6 个/年/股，且形态判断打满后实测为零
- 旧碟形引擎：条件更多（31 参数），通过率更低
- Layer 1 引擎：**1.4% 通过率**，信号量充足，足以支撑后续 Layer 2 标注和用户筛选

## 8. 关键设计决策

1. **分离检测与分类**：Layer 1 只做突破日检测，形态判断全部下放 Layer 2。这是整个架构的根本转变。

2. **谷底驱动法**：摈弃了所有形态特有的前高定位算法（切割法、三段切分、前高距离限制等），用一个通用算法替代。

3. **参数宽松化**：`drawdown_min=5%` 非 `8%`，`drawdown_max=40%` 非 `20%`，`min_recovery=75%` 非 `85%`。宁可多报，不可漏掉。

4. **形态得分而非是非判断**：Layer 2 不输出 `true/false`，而是输出 `0~1` 匹配得分。用户可根据得分自主过滤，不会被引擎的参数选择限制。

5. **前高定位通用化**：所有形态共享同一个前高定位逻辑。不同形态的前高可能略有不同（杯柄的前高是杯左顶，碟形的前高是左侧高点），但在谷底驱动法下统一为"谷之前最近的有效局部高点"。
