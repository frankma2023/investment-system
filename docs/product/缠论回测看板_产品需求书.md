## 1. 总体需求

- 遵循全站 **hanako-glass 设计系统**（`web/shared/css/hanako-glass.css`），共享导航栏 `web/shared/js/nav.js`
- 导航栏与所有页面互链，当前页面在下拉菜单中标记 `class="active"`
- 页面支持**单周期模式**（日/周/月任选其一）和**多周期模式**（日周月三图同屏 + 共振面板）双模式切换
- 当前阶段：**实时分析看板**（选定股票/指数 → 即时计算缠论结构 → 图表+数据面板展示）
- 后续阶段：演进为回测看板（参数配置 + 历史回测 + 信号统计），本节为远期规划预留

## 2. 产品背景

缠中说禅（缠论）是 A 股市场广泛使用的技术分析体系，核心由分型→笔→线段→中枢→背驰→买卖点六个层级递推。在欧奈尔 CAN SLIM 体系中引入缠论分析维度，补齐纯技术面的形态结构判断能力。

### 2.1 当前能力矩阵

| 缠论层级 | 实现状态 | 说明 |
|---------|:--:|------|
| 分型 | ✅ CZSC 计算 | 顶分型/底分型，91个（500根K线） |
| 笔 | ✅ CZSC 计算 | 含方向/高低点/力度/斜率/角度 |
| 线段 | ❌ 未实现 | 待 P1 开发 |
| 中枢 | ✅ 自研算法 | 连续3笔重叠法 + 延伸合并 |
| 背驰 | ✅ 自研算法 | 趋势背驰（间隔同向笔力度衰减）+ 盘整背驰 |
| 买卖点 | ✅ 自研算法 | 一买/一卖 + 三买/三卖 |
| 多周期联立 | ✅ 自研算法 | 日/周/月并行 + 共振评分 |

## 8. K 线周期合成

### 8.1 合成规则

周/月模式下，图表 K 线从日线合成而非直接使用日线。CZSC 笔/中枢计算仍基于日线 RawBar（freq 参数控制级别）。

| 周期 | Pandas Resample | 标注日期 | K线数量（1200 根日线→） |
|------|----------------|---------|------------------------|
| 周线 | `resample('W-FRI')` | 该周周五 | ~132 根 |
| 月线 | `resample('ME')` | 该月最后交易日 | ~31 根 |

### 8.2 合成 OHLC

```
open  = 周期内第一个交易日开盘价
high  = 周期内最高价
low   = 周期内最低价
close = 周期内最后一个交易日收盘价
volume = 周期内成交量合计
```

### 8.3 坐标映射

CZSC 输出的笔日期为日线级别（如 `2026-03-24`），需通过 `_build_period_index_map()` 映射到合成后的周/月 K 线索引。映射逻辑：

```
周线：日线日期所在周的周五 → 周五在合成日期列表中的索引
月线：日线日期所在月的末交易日 → 月末日在合成日期列表中的索引
```

### 8.4 自动 limit 放大

为防止合成后 K 线过少，后端自动放大日线读取量：

| 请求周期 | 实际读取日线量 |
|---------|--------------|
| D | max(limit, 750) |
| W | max(limit × 3, 900) |
| M | max(limit × 3, 900) |

## 9. 深浅色主题

### 9.1 切换机制

- 全局 `toggleTheme()` 函数，`localStorage` 持久化
- 图表通过 `theme` 参数传入 API，后端返回对应配色 option
- 主题切换后自动重新分析（`analyze()` 重新调用）

### 9.2 浅色模式特殊处理

每个页面特有组件（`.mode-btn`、`.ctrl-btn`）必须有独立的浅色覆盖规则：

```css
html[data-theme="light"] .mode-btn { background: rgba(255,255,255,.6); }
html[data-theme="light"] .mode-btn.active { background: var(--accent-subtle); font-weight: 600; }
html[data-theme="light"] .ctrl-btn { background: rgba(255,255,255,.6); }
```

## 11. 已知限制与后续规划

### 11.1 当前限制

1. CZSC Rust 版 `to_echarts()` 不可用，ECharts option 全部手写
2. 周/月线 K 线从日线合成，非交易所原始周/月线数据
3. 线段（至少3笔构成）尚未实现
4. 背驰阈值 0.6 硬编码，不同品种可能需调参
5. 无全市场批量扫描，仅支持逐只分析
6. 无回测功能：无历史参数回测、无信号胜率统计

### 11.2 回测功能规划（远期）

参照其他回测看板（抛盘日/追盘日/拥挤度），未来回测模式需实现：

- 左侧 **参数配置面板**：背驰阈值、中枢最小宽度比例、买卖点过滤器
- 右侧 **回测结果区**：历史信号列表 + 胜率统计 + 收益曲线
- **保存/加载**：参数组合 → YAML 配置文件 → API 读写
- **日期区间**：日历选择器选择回测起止日期
- **信号统计**：信号总数/正确率/平均收益/最大回撤

### 11.3 待实现功能

| 功能 | 优先级 | 说明 |
|------|:--:|------|
| 线段计算 | P1 | 至少3笔构成线段，线段重叠构成高级别中枢 |
| 区间套买卖点 | P2 | 15F→60F→日线精确定位入场区间 |
| 全市场批量扫描 | P2 | 遍历全部股票/指数，输出信号排行榜 |
| 回测对比 | P3 | 缠论买卖点 vs 欧奈尔信号历史胜率 |
