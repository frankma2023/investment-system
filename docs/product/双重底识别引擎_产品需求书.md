# 双重底(Double Bottom)形态识别引擎 — 产品需求书

## 1. 产品定位

**双重底（Double Bottom）** 是 William J. O'Neil 定义的七种经典基部形态之一，在《笑傲股市》（How to Make Money in Stocks）中被列为仅次于杯柄形态的第二大可靠买入形态。

双重底形似字母 **W**：股价经历一段上涨后回调形成第一个底（左底），反弹至中间峰，然后再次下跌形成第二个底（右底），最后突破中间峰的高点完成形态。

**在整体系统中的位置**：个股扫描模块 > 基部识别引擎的子模块。与杯柄形态识别并列，共享同一套 K 线数据管道和回测看板。下游消费者包括：`stock_candidates_daily` 候选管线、口袋支点引擎（基部口袋支点类型需基部区间数据）。

**与杯柄形态的关系**：双重底是杯柄的"近亲"——两者的区别在于回调深度和底部结构。杯柄只有一个底部（U 形），双重底有两个底部（W 形）。双重底通常回调更深（20%~35%），第二底往往会小幅跌破第一底（震仓），形成"假跌破"后快速回升。

---

## 2. 核心理念

双重底的本质是：**机构资金分两次完成吸筹。** 第一次下跌时大资金开始买入（探左底），反弹到中间峰后不再追高，等待价格自然回落到更低的价位再次买入（探右底）。右底的成交量萎缩说明抛压衰竭——该卖的人已经卖完了。突破中间峰是"主力建仓完毕，开始拉升"的信号。

类比：W 的左半部分是"主力试盘——确认有支撑"，中间部分是"主力等待——让不坚定的散户割肉"，右半部分是"主力吃饱——不再给低价买入的机会"。

### 2.1 与杯柄形态的区别

| 维度 | 杯柄形态 | 双重底 |
|---|---|---|
| 底部数量 | 1 个（U 形杯底） | 2 个（W 形双底） |
| 回调深度 | 12%~33% | 20%~35%（通常更深） |
| 震仓特征 | 柄部小幅回调（<10%） | 右底可能跌破左底（假跌破） |
| 形成时间 | 7~65 周 | 通常 7~8 周以上 |
| 成交量 | 底部萎缩 | 左底量大，右底量更小（关键信号） |
| 突破点 | 柄部上沿 | 中间峰（W 的中点） |
| 出现频率 | 最常见 | 第二常见 |

## 3. 形态结构定义

```
        中间峰(Middle Peak) ─── 突破买点
       /    \
      /      \      ┌─ 柄部(可选)
     /        \    /
左峰          \  /
(前高)   左底  右底
(Left Bottom) (Right Bottom)
              ↑ 可能略低于左底(震仓)
```

### 3.1 关键节点

完整的时序链为五个节点：

```
前低 → 前高 → 左底 → 中间峰 → 右底 → 突破
```

| 节点 | 定义 | 作用 |
|---|---|---|
| **前低** | 前高之前的最低收盘价 | 前置上涨的起点。**不是左底，是前高之前的那个谷** |
| **前高** | 前低之后的最高收盘价 | 形态的上边界 |
| **左底** | 前高之后第一次回调的最低点 | W 的第一个底 |
| **中间峰** | 左底反弹后的最高点 | **突破买点 = 中间峰当日最高价 + 0.01** |
| **右底** | 中间峰之后第二次回调的最低点 | 应接近或略低于左底（震仓） |
| **突破日** | 收盘价突破中间峰的日期 | 买入信号日 |

## 4. 量化计算规则

### 4.1 前置条件

在寻找双重底之前，股票必须先找到**前低**和**前高**两个节点：

```
步骤 0.1: 在过去 200 个交易日内找到最高收盘价位置 = 前高
步骤 0.2: 在前高之前（时序上更早）找到最低收盘价位置 = 前低
        注意: 前低不是左底。前低是前高之前的谷，左底是前高之后的谷。
        时序: 前低 → 前高 → 左底 → 中间峰 → 右底
步骤 0.3: 前置上涨 = (前高 - 前低) / 前低 ≥ min_prior_advance（默认 30%）
步骤 0.4: 前高距今 ≥ prior_high_min_age 个交易日（默认 40）
步骤 0.5: 当前价格处于上升趋势中（参考 50 日均线斜率 > 0 或 RS ≥ 70）
```

**为什么前低不能和左底混淆**：如果错误地把左底当作前低，那么 `(前高 - 左底) / 左底` 衡量的是"从底部到顶部的总振幅"而非"前置上涨"。回撤越浅的股票这个值越小，会被错误地排除。这与标准突破引擎的 Bug #3 是同一类问题。

### 4.2 左右底识别（核心算法）

在 K 线序列中寻找以下结构：

#### 步骤 1：定位左底

```
定义: 在 [前高后第 N 天, 当前日期 - 20天] 范围内找到最低收盘价位置 = 左底
约束:
  - 左底距前高 ≥ 10 个交易日（不能刚见顶就到底）
  - 左底价格 ≤ 前高 × (1 - 0.15)（回调至少 15%，A 股波动大，放宽到此下限）
  - 左底价格 ≥ 前高 × (1 - 0.50)（回调不超过 50%，否则是崩盘不是基部）
```

#### 步骤 2：定位中间峰

```
定义: 在左底之后、右底之前找到最高收盘价位置 = 中间峰
约束:
  - 中间峰距左底 ≥ 5 个交易日
  - 中间峰价格 ≥ 左底 × 1.10（反弹至少 10%）
  - 中间峰价格 ≤ 前高 × (1 + middle_peak_tolerance)（默认容忍 2% 的微微突破；设置为 0 则严格不创新高）
```

#### 步骤 3：定位右底

```
定义: 在中间峰之后找到最低收盘价位置 = 右底
约束:
  - 右底距中间峰 ≥ 5 个交易日
  - 右底价格 ≤ 左底 × 1.03（右底接近或略低于左底，震仓特征）
  - 右底价格 ≥ 左底 × 0.85（不会大幅跌破左底，跌幅超 15% 说明趋势坏了）
  - 右底距当前日期 ≥ 5 个交易日（右底不是"刚刚发生"）
```

### 4.3 成交量验证

成交量是区分"真双重底"和"随机 W 形波动"的关键：

#### 左底成交量

```
条件: 左底附近（左底日 ± 2 天）的日均成交量 ≥ 50 日均量的 0.8 倍
说明: 左底是恐慌性抛售的终点，通常伴随放量
```

#### 中间峰成交量

```
条件: 中间峰附近（中间峰日 ± 2 天）的日均成交量 ≥ 50 日均量的 1.0 倍
说明: 反弹到中间峰时应有资金推动
```

#### 右底成交量（最关键）

```
条件: 右底附近（右底日 ± 2 天）的日均成交量 < 左底附近日均成交量的 70%
说明: 第二波下跌时量明显萎缩 → 抛压衰竭 → 主力控盘
```

### 4.4 右底后的反弹确认

```
条件: 右底到当前日期之间，存在至少一个交易日的收盘价 > 右底 × 1.05
说明: 右底已经确认，股价正在回升中，不是"还在下跌探底"
```

### 4.5 突破信号

```
定义: 今日收盘价 ≥ 中间峰当日最高价 + 0.01（A 股精度）
约束:
  - 突破日成交量 ≥ 50 日均量 × 1.4（放量突破）
  - 突破日为阳线（收盘 > 开盘）
  - 收盘位置 ≥ 日内振幅的 50%（收盘在高位，不是冲高回落）
```

### 4.6 可选：柄部（Handle）检测

欧奈尔指出，一些双重底在右底反弹到接近中间峰时，会形成一个小型柄部（类似杯柄形态的柄部），提供更低风险的买入点。

```
条件: 右底反弹后，在接近中间峰时出现一段小幅回调
  - 回调幅度 ≤ 8%（和杯柄的柄部要求一致）
  - 回调持续 5~15 个交易日
  - 回调期间成交量萎缩
  - 突破柄部上沿即买入，而非等待突破中间峰
```

此功能为可选（v2 实现）。

### 4.7 RS 确认（输出而非过滤）

```
输出: RPS_20, RPS_250 附加到每个识别的双重底信号中
建议: RPS ≥ 80 时信号有效性更高（人工综合判断，不硬过滤）
```

## 5. 信号输出格式

### 5.1 输出字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `date` | date | 突破信号日期（收盘突破中间峰当天） |
| `stock_code` | str | 股票代码 |
| `stock_name` | str | 股票名称（可选） |
| `prior_low_date` | date | 前低日期（前高之前的谷） |
| `prior_low_price` | float | 前低价格 |
| `prior_high_date` | date | 前高日期 |
| `prior_high_price` | float | 前高价格 |
| `prior_advance_pct` | float | 前置涨幅 % = (前高 - 前低) / 前低 |
| `left_bottom_date` | date | 左底日期 |
| `left_bottom_price` | float | 左底价格 |
| `middle_peak_date` | date | 中间峰日期 |
| `middle_peak_price` | float | 中间峰价格 |
| `right_bottom_date` | date | 右底日期 |
| `right_bottom_price` | float | 右底价格 |
| `right_undercut_pct` | float | 右底跌破左底幅度 %（正数 = 右底更低，震仓信号） |
| `decline_from_high` | float | 从前高到左底的回撤 % |
| `left_vol_ratio` | float | 左底成交量 / 50 日均量 |
| `right_vol_ratio` | float | 右底成交量 / 50 日均量 |
| `vol_contraction` | float | 右底量 / 左底量（< 0.7 为优） |
| `breakout_vol_ratio` | float | 突破日成交量 / 50 日均量 |
| `has_handle` | bool | 是否存在柄部 |
| `handle_pivot` | float/null | 柄部枢轴点价格（如有） |
| `rps_20` | int | RS 强度 20 日 |
| `rps_250` | int | RS 强度 250 日 |
| `middle_peak_to_now_days` | int | 中间峰距今多少个交易日 |

### 5.2 输出接口

- **API 端点**: `GET /api/double-bottom?stock=600519&date=2026-05-12`
- **批量扫描**: `GET /api/double-bottom/scan?date=2026-05-12&pool=watchlist`

## 6. 数据处理规则

| 场景 | 处理方式 |
|---|---|
| **数据不足** | 上市不满 180 交易日不识别（无法形成完整的双重底） |
| **停牌** | 停牌期间跳过，形态识别中不将停牌日计入"距今天数" |
| **ST/\*ST** | 默认剔除，沿用 `exclude_st` 配置 |
| **涨跌停** | 一字板日期排除（左底/右底不能是一字跌停，中间峰不能是一字涨停） |
| **左底未出现** | 前高后未出现 ≥15% 的回调 → 不产生信号（可通过 left_bottom_decline_min 调整） |
| **右底远低于左底** | 右底 < 左底 × 0.85 → 形态破坏，不识别为双重底 |
| **中间峰超前高** | 中间峰 > 前高 × (1 + middle_peak_tolerance) → 超出容忍度，不属于双重底（V形反转） |
| **两个底相距太近** | 左底和右底间隔 < 10 个交易日 → 不构成"两个底" |
| **形态跨度太大** | 从前高到右底 > 200 个交易日 → 形态跨度太大，可能是长期熊市 |

## 7. 配置项

```yaml
# config/market/double_bottom.yaml
double_bottom:
  # 前置条件
  min_prior_advance: 0.30        # 前置最小涨幅（从前低到前高）
  prior_low_lookback: 200        # 前低/前高搜索窗口（交易日）
  prior_high_min_age: 40         # 前高距今最少交易日数

  # 左底
  left_bottom_decline_min: 0.15  # 左底从前高的最小回撤（A股放宽至15%）
  left_bottom_decline_max: 0.50  # 左底从前高的最大回撤
  left_bottom_min_days_from_high: 10  # 左底距前高最少天数

  # 中间峰
  middle_peak_min_rebound: 0.10  # 中间峰从左底的最小反弹
  middle_peak_min_days: 5        # 中间峰距左底最少天数
  middle_peak_tolerance: 0.02    # 中间峰可略微超过前高的容忍度（0=严格不创新高；回测建议对比0和0.02的胜率差异）

  # 右底（关键）
  right_bottom_max_undercut: 0.03   # 右底可跌破左底的最大幅度（3%）
  right_bottom_min_price_ratio: 0.85  # 右底/左底最小比值
  right_bottom_min_days: 5       # 右底距中间峰最少天数
  right_bottom_max_age: 5        # 右底距今天最少天数（确保底部已确认）

  # 成交量
  left_vol_ratio_min: 0.8        # 左底量 / 50日均量最低
  middle_peak_vol_ratio_min: 1.0 # 中间峰量 / 50日均量最低
  right_vol_contraction: 0.7     # 右底量 / 左底量最大值（越小越好）

  # 突破
  breakout_vol_ratio: 1.4        # 突破日量 / 50日均量最低
  require_green: true            # 突破日是否要求阳线
  close_position_min: 0.50       # 突破日收盘位置最低

  # 全局
  max_pattern_span: 200          # 形态最大跨度（交易日）
  min_pattern_span: 40           # 形态最小跨度（交易日）
  bottom_distance_min: 10        # 左右底最小间隔（交易日）

  # 柄部（可选）
  handle_enabled: false          # 是否启用柄部检测
  handle_drop_max: 0.08          # 柄部最大回调
  handle_days_min: 5             # 柄部最少持续天数
  handle_days_max: 15            # 柄部最多持续天数

  # 过滤
  exclude_st: true               # 排除 ST
  min_listing_days: 180          # 最少上市天数
```

## 8. 验证清单

| 验证项 | 通过标准 |
|---|---|
| 形态结构 | 左底→中间峰→右底 时序正确，右底接近或略低于左底 |
| 成交量 | 右底成交量 < 左底成交量的 70%（萎缩确认） |
| 回撤范围 | 回撤在 15%~50% 范围内（默认阈值，可配置） |
| 突破放量 | 突破日成交量 ≥ 50 日均量的 1.4 倍 |
| 假跌破 | 右底跌破左底 ≤ 3%（轻微震仓，不是趋势破坏） |
| 信号数量 | 全市场每周 2~20 个双重底突破信号（视行情） |
| 回测胜率 | 突破后 20 日上涨概率 > 60% |
| RS 分层 | RPS ≥ 80 的信号胜率显著高于 RPS < 80 |

### 8.1 具体验证方法

1. 人工取 10 个 A 股历史上知名的双重底案例（如茅台 2018 年底），验证引擎能否识别
2. 在回测看板中标注信号日期，观察信号后 60 天的走势
3. 统计假信号（突破后快速回落）的共同特征，尝试添加过滤规则
4. 与杯柄形态对比：同一时间段内双重底和杯柄的信号数量和胜率

## 9. 技术实现建议

### 9.1 实现复杂度

双重底识别比杯柄形态简单，但比口袋支点复杂。核心挑战是**时序定位**——需要在 K 线序列中准确定位前高、左底、中间峰、右底四个关键节点。

**Polars 伪代码思路（单只股票）**：

```python
import polars as pl

# 假设 df 是一只股票的高表，含 date, open, high, low, close, volume
# 按日期升序排列

df = df.sort("date")

# 1. 找出前高（过去 N 天最高收盘价）
df = df.with_columns([
    pl.col("close").rolling_max(200).shift(40).over("stock_code").alias("prior_high_200d"),
    pl.col("close").arg_max().over(pl.int_range(0, pl.len())).alias("prior_high_idx"),
])

# 2. 前高之后找左底（最低收盘价）
# 3. 左底之后找中间峰（最高收盘价）
# 4. 中间峰之后找右底（最低收盘价）

# 这些步骤需要分段扫描，纯 Polars 表达式会比较复杂
# 建议用 Python 循环 + Polars 切片的方式实现单只股票检测
```

### 9.2 性能预估

- 单只股票（2500 个交易日 × 4 节点定位）：O(4N) ≈ 10ms
- 100 只自选股：~1 秒
- 全市场 5000 只：~50 秒（可接受，不是实时需求）
