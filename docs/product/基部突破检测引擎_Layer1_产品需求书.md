# 基部突破检测引擎（Layer 1）— 产品需求书 v1.0

> **核心原则**：只判断"今天是否是一个有效的基部突破日"，不判断"是什么形态"。形态标注由 Layer 2 完成。

---

## 1. 产品定位

本引擎是整个基部形态体系的**唯一入口**。它回答：

> 今天，某只股票是否出现了一个有效的基部突破信号？

"有效"的定义来自 O'Neil 八种经典基部形态的**最小公分母**：

1. 股票有过一段显著上涨（前置上涨 ≥ 30%）
2. 股价从高点回调了一段可观的幅度（5%~40%）
3. 回调后已经反弹，回升接近前高（≥ 75%）
4. 今天放量突破了前高

不关心：回调的形状、底部的平坦度、弧线的圆润度、是否带柄、是否逐次收缩。那些是 Layer 2 的事。

### 信号量预期

以默认宽松参数，串联通过率 ≈ 1.4%。每只股票每年约 3.5 个信号。5000 只 A 股约 1.7 万信号/年。

---

## 2. 形态定义

```
前高 ──────────────────────── 突破买点
   \                            /
    \        基部区间            /
     \    （任何形态的回调）      /
      \   5% ~ 40%              /
       \                       /
        \                     /
         谷 ─────────────────┘
     检测日前 ≥ 20 天内最低收盘价
```

### 关键节点

| 节点 | 定义 |
|---|---|
| **前高** | 谷之前、最近的一个局部高点（谷底驱动法定位） |
| **谷** | 检测日之前 ≥ 20 天内的最低收盘价 |
| **突破买点** | 前高 + 0.01 |

---

## 3. 量化计算规则

### 步骤 1：定位谷和前高（谷底驱动法）

```
1.1 找谷: 检测日 t 之前 ≥ 20 天内，找最低收盘价

1.2 从谷逐日向前回溯找前高:
    初始化 candidate = 谷
    逐日往更早方向走:
      a) 每遇到比 candidate 更高的收盘价 → 更新 candidate
      b) candidate 是前后 10 天内最高收盘价（局部高点）
         且 (candidate-谷)/candidate ≥ 5%
         → candidate = 前高，停止回溯
      c) 回溯到 lookback 边界仍未找到 → 取 candidate 为前高（兜底）

1.3 约束: 前高距谷 ≥ 10 天
```

### 步骤 2-6

```
2. 回调 ∈ [5%, 40%]
3. 回升 ≥ 前高 × 75%，且回升段斜率 > 0
4. 前置上涨 ≥ 30%
5. 突破: 收盘 ≥ 前高 + 0.01，量 ≥ 50均 × 1.3，阳线，收盘高位 ≥ 50%，收盘 > SMA50
6. 假突破排除: 盘中最低 < 前高，量 < 50均，前3日放量下跌
```

---

## 4. 参数配置（10 个）

| 参数 | 默认值 | 说明 |
|---|---|---|
| `lookback` | 120 | 回溯天数 |
| `local_window` | 10 | 局部高点窗口（天） |
| `min_base_days` | 20 | 谷距检测日最少天数 |
| `min_descent_days` | 10 | 前高到谷最少天数 |
| `drawdown_min` | 0.05 | 最小回调 |
| `drawdown_max` | 0.40 | 最大回调 |
| `min_recovery` | 0.75 | 回升到前高比例 |
| `min_prior_advance` | 0.30 | 前置涨幅 |
| `breakout_vol_ratio` | 1.3 | 突破量比 |
| `require_green` | true | 阳线开关 |
| `sma50_check` | true | SMA50开关 |

---

## 5. 信号输出格式

| 字段 | 类型 | 说明 |
|---|---|---|
| `signal_date` | date | 突破日期 |
| `stock_code` | str | 股票代码 |
| `prior_high_date` | date | 前高日期 |
| `prior_high_price` | float | 前高价格 |
| `trough_date` | date | 谷日期 |
| `trough_price` | float | 谷价格 |
| `drawdown_pct` | float | 回调 % |
| `recovery_pct` | float | 回升 % |
| `base_days` | int | 基部天数 |
| `buy_point` | float | 买点 |
| `breakout_vol_ratio` | float | 突破量比 |
| `prior_advance_pct` | float | 前置涨幅 % |

---

## 6. Layer 2 接口

Layer 1 输出的 `prior_high_date`、`trough_date`、`buy_point` 直接作为 Layer 2 标注器的输入，无需重新定位。
