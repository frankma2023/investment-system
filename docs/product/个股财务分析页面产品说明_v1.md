# 个股财务分析页面产品说明 v1.0

> 最后更新：2026-05-10
> 页面路径：`web/stock-analysis/`
> 后端：`src/analysis/financial.py`
> 功能状态：✅ 已完成，已整合至个股扫描「财务评估」Tab

## 1. 产品定位

对单只股票进行深度财务估值分析。输入股票代码，一键生成四项评估报告：**DCF 估值**、**可比公司分析**、**盈利趋势**、**三表预测**，每项均可展开完整推演详解。

## 2. 页面布局

顶部搜索框 + 四项评估卡片（2 列网格）+ 可展开详解面板。

### 2.1 输入区

- **股票代码输入框**：6 位代码，回车或点击「一键分析」
- **分析按钮**：提交后并行调用四个分析模块

### 2.2 评估卡片

| 卡片 | 核心指标 | 含详解 |
|------|---------|:--:|
| 💰 DCF 估值 | 当前价 / DCF 目标价 / 上涨空间 / 企业价值 / WACC / 年度 FCF 预测表 / 敏感性分析 | ✅ |
| 📊 可比公司分析 | 行业中位 PE/PB/PS / 隐含估值 / 可比公司对比表（PE/PB/营收/增长） | ✅ |
| 📈 盈利趋势 | 营收同比 / 净利同比 / 毛利率 / ROE / 加速/减速标记 / 近 4 季度明细表 | ✅ |
| 📋 三表预测 | 基准年参数 / 三年营收/毛利/净利/经营CF/自由CF预测表 / 缺失数据警告 | ✅ |

### 2.3 详解面板

每张卡右上角「📖 详解」按钮，在卡片下方展开完整分析推演：

| 模块 | 详解内容 |
|------|---------|
| DCF 估值推演 | 参数溯源表（每个参数的真实来源）、增长率方法论（历史均值回归递减法）、六步从营收到 FCF 推导、敏感性 WACC→目标价、使用建议 |
| 可比公司分析推演 | 可比公司筛选逻辑、估值倍数分位图、隐含估值计算方法、行业对比结论 |
| 盈利趋势推演 | 近 8 季度数据、趋势拐点判断（加速/减速）、季节性因素分析、利润率稳定性评估 |
| 三表预测推演 | 六参数溯源（SG&A/D&A/CapEx 来自真实年报扩展表）、三张表联动逻辑、从营收到 FCF 六步推导、三年全貌表 |

## 3. 技术实现

### 3.1 数据流

```
输入股票代码 → GET /api/stock-analysis?code=xxx
             → 并行调用四个函数:
               dcf_valuation() / comps_analysis() / earnings_analysis() / three_statement_projection()
             → 返回 JSON {dcf, comps, earnings, model}
             → 前端渲染卡片 + 详解面板
```

### 3.2 后端文件

```
src/analysis/financial.py    ← DCF/Comps/Earnings/3-Statement 四个函数
src/server.py                ← /api/stock-analysis 端点
```

### 3.3 数据来源

| 分析模块 | 数据来源 |
|---------|---------|
| DCF | `stock_financials_annual` + `fundamental_indicator`（EBITDA/折旧/CapEx 从扩展表提取） |
| 可比公司 | `stock_basic`（行业）+ `fundamental_indicator`（PE/PB/PS/营收增速） |
| 盈利趋势 | `stock_financials_quarterly`（近 8 季度） |
| 三表预测 | `stock_financials_annual` + `stock_financials_annual_ext`（SG&A/D&A/利息） |

### 3.4 容错设计

四个分析模块独立容错：一个模块失败不影响其他三个。返回 `{dcf: {error: "..."}, comps: {...}, ...}`。

## 4. 样式

小红书手账风格：`theme.css` + `base.css` + `xhs-cards.css` + `components.css`。

## 6. 使用场景

- 快速估值：输入代码看 DCF 目标价 vs 当前价的偏离
- 行业对比：看公司在可比公司中的估值位置
- 盈利跟踪：看最近 4 个季度的营收/净利趋势
- 财务建模：三表预测提供的参数可直接用于外部模型的输入验证
