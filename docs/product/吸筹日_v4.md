# 产品说明书：大盘吸筹日识别引擎

> **定位声明**：吸筹日本质上是**追盘日（FTD）的增强变体**，非独立新信号。它与追盘日共享同一套反弹尝试窗口机制，但附加更严苛的成交量、收盘强度和多指数共振要求。出现吸筹日意味着**不仅满足追盘日条件，还通过了更强的验证**——是市场强度更高的确认信号。

## 1. 产品定义

本产品用于**量化识别大盘指数的“吸筹日”**。在欧奈尔体系中，“吸筹日”是指市场在下跌后出现**机构资金积极买入**的交易日，是判断底部形成、上升趋势启动的重要辅助信号。它与“追盘日”逻辑相似但更强调**成交量放大与收盘强势**的组合，可作为右侧交易的早期预警。

## 2. 输入数据

| 数据项 | 来源 | 字段 | 频率 |
| :--- | :--- | :--- | :--- |
| 指数日线行情 | index_daily_kline | 开盘价、最高价、最低价、收盘价、成交量（或成交额） | 每日 |
| 指数自由流通市值 | fundamental_indicator | 自由流通市值 | 每日（用于换手率替代成交量） |

**推荐指数池**（至少包含以下3个）：
- 中证全指（000985）—— 必选基准
- 沪深300（000300）—— 大盘股代表
- 科创50（000688）—— 风险偏好风向标

## 3. 量化定义：什么是“大盘吸筹日”？

一个交易日被标记为“吸筹日”，需**同时满足**以下**4个核心条件**：

### 3.1 背景条件：处于下跌后的“反弹尝试”中

**定义反弹尝试起点（Day 1）**：
- 当日最低价创 **20 日新低**（`low = lowest(low, 20)`）
- 且（收盘价 > 前一日收盘价 **或** 收盘价 > (最高价+最低价)/2）

之后第 2～7 天为“反弹窗口”。吸筹日只能出现在**从 Day 1 算起的第 4～7 天**。

> 若在窗口内出现更低的低点+收盘强势，则以新的 Day 1 重置窗口。

### 3.2 价格条件：当日收盘涨幅

- **上涨 ≥ 1.0%**（欧奈尔标准，可配置为 1.2% 或 1.5% 提高可靠性）

### 3.3 成交量条件：显著放大

**满足以下任意一种**：
- 成交量 > 前一日成交量 × **1.2**（相比前日显著放量）
- 成交量 > 过去 20 日均量的 **1.1** 倍（相对近期活跃）

> 建议使用**成交金额**代替成交量，避免除权影响；或者使用**换手率** = 成交金额 / 自由流通市值。

### 3.4 收盘强度（可选增强）

- 收盘价处于当日价格区间的 **偏上方位置**，即：
  ```
  (close - low) / (high - low) > 0.6
  ```
- **例外规则**：若当日振幅 < 1.5%（即 `(high - low) / close < 0.015`），自动视为通过收盘强度检查。此规则用于防止跳空高开后窄幅横盘时公式失效——全天窄幅横盘于高位本身就是强势表现。
- 若 `high == low`（开盘即全天唯一价格），直接通过。

## 4. 辅助验证条件（提高信号可靠性）

| 条件 | 量化规则 | 加分 |
| :--- | :--- | :--- |
| 多指数共振 | 上述条件在 **≥2 个指数**（如中证全指+沪深300）同日满足 | 信号强度 +1 级 |
| A/D 线方向 | 指数的 Chaikin A/D 线（20日累积）斜率 > 0 | 确认资金流入 |
| 无抛盘日干扰 | 过去 5 个交易日内未出现抛盘日 | 避免假信号 |

## 5. 输出结果

### 5.1 单日标记
每日对每个指数输出布尔值：`is_accumulation_day`（True/False）。

### 5.2 综合信号（多指数）
当**中证全指**出现吸筹日，且至少一个辅助指数（沪深300/科创50）同时出现吸筹日，则输出 **“强吸筹确认”**。

### 5.3 信号等级

所有吸筹日信号均**以追盘日已成立为前提**。等级表示附加验证的深度：

| 等级 | 条件 | 操作参考 |
| :--- | :--- | :--- |
| ✅ 标准吸筹 | 单一指数满足 3.1～3.4 全部核心条件 | 追盘日基础上加大仓位 |
| ✅✅ 强力吸筹 | ≥2个指数同日吸筹 + 无抛盘日干扰 | 可进一步加大仓位，市场底部确认概率更高 |

## 6. 与追盘日的关系

吸筹日是追盘日的增强版，二者的关系如下：

| 对比项 | 追盘日 (FTD) | 吸筹日 ★ (本产品) |
| :--- | :--- | :--- |
| 关系 | 基础信号 | FTD + 更强验证 = 吸筹日 |
| 核心目的 | 确认上升趋势开始 | 同上，但置信度更高 |
| 时间窗口 | 反弹尝试第4～7天 | 相同 |
| 涨幅要求 | ≥ 1.0% | 相同 |
| 成交量要求 | > 前一日 | **更严格**：> 前一日 1.2 倍 **且** > 20 日均量 1.1 倍（FTD只需其一） |
| 收盘位置 | 无硬性要求 | 收盘价须处于当日区间偏上位置（振幅 <1.5% 时豁免） |
| 多指数共振 | 不要求 | **要求** ≥ 2 个指数同日满足 |
| 信号含义 | "反弹可能成功了" | "反弹不仅成功，而且很强" |

**一句话总结**：先有 FTD，再有吸筹日。吸筹日不替代 FTD——它是在 FTD 的基础上附加更严苛的过滤器，作为机构大资金强势介入的确认信号。

## 7. 开发建议（AI实现要点）

### 7.1 数据预处理
- 确保指数日线按日期排序，计算前复权收盘价。
- 计算 20 日最低价、20 日均量（滚动窗口）。

### 7.2 反弹尝试窗口管理
```python
def is_rally_attempt(df, idx):
    # 过去20日最低价（不含当日）
    prev_20_min_low = df['low'].iloc[idx-20:idx].min()
    is_new_low = df['low'].iloc[idx] < prev_20_min_low
    if not is_new_low:
        return False
    close_strong = (df['close'].iloc[idx] > df['close'].iloc[idx-1]) or \
                   (df['close'].iloc[idx] > (df['high'].iloc[idx] + df['low'].iloc[idx])/2)
    return close_strong
```

### 7.3 吸筹日判定函数
```python
def is_accumulation_day(df, idx, rally_days, gain_threshold=1.0, vol_ratio_thresh=1.2, vol_ma20_thresh=1.1):
    if not (4 <= rally_days <= 7):
        return False
    if df['close_pct_change'].iloc[idx] < gain_threshold:
        return False
    vol_cond = (df['volume'].iloc[idx] / df['volume'].iloc[idx-1] > vol_ratio_thresh) \
               or (df['volume'].iloc[idx] / df['volume'].rolling(20).mean().iloc[idx] > vol_ma20_thresh)
    if not vol_cond:
        return False
    # 可选：收盘强度
    close_pos = (df['close'].iloc[idx] - df['low'].iloc[idx]) / (df['high'].iloc[idx] - df['low'].iloc[idx])
    if close_pos < 0.7:
        return False
    return True
```

### 7.4 输出存储
- 数据库表：`index_accumulation_daily`（字段：日期、指数代码、is_accumulation、signal_level、close_price、volume、close_pct_change）
- CSV 备份：每日追加
