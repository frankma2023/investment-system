# 个股RS强度计算引擎 产品说明文档

## 1. 产品定位

本引擎用于每日计算全市场A股的**相对强度（Relative Strength, RS）**，是欧奈尔 CAN SLIM 体系中 **“L – 领军股”** 维度的核心量化工具。输出结果可直接用于双强选股、口袋支点、卖出预警等策略。

## 2. 核心理念（欧奈尔原教旨）

- **RS 评级（RPS）**：将个股过去 N 日的涨幅在全市场内排序，转化为 1～99 的百分位分数。**RPS ≥ 80 才值得关注，90 以上为超级强势股**。
- **双周期**：长期（250日）确认趋势稳定性，短期（20日）捕捉爆发动能。两者共振（双强）是最佳买点。
- **RS 线**：个股收盘价 / 大盘收盘价，观察相对强度趋势，背离是重要卖出信号。
- **全市场基准**：采用中证全指（000985.CSI），避免风格偏好。

## 3. 输入数据

| 数据类型 | 字段 | 来源 | 更新频率 |
| :--- | :--- | :--- | :--- |
| 个股日线 | 前复权收盘价、成交量 | 理杏仁 / Tushare | 每日收盘后 |
| 指数日线 | 中证全指（000985）前复权收盘价 | 同上 | 每日收盘后 |
| 基础信息 | 上市日期、停牌标志 | 基础信息API | 每日 |

## 4. 量化计算规则

### 4.1 涨跌幅计算

对每只股票及基准指数，分别计算两个时间窗口的区间涨跌幅：

\[
\text{Ret}_{N} = \left( \frac{P_t}{P_{t-N}} - 1 \right) \times 100\% \quad (N=20,\ 250)
\]

- 使用**前复权收盘价**，消除分红、送股、配股的断层。
- 若 \(t-N\) 日无数据（新股上市不足、停牌），则顺延至最近有效交易日。
- 若有效交易日不足 \(N/2\)，该周期不参与排名（标记为 NaN）。

### 4.2 RS 评级（RPS）计算

在每个交易日，分别对 **250 日涨跌幅** 和 **20 日涨跌幅** 在全市场有效股票内排序，计算百分位排名：

\[
\text{RPS} = \left(1 - \frac{\text{排名} - 1}{\text{有效样本数} - 1}\right) \times 99
\]

- 结果四舍五入取整，范围 **0～99**。
- $\text{RPS}=90$ 表示该股票表现优于市场中 90% 的股票。
- 新股、停牌股票不参与排名，也不出现在结果中。

### 4.3 RS 线（可选，用于趋势监控）

\[
\text{RS\_Line} = \frac{\text{个股前复权收盘价}}{\text{中证全指收盘价}}
\]

- 不参与选股，仅用于观察个股相对于大盘的强度趋势。
- **使用场景**：股价新高但 RS 线未新高 → 顶背离，警惕转弱。

## 5. 双强选股标准（核心输出）

每日筛选满足以下**任一模式**的股票：

| 模式 | RPS_250 | RPS_20 | 含义 | 操作参考 |
| :--- | :--- | :--- | :--- | :--- |
| **稳健龙头** | ≥ 90 | ≥ 85 | 长期优异+近期持续强势 | 均线附近低吸 |
| **加速爆发** | ≥ 80 | ≥ 95 | 长期良好+短期急剧爆发 | 突破放量时追入 |

输出字段：日期、股票代码、名称、RPS_250、RPS_20、收盘价、成交额、所属行业（可选）。

## 6. 数据处理规则

| 场景 | 处理方式 |
| :--- | :--- |
| **新股** | 上市不满 **250 交易日**不计算 RPS_250；不满 20 日不计算 RPS_20 |
| **停牌** | 停牌期间不更新，复牌后按实际交易日回溯计算 |
| **退市/风险警示** | 剔除 ST、*ST 股票（可配置） |
| **流动性过滤（建议）** | 可选剔除近 20 日均成交额 < 5000 万元的股票 |

## 7. 欧奈尔理念验证清单（开发后必做）

| 验证项 | 通过标准 |
| :--- | :--- |
| RPS 分布 | 每日全市场中位数约为 50，最高 99，最低 1 |
| 双强股票数量 | 牛市通常 30～100 只，熊市可能仅 0～10 只 |
| 滞后性 | 回测确认 RPS 向上突破后的股票未来 1~3 个月超额收益显著 |
