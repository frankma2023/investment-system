# 口袋支点(Pocket Pivot)买入信号识别引擎 — 产品需求书

## 1. 产品定位

**口袋支点（Pocket Pivot）** 是由 Gil Morales 和 Chris Kacher（两位前 William O'Neil 公司基金经理，欧奈尔门徒）提出的一种**早期买入信号**。它与标准突破买点（杯柄上沿突破）互补：突破买点是"确认入场"，口袋支点是"抢先入场"。

**在整体系统中的位置**：个股扫描模块的第一个功能。上游依赖指数扫描模块（已完成：指数 RS、拥挤度、机构吸筹出货、指数背离）；下游消费者包括：`stock_candidates_daily` 候选管线、`index_rs.yaml` resonance 指数共振判断。

**与基部识别引擎的关系**：基部识别识别"形态"，口袋支点识别"量价信号时刻"。两者互补——基部告诉你"哪里有值得关注的结构"，口袋支点告诉你"这一刻值得扣扳机"。许多口袋支点发生在基部内部，作为基部突破的前置信号。

---

## 2. 核心理念

口袋支点的核心思想是：**在一个上升趋势中，某一天出现了异常的大成交量买入行为，且收盘价收在日内高位——说明机构资金正在悄悄吸筹，但价格还没有大幅拉升。**

类比：火箭尚未发射，但引擎已经点火，尾部冒出浓烟——你在发射前上车。

传统突破（杯柄上沿）的问题是：等你确认突破时，价格可能已经拉高了 5-8%。口袋支点让你在突破前约 5-15 个交易日内入场。

### 2.1 三种口袋支点类型

| 类型 | 位置 | 信号含义 |
|---|---|---|
| **基部口袋支点** | 基部内部（杯底右侧、柄部区域） | 机构在基部底部完成吸筹，启动拉升 |
| **延续口袋支点** | 上升趋势中短暂回调后 | 趋势健康，回调是加仓机会 |
| **10日线反弹口袋支点** | 回踩 10 日均线反弹 | 强势股正常回调不破 10 日线 |

### 2.2 与标准突破的区别

| 维度 | 标准突破 | 口袋支点 |
|---|---|---|
| 入场时机 | 突破基部上沿后 | 突破上沿前 5-15 天 |
| 价格位置 | 历史最高价附近 | 距高点尚有距离 |
| 成交量要求 | 突破日成交量高于均量 | 突破日成交量高于过去 10 天内**所有下跌日**的最大成交量 |
| 风险 | "确定性"入场，但晚了 | 更早入场，但底部可能尚未确认 |
| 假信号率 | 较低 | 较高（需结合其它信号过滤） |

## 3. 量化计算规则

### 3.1 基础数据需求

| 数据 | 来源 | 说明 |
|---|---|---|
| 日 K 线（OHLCV） | `daily_kline` 表 | 前复权价格 + 成交量 |
| RS 强度 | RS 引擎（实时计算） | RPS_20、RPS_250 |
| 均线 | 盘中计算 | 10 日、50 日简单移动平均 |

### 3.2 口袋支点判断规则（核心算法）

对每个交易日的每一只股票，执行以下五步判断：

#### 步骤 1：趋势基础 — 股票必须处于上升趋势

```
条件 1.1: 收盘价 > 50日简单移动均线（SMA50）
条件 1.2: 50日 SMA 斜率 > 0（过去 10 天内至少抬高 0.5%）
条件 1.3: 收盘价 > 10日 SMA
```

任一不满足 → 不产生信号。

**设计理由**：口袋支点是上升趋势中的买入信号，不适用于下跌趋势或盘整中的股票。条件 1.2 的斜率要求排除了"SMA50 还在下降但价格短暂冲上均线"的假突破。

#### 步骤 2：日内行为 — 今天出现了"机构买入"

```
条件 2.1: 收盘价 ≥ 日内振幅的 50%（上影线不过长，收盘在日内中位以上）
条件 2.2: （可选严格版）收盘价 ≥ 日内振幅的 62%（收盘在上 1/3 区域）
条件 2.3: 收盘价 > 开盘价（阳线）[可配置关闭]
```

**设计理由**：机构资金不会在开盘拉高然后收盘回落——他们会在全天持续买入，收盘在日内高位。大成交量 + 高收盘位置 = 真正的买方主导。

#### 步骤 3：成交量特征 — "口袋"里的量爆发（核心）

这是口袋支点名字的由来，也是与普通放量阳线最本质的区别：

```
条件 3.1: 今日成交量 > 过去 10 个交易日内所有下跌日（close < open）的最大成交量
```

简记为：`vol_today > max(vol_day_i) for all day_i in [-10, -1] where close_i < open_i`

**设计理由**：过去 10 天里，有若干天是阴线（下跌日）。那些阴线代表了抛压。不管抛压有多大，今天的买入力量都能超越过去 10 天所有抛售日的量——这就是"口袋里装满弹药，一举打穿空头防线"的信号。

**宽松版（可选）**：
```
条件 3.2: 今日成交量 > 过去 50 日均量的 1.5 倍（适用于低波动期）
```

如果过去 10 天内没有阴线（极端上涨行情），退化为条件 3.2。

#### 步骤 4：不在"超买延伸区" — 不要追高

```
条件 4.1: 收盘价 ≤ 过去 65 日最低收盘价 × 1.50（距离低点不超过 50%）
条件 4.2: （可选）收盘价 ≤ 最近一个基部的枢轴点 × 1.05
```

**设计理由**：即使量价行为完美，如果股票已经从最低点涨了 80%，进入口袋支点也可能买在顶部。65 日低点 1.5x 是一个经验阈值——涨超 50% 的股票视为"延伸"，不产生信号。

如果不使用基部识别（无枢轴点数据），仅使用条件 4.1。

#### 步骤 5：RS 强度确认

```
条件 5.1: RPS_20 ≥ 80 或 RPS_250 ≥ 80
条件 5.2: （推荐）RPS_20 ≥ 80 且 RPS_250 ≥ 80
```

**设计理由**：不是所有放量阳线都值得买入。只有市场中的强势股才值得。RPS ≥ 80 确保股票在过去一段时间内表现优于市场中 80% 的股票。

---

### 3.3 特殊类型判断

#### 3.3.1 10 日线反弹口袋支点

在步骤 1~5 的基础上，增加：
```
额外条件: 今日最低价 ≤ 10日 SMA × 1.02（回踩到 10 日线附近）
        且 今日收盘价 > 昨日收盘价（反弹确认）
```

这是三种类型中最早、风险最低的入场点——在股价回踩均线获得支撑时入场。

#### 3.3.2 基部口袋支点

如果与基部识别引擎联动，进一步过滤：
```
额外条件: 今日日期在某个已识别的杯柄基部的"谷底后、突破前"区间内
        且 口袋支点出现后，未来 5-15 天内有明确突破信号
```

## 4. 信号输出格式

### 4.1 输出字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `date` | date | 信号日期 |
| `stock_code` | str | 股票代码 |
| `stock_name` | str | 股票名称（可选） |
| `close` | float | 当日收盘价 |
| `volume` | int | 当日成交量 |
| `vol_ratio_to_down_max` | float | 成交量 / 过去10日最大阴线量 |
| `close_position` | float | 收盘在日内振幅的百分位（0-1） |
| `rps_20` | int | RS 强度 20 日 |
| `rps_250` | int | RS 强度 250 日 |
| `sma50` | float | 50 日简单移动均线 |
| `sma10` | float | 10 日简单移动均线 |
| `pivot_type` | str | 类型：`base`（基部）、`continuation`（延续）、`10ma_bounce`（10日线反弹） |
| `days_since_65d_low` | int | 距 65 日低点天数 |
| `pct_from_65d_low` | float | 距 65 日低点涨幅 % |
| `base_id` | str/null | 所在基部 ID（若与基部引擎联动） |

### 4.2 输出接口

- **API 端点**: `GET /api/stock-pocket-pivot?stock=600519&date=2026-05-11`
- **批量扫描**: `GET /api/stock-pocket-pivot/scan?date=2026-05-11&pool=watchlist`
- **回测看板**: `/stock-pocket-pivot-backtest/` 页面，与现有 RS 回测看板风格一致

## 5. 数据处理规则

| 场景 | 处理方式 |
|---|---|
| **数据不足** | 上市不满 120 交易日不计算（无法获得有效均线和 RS 值） |
| **停牌** | 复牌首日不产生信号（成交量失真） |
| **ST/\*ST** | 默认剔除，沿用 `exclude_st` 配置 |
| **涨停/跌停** | 涨停日（收盘 = 最高限价）且成交量极小（一字板），排除 |
| **分红除权** | 使用前复权价格，无需特殊处理 |
| **无阴线的过去10日** | 退化为 50 日均量 × 1.5 判断 |
| **RPS 数据缺失** | 警告并跳过该股票（RS 是口袋支点的硬条件） |

## 6. 配置项

```yaml
# config/market/pocket_pivot.yaml
pocket_pivot:
  # 趋势基础
  sma10_check: true           # 是否要求价格 > 10日SMA
  sma50_check: true           # 是否要求价格 > 50日SMA
  sma50_slope_min: 0.005      # 50日SMA 10日最低斜率

  # 日内行为
  close_position_min: 0.50    # 收盘在日内振幅的最低百分位
  close_position_strict: 0.62 # 严格版阈值（可选）
  require_green: true         # 是否要求阳线

  # 成交量
  vol_down_lookback: 10       # 对比过去几天的下跌日最大量
  vol_ma_fallback: 50         # 无阴线时退化为 N 日均量
  vol_ma_fallback_ratio: 1.5  # 均量倍数阈值

  # 延伸区限制
  extension_lookback: 65      # 延伸区回看天数
  extension_max_pct: 0.50     # 距最低点最大涨幅

  # RS 强度
  rs_threshold: 80            # RPS 最低阈值
  rs_dual_require: false      # 是否要求 RPS_20 和 RPS_250 同时 ≥ 阈值

  # 10日线反弹（特殊类型）
  ma10_bounce_enabled: true   # 是否启用 10 日线反弹类型
  ma10_bounce_proximity: 0.02 # 距 10 日线距离阈值（2%）

  # 过滤
  exclude_st: true            # 排除 ST
  exclude_limit_up: true      # 排除一字涨停
  min_listing_days: 120       # 最少上市天数
```

## 7. 与现有系统的衔接

### 7.1 基部识别引擎（开发中）

口袋支点可独立运行（仅使用量价和 RS 数据），因为基部识别引擎未就绪 → 口袋支点对所有满足趋势+量价条件的日期独立判断

### 7.2 RS 引擎

口袋支点依赖 RPS 值（与标准突破引擎共享）。RS 引擎需要先完成当日全市场 RPS 计算。

### 7.3 回测看板

与现有 RS 回测看板风格一致。信号列表 + K 线图标注口袋支点日期。关键验证指标：
- 信号后 20 日最大涨幅分布
- 信号后 60 日胜率（按 RPS 分组）
- 不同类型（基部 vs 延续 vs 10日线反弹）的胜率对比

## 8. 技术实现建议

### 8.1 实现复杂度

口袋支点的量化规则比杯柄形态识别简单得多——大多数条件是单日统计量，不需要窗口内的时序模式匹配。Polars 一行表达式可实现核心判断。

**Polars 伪代码（高表形态）**：

```python
import polars as pl

# 假设 df 是高表，含 stock_code, date, open, high, low, close, volume, rps_20, rps_250

df = df.sort(["stock_code", "date"])

# 步骤1: 均线
df = df.with_columns([
    pl.col("close").rolling_mean(10).over("stock_code").alias("sma10"),
    pl.col("close").rolling_mean(50).over("stock_code").alias("sma50"),
])

# 步骤2: 日内位置
df = df.with_columns([
    ((pl.col("close") - pl.col("low")) / (pl.col("high") - pl.col("low")))
    .alias("close_position"),
])

# 步骤3: 过去10日最大阴线量（核心）
df = df.with_columns([
    pl.when(pl.col("close") < pl.col("open"))
      .then(pl.col("volume"))
      .otherwise(0)
      .rolling_max(10, min_periods=1)
      .shift(1)  # 不含今天
      .over("stock_code")
      .alias("max_down_vol_10d"),
])

# 步骤4: 65日最低收盘
df = df.with_columns([
    pl.col("close").rolling_min(65).over("stock_code").shift(1).alias("lowest_close_65d"),
])

# 口袋支点判断
df = df.with_columns([
    (
        (pl.col("close") > pl.col("sma50")) &
        (pl.col("close") > pl.col("sma10")) &
        (pl.col("close_position") >= 0.50) &
        (pl.col("close") > pl.col("open")) &
        (pl.col("volume") > pl.col("max_down_vol_10d")) &
        (pl.col("close") <= pl.col("lowest_close_65d") * 1.50) &
        ((pl.col("rps_20") >= 80) | (pl.col("rps_250") >= 80))
    ).alias("is_pocket_pivot")
])
```

### 8.2 性能预估

基于高表（5000 只股票 × 2600 个交易日 = 1300 万行）：
- 均线计算：O(N)，毫秒级
- 窗口函数（rolling_max, rolling_min）：Polars lazy API 并行执行，预计单日全市场 < 2 秒
- 总计算量远小于 RS 引擎（无需每日全市场排序）

## 9. 验证清单

| 验证项 | 通过标准 |
|---|---|
| 信号数量 | 全市场每日 5-50 个口袋支点信号（视行情波动） |
| 趋势过滤 | 所有信号均满足 close > SMA50 |
| 成交量规则 | 所有信号的 vol > 过去10日最大阴线量 |
| RS 确认 | 所有信号 RPS ≥ 80 |
| 延伸区 | 所有信号距 65 日低点 ≤ 50% |
| 回测胜率 | 信号后 20 日上涨概率 > 55%（优于随机） |
| 与标准突破对比 | 口袋支点信号平均比标准突破早 5-15 个交易日 |
| RPS 分层 | RPS ≥ 90 的信号胜率显著高于 RPS 80-85 |

### 9.1 具体验证方法

人工取 20 个历史口袋支点信号，在回测看板中标记，观察：
1. 信号出现后 5-20 天是否发生了标准突破？
2. 信号出现后 60 天最大涨幅？
3. 假信号（买入后持续下跌）的共同特征是什么？（尝试加过滤规则）

## 10. 优先级与迭代计划

| 阶段 | 内容 | 预计工作量 |
|---|---|---|
| **v1** | 核心五步判断（趋势+日内+量+延伸+RS），延续和 10 日线反弹两种类型 | 1-2 天 |
| **v2** | 与基部识别引擎联动（基部口袋支点），回测看板集成 | 1 周 |
| **v3** | 基于回测数据优化阈值（SHAP/XGBoost 调参），加入大盘环境过滤 | 持续迭代 |

## 附录：口袋支点 vs 标准突破 — 使用场景

| 投资者类型 | 推荐信号 | 理由 |
|---|---|---|
| 激进型 | 口袋支点优先 | 更早入场，更多利润空间 |
| 保守型 | 标准突破为主，口袋支点为辅 | 等待突破确认后入场 |
| 仓位管理 | 口袋支点做初始仓位（1/3），突破加仓（2/3） | 分散入场时机 |
| 卖出 | 口袋支点入场者可用 10 日线做止损 | 10 日线跌破即走 |
