# 缠论区间套买卖点引擎 · 产品需求书

> 版本：v1.0 | 日期：2026-05-29 | 状态：待开发 | 优先级：P2  
> 前置依赖：`src/scanners/chanlun.py` 缠论引擎 v2.0（含线段计算）  
> 数据依赖：分钟级 K 线数据（60分钟 + 15分钟）

---

## 1. 产品概述

### 1.1 定位

在现有缠论引擎完成分型→笔→线段→中枢→背驰→买卖信号六级分析的基础上，实现**多周期区间套**定位——从大级别确定方向，到中级别确认结构，到小级别锁定精确入场点。

### 1.2 核心价值

- **入场精度跃升**：日线一买信号的入场区间从「3~5 天」缩小到「15 分钟级别的一两根 K 线」
- **假信号过滤**：日线出现背驰但 60 分钟/15 分钟未确认 → 降低置信度或过滤
- **缠论实战标准配置**：区间套是缠论交易者的核心武器，没有它就没有精确买卖点

### 1.3 三级定位逻辑

```
日线（大级别）         → 确定方向（一买/一卖信号）
    ↓ 确认
60分钟线（中级别）     → 验证结构（同向背驰确认）
    ↓ 确认  
15分钟线（小级别）     → 锁定入场（精确买卖点）
    ↓
输出：区间套信号 + 三级综合置信度
```

### 1.4 与现有功能的关系

| 现有功能 | 关系 |
|---------|------|
| `analyze()` 日线缠论 | 提供大级别方向（一买/一卖信号） |
| `compute_segments()` 线段 | 提供日线线段中枢（更稳定的参考） |
| `get_echarts_option()` 图表 | 新增分钟级图表渲染 |
| 多周期联立（日/周/月） | 区间的三个级别在同一时间维度（小时级），互补而非替代 |

## 2. 数据源

### 2.1 选择：Baostock

| 维度 | 评估 |
|------|------|
| 数据覆盖 | 15分钟 + 60分钟，历史从 1999-07-26 至今 |
| 复权支持 | 前复权（adjustflag="2"），保证价格连续性 |
| 费用 | 完全免费，无需注册/Token/申请 |
| 接入方式 | `pip install baostock`，三行代码获取全量 |
| 限制 | 指数无分钟线数据（区间套仅对个股，不影响） |
| 单次请求 | 返回指定股票全量数据，无需分页 |

### 2.2 分钟 K 线数据

| 周期 | 用途 | 典型数据量（3个月） |
|------|------|-------------------|
| 60 分钟 | 中级确认 | ~250 根 |
| 15 分钟 | 精确定位 | ~1000 根 |

### 2.3 数据拉取策略

- **按需拉取**：用户触发区间套分析时才拉取，不做全市场预加载
- **缓存优先**：拉取后存入 SQLite（`stock_kline_15min` / `stock_kline_60min` 表），同日重复分析直接查库
- **增量机制**：已有缓存时，只拉取 `MAX(date) ~ 今天` 的增量数据
- **会话管理**：baostock 需 `login()`/`logout()`，封装为上下文管理器（`with _session():`），防止连接泄漏

### 2.4 数据拉取模块

在 `data/lixr_api/` 下新建 `api_stock_minute.py`，参照现有 `api_stock_candlestick.py` 的模式：

```
api_stock_minute.py
├── _to_bs_code(code)        # '000338' → 'sz.000338'（深市/沪市自动判断）
├── _session()               # baostock 登录上下文管理器
├── fetch_minute_kline(code, frequency, start_date, end_date)
│                            # frequency: '15' | '60'
│                            # adjustflag: "2"（前复权）
└── sync_to_db(code, frequency)
                             # 拉取 → 写入 stock_kline_15min / stock_kline_60min
```

**代码骨架**：

```python
import baostock as bs
import sqlite3
from contextlib import contextmanager

@contextmanager
def _session():
    bs.login()
    try:
        yield
    finally:
        bs.logout()

def _to_bs_code(code):
    """000338 → sz.000338, 600519 → sh.600519"""
    if code.startswith(('0', '3', '1')):
        return f'sz.{code}'
    elif code.startswith(('6', '9')):
        return f'sh.{code}'
    return f'sz.{code}'

def fetch_minute_kline(code, frequency, start_date, end_date):
    with _session():
        rs = bs.query_history_k_data_plus(
            _to_bs_code(code),
            "date,time,code,open,high,low,close,volume,amount,adjustflag",
            start_date=start_date, end_date=end_date,
            frequency=frequency, adjustflag="2"
        )
        return rs.get_data()
```

### 2.5 数据库表

```sql
CREATE TABLE IF NOT EXISTS stock_kline_60min (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    stock_code TEXT NOT NULL,
    date TEXT NOT NULL,        -- 交易日期 YYYY-MM-DD
    time TEXT NOT NULL,        -- 时分 HH:MM:SS
    open REAL, high REAL, low REAL, close REAL,
    volume REAL, amount REAL,
    created_at TEXT DEFAULT (datetime('now','localtime')),
    UNIQUE(stock_code, date, time)
);

CREATE TABLE IF NOT EXISTS stock_kline_15min (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    stock_code TEXT NOT NULL,
    date TEXT NOT NULL,
    time TEXT NOT NULL,
    open REAL, high REAL, low REAL, close REAL,
    volume REAL, amount REAL,
    created_at TEXT DEFAULT (datetime('now','localtime')),
    UNIQUE(stock_code, date, time)
);
```

### 2.6 时段处理

- baostock 返回的 15 分钟和 60 分钟 K 线包含全时段（含午休 11:30-13:00 之前的数据），时间连续
- CZSC 处理分钟 K 线时不做特殊时段过滤，直接传入 RawBar 列表
- 注意：baostock 的 `time` 字段格式为 `HHMMSS`（如 `093000`、`150000`），需转换为 `HH:MM:SS` 再用于 pandas 时间索引

## 3. 核心算法

### 3.1 区间套匹配流程

```
输入: stock_code, 日线背驰信号日期 dt_daily
输出: 区间套信号列表

Step 1: 拉取分钟数据
  60min_bars = fetch_klines(stock_code, '60min', dt_daily - 30天, dt_daily + 5天)
  15min_bars = fetch_klines(stock_code, '15min', dt_daily - 10天, dt_daily + 3天)

Step 2: 中级确认（60分钟）
  czsc_60 = CZSC(60min_bars, Freq.F60)
  bi_60 = czsc_60.bi_list
  zs_60 = compute_zhongshu(bi_60)
  div_60 = detect_divergence(bi_60, zs_60)
  
  匹配条件:
    - 60分钟的背驰方向与日线一致（同为底背驰或同为顶背驰）
    - 60分钟背驰日期在日线背驰日期 ± 3 天内

Step 3: 小级别定位（15分钟）
  czsc_15 = CZSC(15min_bars, Freq.F15)
  bi_15 = czsc_15.bi_list
  zs_15 = compute_zhongshu(bi_15)
  div_15 = detect_divergence(bi_15, zs_15)
  
  定位条件:
    - 15分钟的背驰方向与60分钟一致
    - 15分钟背驰日期在60分钟背驰日期 ± 1 天内
    - 取 15 分钟背驰点 ± 2 根 K 线作为精确入场区间

Step 4: 置信度评分
  三级全过 → 高（入场区间精确到 15 分钟级）
  两级通过 → 中（60 分钟确认但 15 分钟未定位）
  仅日线   → 低（退化为现有日线买卖信号）
```

## 4. API 端点

### 4.1 `/api/chanlun/cascade`

| 属性 | 值 |
|------|-----|
| 方法 | GET |
| 参数 | `code`（股票代码），`date`（日线背驰信号日期，可选），`side`（buy/sell，可选） |
| 返回 | `{daily_signal, cascade_signals, 60min_chart, 15min_chart}` |

**逻辑**：
- 若未指定 `date`，自动取日线最新的买卖信号进行区间套分析
- 若指定了 `date`，对该日期的信号进行分析
- 同时返回 60 分钟和 15 分钟的 ECharts option，供前端三图并排展示

### 4.2 `/api/chanlun/cascade/echarts`

| 属性 | 值 |
|------|-----|
| 方法 | GET |
| 参数 | `code`, `freq`（60min/15min），`limit`，`theme` |
| 返回 | ECharts option（分钟级 K 线 + 笔标记 + 中枢 + 背驰标记） |

## 8. 已知限制与风险

1. **baostock 依赖**：baostock 是第三方免费库，无商业 SLA，若服务中断需切换备用数据源
2. **CZSC 分钟级兼容性**：CZSC 设计时以日线为主，分钟级 K 线的笔划分行为可能与日线不同，需实际验证后调参
3. **计算性能**：三级别并行分析 + CZSC × 3（日/60分/15分），单次分析约 3~5 秒，可接受
4. **时段处理**：baostock 的 `time` 字段为 `HHMMSS` 格式（如 `093000`），需转换后用于 pandas 时间索引
5. **仅支持个股**：baostock 不提供指数分钟线，区间套功能仅限个股
6. **先做买入方向**：卖出方向（区间套一卖）逻辑对称，后续补充
