## 1. 产品概述

### 1.1 定位

在现有缠论引擎（分型→笔→中枢→背驰→买卖信号）的基础上，增加**线段**计算层，实现笔到线段的级别跃迁。线段是比笔更高级别的走势单元，是构建高级别中枢和实现真正多级别联立的基础。

### 1.2 核心价值

- **级别跃迁**：从「笔级别」上升到「线段级别」，每一级对应不同的交易周期
- **高级别中枢**：三段线段重叠构成线段中枢，比笔中枢更稳定、信号更可靠
- **为区间套铺路**：有了线段，才能在同一周期内做多级别分析（线段构成高级别笔，高级别笔构成更高级别线段）

### 1.3 当前状态

| 笔 | ✅ CZSC 计算 |

## 2. 缠论线段定义

### 2.1 标准定义

在缠论中，线段是连续笔构成的更高级别走势，由至少三笔组成：

```
线段 = 连续 N 笔（N ≥ 3），满足：
  1. 前三笔必须有重叠区间
  2. 线段方向由第一笔的方向决定
  3. 线段终点是最后一笔的终点
  4. 线段被反向线段破坏时才终结
```

段落的起始和终结：
- **向上线段**：从向上笔开始，到向下笔结束。包含奇数笔（3/5/7...），最后一笔是向下笔
- **向下线段**：从向下笔开始，到向上笔结束。包含奇数笔，最后一笔是向上笔

### 2.2 简化版（MVP 采用）

为避免过度复杂化，MVP 采用简化规则：

```
遍历笔列表，按「至少 3 笔 + 有重叠」原则构建线段：
  1. 找到第一个满足「连续 3 笔有重叠区间」的起点
  2. 从该起点开始逐笔延伸
  3. 当延伸笔的方向与线段方向一致时继续延伸
  4. 当出现反向笔且反向笔之后又有同向笔时：
     - 如果新的同向笔破坏了之前的线段结构，则前一线段终结
     - 终结条件：反向笔的低点/高点突破了前一线段的极值
  5. 线段终结后，下一段线段从终结笔开始

输出：线段列表 [{start_dt, end_dt, direction, high, low, bi_count, bi_indices}]
```

### 2.3 线段构成示例

```
笔序列: ↑1 → ↓2 → ↑3 → ↓4 → ↑5 → ↓6 → ↑7 → ↓8

线段1 (向上): ↑1 + ↓2 + ↑3
  → 三笔有重叠 → 构成向上线段
  → 包含 bi[0], bi[1], bi[2]
  → 起点=↑1.sdt, 终点=↑3.edt

线段2 (向下): ↓4 + ↑5 + ↓6 + ↑7 + ↓8
  → ↓4 从线段1的终点开始
  → 重叠区间检查通过
  → 包含 bi[3], bi[4], bi[5], bi[6], bi[7]
  → 起点=↓4.sdt, 终点=↓8.edt
```

## 4. 线段中枢算法

### 4.1 与笔中枢的关系

```
笔中枢：连续 3 笔重叠 → 低级别中枢
线段中枢：连续 3 段线段重叠 → 高级别中枢

关系：线段中枢的稳定性 > 笔中枢
      线段中枢的信号可靠性 > 笔中枢
      线段中枢更适合中长线交易者
```

### 4.2 算法

```python
def compute_segment_zs(segment_list):
    """
    从线段列表计算线段级别中枢
    逻辑与 compute_zhongshu() 完全一致，只是输入从笔列表换成线段列表
    """
    n = len(segment_list)
    if n < 3:
        return []
    
    zs_list = []
    i = 0
    while i <= n - 3:
        s0, s1, s2 = segment_list[i], segment_list[i+1], segment_list[i+2]
        ZG = min(s0.high, s1.high, s2.high)
        ZD = max(s0.low, s1.low, s2.low)
        
        if ZG > ZD:
            ZZ = (ZG + ZD) / 2
            zs_list.append({
                "start_dt": s0.sdt,
                "end_dt": s2.edt,
                "zg": ZG, "zd": ZD, "zz": ZZ,
                "width": ZG - ZD,
                "width_pct": (ZG - ZD) / ZZ * 100 if ZZ else 0,
                "segment_indices": [i, i+1, i+2],
                "level": "线段级别"
            })
        i += 1
    
    return zs_list
```

## 5. K 线图渲染

### 5.1 线段标记

线段在 K 线图上用 **粗线条或半透明色带** 标记，区别于笔的箭头标记：

| 标记 | 样式 |
|------|------|
| 向上线段 | 红色半透明带（`rgba(239,68,68,0.08)`），从线段起点到终点 |
| 向下线段 | 绿色半透明带（`rgba(16,185,129,0.08)`），从线段起点到终点 |
| 线段端点 | 小方块标记（symbol: 'rect', size: 8） |

### 5.2 线段中枢

线段中枢的渲染方式与笔中枢一致（金色半透明矩形 + ZG/ZD 虚线），但用更深的金色（`rgba(245,158,11,0.18)`）或更粗的虚线区分级别。

## 8. 已知限制

1. MVP 采用简化版线段规则，非缠论原文严格定义，可能存在边界情况未覆盖
2. 线段终结的判断条件（「被反向线段破坏」）在简化版中用「极值突破」替代，可能在震荡行情中产生偏差
3. 线段计算仅支持日线级别，周/月线暂不计算线段
4. 线段的背驰检测和买卖信号不在本版本范围内
