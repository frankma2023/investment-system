#### 2.5.3 突破失败形态（Failed Patterns）

##### 2.5.3.1 定义
“失败形态”是指欧奈尔体系中的经典买入形态（如杯柄、双重底、口袋支点、扁平基部等）在完成突破后，未能延续上涨趋势，反而在短时间内（通常1-2周）跌回突破点下方。这是**最强烈的卖出信号之一**，因为它直接证明之前的买入判断是错误的。

##### 2.5.3.2 常见失败形态

##### 类型一：杯柄形态突破失败
| 特征 | 量化规则 |
|------|----------|
| 前期 | 识别出杯柄形态，股价突破枢轴点，且突破日放量 |
| 失败表现 | 突破后5-10个交易日内，股价收盘价跌破突破日最低价，或跌破柄部最低价 |
| 成交量 | 下跌时成交量可能放大或维持高位，显示抛压沉重 |

**卖出点**：跌破突破日最低价或柄部最低价时，立即止损。

##### 类型二：双重底突破失败
| 特征 | 量化规则 |
|------|----------|
| 前期 | 双重底形成，股价突破中间高点 |
| 失败表现 | 突破后迅速回落，收盘价跌破中间高点（即突破点） |

**卖出点**：跌破中间高点时卖出。

##### 类型三：口袋支点失败
| 特征 | 量化规则 |
|------|----------|
| 前期 | 出现口袋支点信号（放量阳线） |
| 失败表现 | 支点出现后3-5个交易日内，股价跌破支点K线的最低价 |

**卖出点**：跌破支点K线最低价时卖出。

##### 类型四：高窄旗形失败
| 特征 | 量化规则 |
|------|----------|
| 前期 | 高窄旗形形成，股价突破旗面高点 |
| 失败表现 | 突破后一周内，股价跌破旗面高点或旗面最低价 |

**卖出点**：跌破旗面高点时减仓，跌破旗面最低价时清仓。

##### 2.5.3.3 失败形态的量化识别

```python
def is_failed_breakout(signal_df, price_data, days_lookback=10):
    """
    检测买入信号是否失败
    signal_df: 包含 signal_type, entry_price, breakout_low (信号K线最低价)
    price_data: 信号发生后的价格数据
    """
    if signal_df.empty:
        return False
    
    entry_low = signal_df['breakout_low'].iloc[0]  # 突破日最低价
    pivot_price = signal_df['pivot_price'].iloc[0] # 枢轴点
    
    # 检查后续价格行为
    for i in range(1, min(days_lookback, len(price_data))):
        # 条件1：收盘价跌破突破日最低价
        if price_data['close'].iloc[i] < entry_low:
            return True
        # 条件2：收盘价跌破枢轴点（更严格的失败）
        if price_data['close'].iloc[i] < pivot_price:
            return True
    return False
```

##### 2.5.4 头部与失败形态的卖出策略整合

| 信号类型 | 确认条件 | 卖出动作 | 优先级 |
|----------|----------|----------|--------|
| 头肩顶跌破颈线 | 收盘价 < 颈线 × 0.97 | 清仓 | 1（最高） |
| 双重顶跌破颈线 | 收盘价 < 中间低谷 × 0.97 | 清仓 | 1 |
| 三重顶跌破颈线 | 收盘价 < 最低回调点 × 0.97 | 清仓 | 1 |
| 杯柄突破失败 | 收盘价 < 突破日最低价 | 立即止损 | 2 |
| 双重底突破失败 | 收盘价 < 中间高点 | 立即止损 | 2 |
| 口袋支点失败 | 收盘价 < 支点K线最低价 | 立即止损 | 2 |

**注意**：失败形态的止损通常比7%止损更早触发（可能只有3%-5%的亏损），应严格执行，避免小亏变巨亏。

##### 2.5.5 关于失败形态量化的复杂度评估与实现建议

杯柄形态、口袋支点、高窄旗形等本身的识别逻辑已经相当复杂（涉及基部检测、波动率计算、成交量比较等），如果再叠加“突破失败”的量化——比如判断“突破后5日内是否跌回枢轴点下方”或“是否跌破突破日最低价”——确实会让代码膨胀，边界条件增多，维护成本上升。

但好消息是：**失败形态的检测可以独立于形态识别之外，且实现难度远低于形态识别本身**。下面给出具体分析和简化方案。

##### 2.5.5.1 失败形态检测的复杂度来源

| 复杂度来源 | 说明 | 是否必须 |
|-----------|------|---------|
| 需要形态识别结果 | 失败检测依赖“突破日”和“关键价位”（如枢轴点、突破日最低价） | ✅ 必须 |
| 需要后续价格监控 | 需要信号产生后5-10个交易日的价格数据 | ✅ 必须 |
| 需要区分不同形态的失败条件 | 杯柄失败看柄部最低价，口袋支点失败看支点K线最低价，高窄旗形看旗面高点 | ⚠️ 可统一简化 |
| 需要处理多信号重叠 | 同一股票可能同时有多个信号，失败条件不同 | ⚠️ 可简化 |

**结论**：核心复杂度不在于“失败检测”本身，而在于**需要准确获取形态识别阶段产生的关键价位**。如果形态识别代码已经能输出 `pivot_price`、`entry_price`、`breakout_low` 等字段，失败检测就是一个简单的价格比较循环。

###### 2.5.5.2 简化方案：统一失败检测逻辑

- 1) 关键价位统一定义
无论哪种形态，在生成买入信号时，**统一记录以下三个价位**：

| 字段 | 定义 | 示例（杯柄） | 示例（口袋支点） |
|------|------|-------------|-----------------|
| `pivot_price` | 形态的关键阻力位（突破目标价） | 柄部最高点 | 基部最高点 |
| `entry_low` | 信号K线的最低价（激进止损参考） | 突破日最低价 | 支点日最低价 |
| `pivot_low` | 形态内最近的一个支撑低点（保守止损参考） | 柄部最低价 | 10日均线价 |

- 2) 失败条件统一规则
信号产生后，每日监控以下条件（可配置监控天数，如5-10个交易日）：

| 条件 | 含义 | 触发动作 |
|------|------|----------|
| `收盘价 < entry_low` | 突破日买盘被完全吞噬 | 立即止损（失败确认） |
| `收盘价 < pivot_price` | 跌破关键阻力位 | 减仓50%或清仓 |
| `收盘价 < pivot_low × 0.98` | 跌破形态支撑 | 立即清仓 |

**伪代码**：
```python
def check_failed_breakout(signal, daily_prices, monitor_days=10):
    for day in daily_prices[:monitor_days]:
        if day['close'] < signal['entry_low']:
            return {'failed': True, 'reason': '跌破突破日最低价', 'day': day['date']}
        if day['close'] < signal['pivot_price']:
            return {'failed': True, 'reason': '跌破枢轴点', 'day': day['date']}
        if day['close'] < signal['pivot_low'] * 0.98:
            return {'failed': True, 'reason': '跌破形态支撑', 'day': day['date']}
    return {'failed': False}
```

##### 2.5.5.3 实际可行性：从“形态识别”到“失败监控”的工程路径

- 1）**已有基础**
之前脚本已经实现对口袋支点、标准突破等形态的识别，有以下字段的信号字典：
```python
{
    'signal_type': 'cup_handle_breakout',
    'entry_price': 125.50,       # 枢轴点
    'entry_price_high': 129.27,
    'pivot_price': 125.50,
    'volume_ratio': 1.8,
    'confidence': 'high',
    'details': {
        'cup_depth': 0.28,
        'handle_low': 118.20,    # 这就是 pivot_low
        ...
    }
}
```

只需要在信号生成时**额外存储 `entry_low`（突破日最低价）**，即可支持失败监控。

- 2）**需要新增的组件（仅供参考）**
  - **信号存储表**：增加字段 `breakout_low`、`pivot_low`、`monitor_end_date`。
  - **每日更新任务**：对所有未过监控期的信号，检查是否触发失败条件，更新状态。
  - **前端预警**：在K线图上标注失败信号，提示用户。

- 3）**不要过度设计**

  - **不需要**为杯柄、口袋支点、高窄旗形分别实现“专属失败规则”。统一规则已足够。
  - **不需要**实时计算“形态失效概率”等高级指标。简单的价格跌破即可。
  - **不需要**处理“假跌破后收回”的复杂逻辑。欧奈尔原则是“跌破关键位即卖出”，不猜测后市。

    **核心原则**：失败形态检测不是为了完美预测，而是为了**快速止损**。宁可误判一两次，也不要让一次真正的失败形态造成大额亏损。

##### 2.5.5.4 总结

| 问题 | 答案 |
|------|------|
| 失败形态量化的复杂度高吗？ | **不高**。独立于形态识别，只需比较价格与关键价位 |
| 需要为每种形态单独写规则吗？ | **不需要**。统一使用“跌破突破日最低价/枢轴点”即可 |
| 实现难度有多大？ | 约半天工作量，远低于形态识别 |
| 准确性能接受吗？ | 能覆盖80%以上的失败场景，且执行纪律优于复杂判断 |

**最终建议**：先实现统一失败检测（3-5个条件），运行一段时间后，再根据实际案例决定是否需要细化。不要一开始就追求完美，快速迭代更符合工程实践。
