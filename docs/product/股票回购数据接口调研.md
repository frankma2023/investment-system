# A股股票回购数据接口调研

> 调研日期：2026-05-14

---

## 1. 理杏仁

### 1.1 公告 API — 回购事件发现

**接口**：`POST https://open.lixinger.com/api/cn/company/announcement`

公告类型枚举中包含回购分类：

| 中文描述 | 代码 |
|----------|------|
| 回购 | `srp` |

返回的每条公告带有 `types` 数组，筛选 `types` 包含 `"srp"` 的记录即为回购公告。

**请求示例**：

```json
{
    "token": "...",
    "stockCode": "600519",
    "startDate": "2025-01-01",
    "endDate": "2026-05-14"
}
```

**局限**：仅返回公告标题、PDF 链接、公告类型，无结构化的回购金额/股数/价格数据。需解析公告 PDF 或配合其他数据源。

### 1.2 股本变动 API — 回购导致的股本变化

**接口**：`POST https://open.lixinger.com/api/cn/company/equity-change`

返回 `changeReason`（变动原因）和 `changeReasonTypes`。文档未列出完整枚举，示例中包含 `SPO`（增发）、`equityIncentive`（股权激励）、`equityOption`（期权行权）。是否包含 `buyback`/`repurchase` 类型需实测确认。

### 1.3 结论

理杏仁**没有独立的回购数据 API**。公告 API 能做事件发现，股本变动 API 能做定量验证（回购注销会导致总股本减少），但无法直接获取结构化的回购金额、股数、价格区间。

## 2. 东方财富

东方财富数据中心提供**两个完整的回购 API**，数据覆盖全面。

**API 基础地址**：`https://datacenter-web.eastmoney.com/api/data/v1/get`

**通用请求参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| `reportName` | 报表名 | 见下方两张表 |
| `columns` | `"ALL"` | 返回全部字段 |
| `pageNumber` | `1` | 页码 |
| `pageSize` | `500` | 每页条数（最大 500） |
| `source` | `"WEB"` | 固定 |
| `client` | `"WEB"` | 固定 |
| `filter` | 可选 | 如 `(DIM_SCODE="600519")` |

---

### 2.1 回购计划列表 — `RPTA_WEB_GETHGLIST_NEW`

> 每只股票的回购方案，含金额/股数/价格/目的/进度。
> 全量约 5100 条记录。

**请求**：

```json
{
    "reportName": "RPTA_WEB_GETHGLIST_NEW",
    "columns": "ALL",
    "pageNumber": 1,
    "pageSize": 500,
    "sortColumns": "NOTICEDATE",
    "sortTypes": -1,
    "source": "WEB",
    "client": "WEB"
}
```

**按股票筛选**：添加 `"filter": "(DIM_SCODE=\"600519\")"`

**主要字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `DIM_SCODE` | String | 股票代码 |
| `SECURITYSHORTNAME` | String | 股票简称 |
| `REPURCODE` | String | 回购编号 |
| `REPURPROGRESS` | String | 进度（`001`=实施中） |
| `REPUROBJECTIVE` | String | 回购目的（如"股权激励""注销""市值管理"） |
| `SHARETYPE` | String | 股份类型（如"流通A股"） |
| **金额** | | |
| `REPURAMOUNTLIMIT` | Number | 回购金额上限（元） |
| `REPURAMOUNTLOWER` | Number | 回购金额下限（元） |
| `REPURAMOUNT` | Number | 已回购金额（元） |
| **股数** | | |
| `REPURNUMCAP` | Number | 回购股数上限 |
| `REPURNUMLOWER` | Number | 回购股数下限 |
| `REPURNUM` | Number | 已回购股数 |
| **价格** | | |
| `REPURPRICECAP` | Number | 回购价格上限（元） |
| `REPURPRICELOWER` | Number | 回购价格下限（元） |
| **日期** | | |
| `REPURSTARTDATE` | Date | 回购起始日期 |
| `REPURENDDATE` | Date | 回购截止日期 |
| `NOTICEDATE` | Date | 公告日期 |
| `UPDATEDATE` | Date | 更新日期 |
| `FINISHDATE` | Date | 完成日期 |
| **进度统计** | | |
| `ZJSL` | Number | 已回购股数（汇总） |
| `ZJJE` | Number | 已回购金额（汇总） |
| `ZJSZBL` | Number | 占总股本比例 |

**示例数据**（002332 仙琚制药）：

```json

    "REPURAMOUNTLIMIT": 150000000,
```

### 2.2 回购实施明细 — `RPTA_WEB_GPHG`

> 每次实际回购的执行记录，含数量/金额/价格区间。
> 全量约 2552 条记录。

**请求**：

```json
{
    "reportName": "RPTA_WEB_GPHG",
    "columns": "ALL",
    "pageNumber": 1,
    "pageSize": 500,
    "sortColumns": "TDATE",
    "sortTypes": -1,
    "source": "WEB",
    "client": "WEB"
}
```

**按股票筛选**：添加 `"filter": "(SCODE=\"600519\")"`

**主要字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `SCODE` | String | 股票代码 |
| `SNAME` | String | 股票简称 |
| `HKCODE` | String | 回购批次编号 |
| `HGSL` | Number | 回购数量（股） |
| `HGJE` | Number | 回购金额（元） |
| `JLBZ` | String | 计价币种（CNY） |
| `HGZDJ` | Number | 回购最低价 |
| `HGZGJ` | Number | 回购最高价 |
| `SPJ` | Number | 当日收盘价 |
| `HGFS` | String | 回购方式（`001`/`002`） |
| `ZDF` | Number | 涨跌幅 |
| `GGRQ` | Date | 公告日期 |

## 3. 接口对比

| 维度 | 理杏仁 | 东方财富 |
|------|--------|----------|
| 回购方案（金额/股数/价格） | ❌ 无 | ✅ `RPTA_WEB_GETHGLIST_NEW` |
| 回购实施明细 | ❌ 无 | ✅ `RPTA_WEB_GPHG` |
| 回购公告事件 | ✅ 公告API (`srp`) | — |
| 回购导致股本变化 | ⚠️ 股本变动API（需实测） | — |
| 回购目的/进度 | ❌ 无 | ✅ 含目的文本+进度状态 |
| 数据量 | — | 方案 5100 条 + 明细 2552 条 |
| 全量 A 股覆盖 | ✅ | ✅ |

---

## 4. 建议使用策略

- **批量拉取回购方案**：用东方财富 `RPTA_WEB_GETHGLIST_NEW`，按 `NOTICEDATE` 排序获取最新回购计划
- **追踪回购执行**：用东方财富 `RPTA_WEB_GPHG`，按股票代码筛选获取历史执行明细
- **关注回购注销**：在 `REPUROBJECTIVE` 字段中搜索"注销"关键词
- **辅助验证**：理杏仁股本变动 API 验证回购注销是否导致总股本减少

## 5. Python 调用示例

```python
import requests

url = "https://datacenter-web.eastmoney.com/api/data/v1/get"
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)",
    "Referer": "https://data.eastmoney.com/gphg/",
}

# 5.1 获取全部回购计划（分页）
params = {
    "reportName": "RPTA_WEB_GETHGLIST_NEW",
    "columns": "ALL",
    "pageNumber": 1,
    "pageSize": 500,
    "sortColumns": "NOTICEDATE",
    "sortTypes": -1,
    "source": "WEB",
    "client": "WEB",
}
r = requests.get(url, params=params, headers=headers)
data = r.json()
print(f"回购计划总数: {data['result']['count']}")

# 5.2 获取单只股票的回购明细
params2 = {
    "reportName": "RPTA_WEB_GPHG",
    "columns": "ALL",
    "filter": '(SCODE="600519")',
    "pageNumber": 1,
    "pageSize": 100,
    "sortColumns": "TDATE",
    "sortTypes": -1,
    "source": "WEB",
    "client": "WEB",
}
r2 = requests.get(url, params=params2, headers=headers)
details = r2.json()['result']['data']
for d in details[:5]:
    print(f"  {d['TDATE'][:10]} 回购{d['HGSL']}股 {d['HGJE']}元 价格{d['HGZDJ']}-{d['HGZGJ']}")
```

## 6. 附录

### 6.1 回购进度状态码

| 代码 | 含义（推测） |
|------|-------------|
| `001` | 实施中 |
| `002` | 已完成 |
| `003` | 已失效/终止 |

### 6.2 回购方式代码

| 代码 | 含义 |
|------|------|
| `001` | 集中竞价 |
| `002` | 要约收购 / 其他 |

### 6.3 回购目的常见模式

- `"股权激励或员工持股计划"` — 不注销，未来会重新授予
- `"注销"` / `"减少注册资本"` — 回购注销，直接减少总股本
- `"市值管理"` — 维护股价
- `"可转债转股"` — 配合可转债

> 可通过 `REPUROBJECTIVE` 字段关键词筛选区分回购注销 vs 股权激励回购。
