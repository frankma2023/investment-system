## 2. 分层详解

### 2.1 数据层

职责：从 `lixinger.db` 读取指定股票/指数在指定日期范围的 OHLCV 数据。

```python
# 伪代码
conn = sqlite3.connect("data/lixinger.db")
df = pd.read_sql("""
    SELECT date, open, high, low, close, volume
    FROM daily_kline
    WHERE stock_code = ? AND date BETWEEN ? AND ?
    ORDER BY date
""", conn, params=[code, start, end])
```

**输出：** `pd.DataFrame` 按日期升序，列 `[open, high, low, close, volume]`。

### 2.2 指标层（TA-Lib）

职责：在原始 OHLCV 上计算技术指标，供下游形态引擎使用。**替代手写 Python 实现，性能提升 10-100 倍。**

```python
import talib
import numpy as np

close = df['close'].values.astype(np.float64)
high  = df['high'].values.astype(np.float64)
low   = df['low'].values.astype(np.float64)
open_ = df['open'].values.astype(np.float64)
vol   = df['volume'].values.astype(np.float64)

indicators = {
    'sma5':   talib.SMA(close, 5),
    'sma10':  talib.SMA(close, 10),
    'sma20':  talib.SMA(close, 20),
    'sma50':  talib.SMA(close, 50),
    'sma120': talib.SMA(close, 120),
    'sma250': talib.SMA(close, 250),
    'bb_upper': talib.BBANDS(close, 20, 2, 2, 0)[0],
    'bb_middle': talib.BBANDS(close, 20, 2, 2, 0)[1],
    'bb_lower': talib.BBANDS(close, 20, 2, 2, 0)[2],
    'atr':    talib.ATR(high, low, close, 14),
    'rsi':    talib.RSI(close, 14),
    'vol_ma50': talib.SMA(vol, 50),
}
```

**对现有引擎的影响：**

| 引擎 | 原来 | 替换为 TA-Lib |
|---|---|---|
| breakout_scanner | 手写 SMA(5/10/20/50) 循环 | `talib.SMA(close, period)` |
| pocket_pivot | 手写 SMA(10/50) + MA50 | `talib.SMA` + `talib.SMA` |
| flat_base (新) | 需手写 BBANDS | `talib.BBANDS(close, 20, 2, 2, 0)` |

### 2.3 形态层（自研引擎）

职责：每个引擎独立输入 K 线数据 + 指标，输出自身形态的信号列表。引擎之间无依赖，可并行运行。

**统一信号格式：**

```python
{
    "type": "double_bottom",    # 信号类型标识
    "date": "2025-03-15",       # 信号日期
    "pivot": 32.50,             # 关键价格（突破点/枢轴点）
    "confidence": "medium",     # 置信度（可选, 由确认层填充）
    "details": { ... }          # 形态专属详细字段
}
```

**引擎调用：**

```python
def scan_all(klines, indicators):
    signals = []
    signals.extend(double_bottom_detect(klines, indicators))
    signals.extend(breakout_detect(klines, indicators))
    signals.extend(pocket_pivot_detect(klines, indicators))
    signals.extend(flat_base_detect(klines, indicators))

```

### 2.4 确认层（TA-Lib CDL，可选）

职责：对形态层输出的信号日期，检查附近是否出现日本蜡烛图模式。**不是独立信号源，而是信号增强/过滤层。**

```python
def enhance_signals(signals, open_, high, low, close):
    """对每个信号, 检查突破日附近的蜡烛图确认"""
    for sig in signals:
        idx = sig['date_index']
        if idx < 3:
            continue
        
        # 检查突破日 ±1 天的蜡烛图信号
        bullish = 0
        bearish = 0
        for offset in [-1, 0, 1]:
            i = idx + offset
            if talib.CDLENGULFING(open_, high, low, close)[i] > 0:
                bullish += 1
            if talib.CDLHAMMER(open_, high, low, close)[i] > 0:
                bullish += 1
            if talib.CDLEVENINGSTAR(open_, high, low, close)[i] < 0:
                bearish += 1
        
        if bullish >= 1 and bearish == 0:
            sig['confidence'] = 'high'
        elif bearish >= 1:
            sig['confidence'] = 'low'
        else:
            sig['confidence'] = 'medium'
    
    return signals
```

**确认层规则：**

- **绿灯（high）**：突破日附近出现吞没/锤子线/晨星 → 提高信号可信度
- **黄灯（medium）**：无蜡烛图信号 → 仅依赖形态层判断
- **红灯过滤（low）**：突破日附近出现黄昏之星/乌云盖顶/墓碑十字 → 降低信号可信度（仍输出但标注）

## 3. TA-Lib 与 py-candlekit 的对比决策

### 3.1 py-candlekit — 不安装

| 理由 | 说明 |
|---|---|
| 功能冗余 | TA-Lib 的 CDL 函数已包含 py-candlekit 全部形态（且更多：61 vs 20） |
| 性能劣势 | 纯 Python + Pandas，大数据量下比 TA-Lib 的 C 底层慢 10-50 倍 |
| 依赖叠加 | 多一个依赖多一层维护成本，无增量价值 |
| 生态成熟度 | TA-Lib 是行业标准，py-candlekit 是个人项目（v1.0.1，GitHub 更新少） |

### 3.2 TA-Lib — 已安装，用于三个场景

| 场景 | 用途 | 优先级 |
|---|---|---|
| **技术指标计算** | SMA、BBANDS、ATR、RSI → 替代所有引擎的手写指标实现 | **P0 必须** |
| **K 线确认信号** | 对形态层输出的信号做蜡烛图增强，提高置信度 | P2 可选 |
| **反向过滤** | 排除出现看跌蜡烛图形态的假突破 | P2 可选 |

## 5. 技术要点备忘

### 5.1 TA-Lib 调用注意事项

```python
# TA-Lib 要求 float64 输入, 否则静默出错
close = df['close'].values.astype(np.float64)

# 早期数据不足时返回 NaN, 需过滤
sma50 = talib.SMA(close, 50)  # 前 49 个值为 NaN

# CDL 函数返回值: 正数=看涨, 负数=看跌, 0=无信号
engulf = talib.CDLENGULFING(open_, high, low, close)
# engulf > 0 → 看涨吞没 (Bullish Engulfing)
# engulf < 0 → 看跌吞没 (Bearish Engulfing)
```

### 5.2 性能对比

| 操作 | 纯 Python（手写） | TA-Lib | 提升 |
|---|---|---|---|
| SMA(50) × 2500 天 | ~5ms | ~0.05ms | 100x |
| BBANDS(20,2) × 2500 天 | ~15ms | ~0.1ms | 150x |
| CDL 函数 × 2500 天 | — | ~0.3ms | — |
