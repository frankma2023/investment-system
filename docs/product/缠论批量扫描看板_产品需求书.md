---

## 1. 产品概述

### 1.1 定位

在缠论分析引擎（`chanlun.py`）和区间套功能的基础上，提供**关注池股票的批量扫描看板**。每天盘后自动对观察池+精选池的股票运行缠论分析，次日用户在统一看板上按信号类型筛选、排序、跳转详情。

### 1.2 为什么不做全市场

全市场 5000 只股票单线程约需 2~4 小时，性价比低。观察池+精选池约 670 只，实测约 8 分钟即可完成，且这些恰是用户实际关注的标的，信号更有决策价值。

### 1.3 核心价值

- **一次看全部**：不用逐只输入代码，打开看板就能看到所有关注股票的缠论信号概览
- **快速筛选**：按信号类型（一买/一卖/三买/三卖/底背驰/顶背驰）和笔方向过滤
- **一键跳转**：点击代码直接进入详细分析页面

## 3. 数据表设计

### 3.1 `chanlun_scan_daily`

```sql
CREATE TABLE IF NOT EXISTS chanlun_scan_daily (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    scan_date TEXT NOT NULL,           -- 扫描日期 YYYY-MM-DD
    stock_code TEXT NOT NULL,          -- 股票代码
    stock_name TEXT,                   -- 股票名称
    bi_count INTEGER DEFAULT 0,        -- 笔数量
    zs_count INTEGER DEFAULT 0,        -- 中枢数量
    segment_count INTEGER DEFAULT 0,   -- 线段数量
    latest_bi_dir TEXT,                -- 最新笔方向 (向上/向下)
    latest_bi_power REAL,              -- 最新笔力度
    divergence_count INTEGER DEFAULT 0,-- 背驰信号数
    latest_div_type TEXT,              -- 最新背驰类型 (底背驰/顶背驰)
    trade_signal_count INTEGER DEFAULT 0, -- 买卖信号数
    latest_trade_type TEXT,            -- 最新买卖信号类型 (一买/一卖/三买/三卖)
    latest_trade_side TEXT,            -- buy/sell
    latest_trade_price REAL,           -- 最新买卖信号价格
    resonance_strength TEXT,           -- 多周期共振强度（待实现）
    created_at TEXT DEFAULT (datetime('now','localtime')),
    UNIQUE(scan_date, stock_code)
);
```

### 3.2 字段说明

| 字段 | 用途 | 筛选器使用 |
|------|------|:--:|
| `latest_bi_dir` | 当前走势方向 | ✓ |
| `latest_trade_type` | 最新买卖信号类型 | ✓ |
| `latest_div_type` | 最新背驰类型 | ✓ |
| `latest_trade_side` | 看涨/看跌 | ✓ |


### 4.2 筛选栏

| 筛选项 | 类型 | 选项 |
|--------|------|------|
| 信号类型 | 按钮组 | 全部 / 一买 / 一卖 / 三买 / 三卖 |
| 背驰类型 | 按钮组 | 全部 / 底背驰 / 顶背驰 |
| 方向 | 按钮组 | 全部 / 看涨(buy) / 看跌(sell) |
| 笔方向 | 按钮组 | 全部 / ↑向上 / ↓向下 |

> 四个维度可**组合筛选**（如同时选「一买」+「底背驰」+「↑向上」），再次点击取消。维度间独立，维度内互斥。

### 4.3 表格列

| 列 | 内容 | 样式 |
|----|------|------|
| 代码 | 股票代码，可点击 | 链接跳转到 `../chanlun-backtest/?code=xxx` |
| 名称 | 股票简称 | — |
| 笔 | `bi_count` | tabular-nums |
| 中枢 | `zs_count` | — |
| 线段 | `segment_count` | — |
| 方向 | ↑/↓ | 红涨绿跌 |
| 力度 | `latest_bi_power` | tabular-nums |
| 背驰 | `divergence_count` + `latest_div_type` | 底背驰红/顶背驰绿 |
| 信号 | `latest_trade_type` | 买点红底/卖点绿底标签 |
| 价格 | `latest_trade_price` | tabular-nums |

### 4.4 详情跳转

点击股票代码 → `../chanlun-backtest/?code=xxx`，在新标签页打开完整分析。

## 5. 实现方式

### 5.1 无 API 依赖

看板采用**静态 HTML + 内嵌数据**的方式，和 `daily-pattern-scan` 一致：

- 扫描脚本在写入数据库的同时，生成一个 HTML 文件
- HTML 文件内嵌当日扫描的 JSON 数据
- 前端纯客户端渲染：筛选、排序、分页全部由 JS 在浏览器内完成
- 不需要额外的 Flask API 端点

### 5.2 生成流程

```
chanlun_scan.py --date 2026-05-30
  → 写入 chanlun_scan_daily 表
  → 读取当日数据
  → 渲染为 HTML（内嵌 JSON）
  → 保存到 web/chanlun-scan/index.html
```

### 5.3 前端技术

- 纯原生 JS，无框架
- `hanako-glass.css` 设计系统
- `nav.js` 统一导航
- 客户端筛选+排序（数据量小，约 100~200 行）
