# 指数拥挤度回测页面产品说明 v1.0

> 最后更新：2026-05-06  
> 对应引擎：`src/scanners/index_crowding.py`  
> 配置文件：`config/index_crowding.yaml`  
> 计算依赖：`index_fundamental_daily` 表（`scripts/fetch_index_fundamental.py` 拉取）

## 1. 回测页面总体需求

- 使用**小红书卡片风格**（与抛盘日回测、追盘日回测、指数RS强度回测页面保持一致），共享 `web/shared/themes/xhs-backtest.css` 主题文件
- 导航栏与所有回测看板互链：抛盘日 / 追盘日 / 指数RS强度 / 指数拥挤度
- 页面布局：**左侧参数配置面板 + 右侧结果展示区**
- 日期区间选择：使用日历选择器（`<input type="date">`），用户选择起止日期区间。拥挤度为时间序列计算，需要滚动窗口（120个交易日），因此使用日期范围而非单日
- 用户调整权重参数后点击「运行回测」按钮，触发 `POST /api/crowding/backtest` 请求，右侧展示结果
- 点击「保存配置」，将当前参数写入 `config/index_crowding.yaml`，拥挤度引擎从该文件读取权重和等级阈值
- **不持久化计算结果**，每次请求实时计算

## 2. 产品背景

指数拥挤度（Index Crowdedness）用于衡量市场资金在某一指数上的集中程度与交易热度。高拥挤度预示交易过度一致、筹码拥挤，后续可能出现趋势反转或波动加剧；低拥挤度代表关注度低、安全边际较高。

在最强指数看板中，拥挤度作为风险预警与辅助决策信号：用户筛选强势指数后，结合拥挤度判断是否高估风险，避免追高「强势但拥挤」的指数。

### 2.1 指标体系概览

| 维度 | 权重(v1) | 包含指标 |
|------|:--:|------|
| 交易热度 | 35% | 成交额占比分位点、换手率分位点 |
| 资金流向 | 25% | 融资余额占比分位点、融资买入额占比分位点 |
| 估值水位 | 25% | PE十年分位点、PB十年分位点、股息率十年分位点 |
| 机构行为 | 15% | 基金重仓占比分位点（v1暂无数据，权重按比例重分配） |

> ⚠️ v1 阶段跳过了「成分股相关系数」和「卖方研报覆盖量」两个指标（计算资源与数据源限制），待 v2 评估。

## 3. 左侧参数配置面板

### 3.0 通用选择器
- **指数池选择**：下拉菜单，5 个分类可选（全市场30 / 一级行业11 / 二级行业45 / 主题229 / 策略93），也可选「全部指数(407)」
- **日期区间**：日历组件选择起止日期，默认 2026-01-01 ~ 最新交易日

### 3.1 卡片1：交易热度（权重 35%）

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 成交额占比权重 | 25% | 0~50% | 指数成交额/全市场成交额（中证全指000985），取120日滚动分位点 |
| 换手率权重 | 10% | 0~50% | 指数自由流通股本换手率，取120日滚动分位点 |

### 3.2 卡片2：资金流向（权重 25%）

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 融资余额占比权重 | 15% | 0~50% | 融资余额/自由流通市值，取120日滚动分位点 |
| 融资买入额占比权重 | 10% | 0~50% | 融资买入额/指数成交额，取120日滚动分位点 |

### 3.3 卡片3：估值水位（权重 25%）

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| PE十年分位点权重 | 15% | 0~50% | 市值加权PE-TTM在近10年中的百分位，理杏仁API原生返回 |
| PB十年分位点权重 | 5% | 0~50% | 市值加权PB在近10年中的百分位 |
| 股息率分位点权重 | 5% | 0~50% | 市值加权股息率在近10年中的百分位 |

### 3.4 卡片4：机构行为（权重 15%）

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 基金重仓占比权重 | 15% | 0~50% | 主动权益基金持有指数成分股总市值/基金净值总额（季度），取历史分位点 |

> ⚠️ v1 阶段基金重仓数据尚未拉取，该指标无实际数值。引擎自动检测缺失指标，将其权重按比例重新分配给其余可用指标。卡片底部标注「暂无数据，权重将按比例重分配」。

### 3.5 权重归一化说明

滑块设置的是各指标的相对重要程度，引擎在计算时自动对可用指标的权重归一化到总和 100%。用户无需手动保持权重总和为 100%，只需调整相对比例。

### 3.6 等级阈值（固定在引擎中，v1 不开放调整）

| 得分区间 | 拥挤等级 | 颜色标识 |
|---------|---------|---------|
| < 30 | 低拥挤 | 🟢 绿色 |
| 30 ~ 60 | 正常 | ⚪ 灰色 |
| 60 ~ 80 | 偏高 | 🟡 黄色 |
| ≥ 80 | 高拥挤 | 🔴 红色 |

## 4. 右侧结果展示区

### 4.1 统计卡片

页面顶部 5 格统计卡片，显示当前回测结果的分布概览：

| 卡片 | 内容 |
|------|------|
| 指数总数 | 本次回测覆盖的指数数量 |
| 高拥挤 | 拥挤度 ≥ 80 的指数数量（红色） |
| 偏高 | 拥挤度 60~80 的指数数量（黄色） |
| 正常 | 拥挤度 30~60 的指数数量（灰色） |
| 低拥挤 | 拥挤度 < 30 的指数数量（绿色） |

### 4.2 拥挤度排名表格

按复合拥挤度得分降序排列，展示全部计算结果（最多100个指数）：

| 列名 | 数据来源 | 格式 |
|------|---------|------|
| 代码 | stock_code | 蓝色可点击链接，弹出成分股弹窗 |
| 名称 | 指数中文名称 | 加粗 |
| 拥挤度 | composite_score | 0~100，保留1位小数，加粗 |
| 等级 | crowd_level | 色块徽章：红底（高拥挤）/ 黄底（偏高）/ 灰底（正常）/ 绿底（低拥挤） |
| 热度 | heat_score | 交易热度维度得分，0~100 |
| 资金 | flow_score | 资金流向维度得分，0~100 |
| 估值 | valuation_score | 估值水位维度得分，0~100 |
| PE分位 | pe_pct | 百分比，如 85% |
| PB分位 | pb_pct | 百分比 |
| 成交占比分位 | turnover_ratio_pct | 百分比 |
| 换手率分位 | turnover_rate_pct | 百分比 |

> 所有列支持点击表头升序/降序排序。

### 4.3 成分股弹窗

点击表格中指数代码，弹出模态弹窗显示该指数的成分股及权重：

- 弹窗标题：指数代码 + 名称
- 副标题：快照日期 + 成分股数量
- 表格列：代码、名称、权重（前十大有精确百分比，其余显示「—」）
- 点击遮罩层或右上角 ✕ 关闭
- 数据来源：`/api/index-constituents?index_code=xxx`

## 5. 技术实现

### 5.1 样式
- 使用小红书回测主题 `web/shared/themes/xhs-backtest.css`
- 子弹日记点阵背景、贴纸卡片、和纸胶带装饰
- 支持深色/浅色模式切换（默认浅色）

### 5.2 数据流

**回测计算**：
```
前端参数 → POST /api/crowding/backtest
         → server.py 调用 index_crowding.compute_for_api()
         → 从 index_fundamental_daily 按需加载指数数据
         → 计算各指标滚动分位点 → 合成加权得分
         → 返回 JSON { results: [...], params: {...}, count: N }
         → 前端渲染表格和统计
```

**配置保存**：
```
前端参数 → POST /api/crowding/config
         → 写入 config/index_crowding.yaml
```

**指数池加载**：
```
页面初始化 → GET /api/crowding/indices
           → 从 index_fundamental_daily 取所有有数据的指数
           → 从 config/index_style.yaml 读取分类名称
           → 返回 408 个指数的 {code, name, category}
```

**成分股弹窗**：
```
点击指数代码 → GET /api/index-constituents?index_code=xxx
             → 查询 index_constituents + index_constituent_weightings
             → 返回 {constituents: [{stock_code, name, weighting}], snapshot_date, count}
```

### 5.3 计算引擎

`src/scanners/index_crowding.py` 核心函数：

| 函数 | 用途 |
|------|------|
| `compute_for_api(index_codes, start_date, end_date, weights, levels)` | API 入口，返回所有指数的拥挤度得分 |
| `calc_crowding(index_data, market_ta_dict, weights, levels)` | 单指数拥挤度序列计算 |
| `rolling_percentile(values, window=120)` | 滚动分位点计算（滑动窗口120交易日） |
| `load_config()` / `save_config()` | YAML 配置读写 |

## 6. 使用场景与局限

### 6.1 典型使用流程
1. 选择指数池（如「一级行业指数」）
2. 设置日期区间（如 2025-01-01 ~ 2026-05-05）
3. 调整权重参数（如提高估值水位的占比）
4. 点击「运行回测」查看结果
5. 对高拥挤指数点击代码，查看成分股构成
6. 满意参数组合后，点击「保存配置」持久化

### 6.2 局限与注意事项
- **滞后性**：基金持仓数据更新频率低（季度），v1 暂无该数据
- **阈值敏感性**：各指标分位数阈值及合成权重可通过历史回测优化
- **结构性牛市**：长期主线（如2019-2021白酒、医药）中拥挤度可能长期维持高位但趋势继续，应结合RS趋势和均线系统共同判断
- **最大100指数限制**：单次回测最多计算100个指数（API性能考虑）
