# 杯柄形态（Cup with Handle）突破检测引擎 — 产品需求书 v1.0

> **突破日驱动 + 经典杯柄结构 + 参数即标准**。判断"今天是否为杯柄形态的有效突破日"。O'Neil 体系中最重要的买入形态，特征为 U 形杯身 + 紧凑柄部。

---

## 1. 产品概述

本产品回答一个单一问题：**今天，某只股票是否出现杯柄形态的有效突破信号？**

杯柄形态（Cup with Handle）是 William J. O'Neil 定义的七种经典基部形态中最常见、最可靠的一种。结构上由两部分组成：一个 U 形的杯身（股价从前高回调 12%-33% 后回升），以及一个紧凑的柄部（回升到杯口附近后出现短暂回落，约 1-2 周）。

**核心流程**：检测日 t 的收盘价突破柄部高点（或无柄时突破杯口上沿）→ 向前回溯验证是否存在完整的杯柄结构 → 通过则输出信号。

**信号是二元的。** 没有评分。想要更严格？收紧参数。

### 1.1 与其他基部形态的区别

| 维度 | 杯柄形态 | 双重底 | 扁平基部 | 碟形基部 |
|---|---|---|---|---|
| 回调深度 | **12%~33%** | 15%~35% | 8%~15% | 8%~20% |
| 底部形状 | **U 形圆底** | W 形双底 | 水平窄箱体 | 浅圆弧形 |
| 形成时间 | 7~65 周 | 7~8 周以上 | 5~7 周 | 7~40 周 |
| 柄部 | **必须有（或极罕见无柄）** | 可选 | 无 | 可选 |
| 突破点 | **柄部上沿** | 中间峰最高价 | 基部最高点 | 碟口上沿（前高） |

## 2. 形态结构定义

```
杯口上沿（前高）──────────────────── 前高
   \                                   /
    \    ┌─── 柄部（回调 ≤ 12%）────┐  /
     \   │  成交量极度萎缩           │ /
      \  │  柄部低点在上半部         │/
       \ │                          /
        \│  杯底（U 形圆底）        /
         └────────────────────────┘
          ← 回调 12%~33%，≥ 7 周 →
```

### 2.1 关键节点

| 节点 | 定义 | 作用 |
|---|---|---|
| **前高** | 杯口上沿，形态开始前的最高收盘价 | 形态的上边界，前置上涨的终点 |
| **杯底** | 前高之后回调的最低收盘价 | 杯身的最低点，回调 12%~33% |
| **杯口** | 杯底反弹后接近前高的位置 | 突破买点参考（无柄时直接突破此处） |
| **柄部高点** | 回升到杯口附近后形成的局部高点 | **突破买点 = 柄部高点 + 0.01** |
| **柄部低点** | 柄部回调的最低点 | 必须在上半部，回调 ≤ 12% |
| **突破日** | 收盘价突破柄部高点的日期 | 买入信号日 |

## 3. 量化计算规则

> **设计原则**：所有条件分两级——**必要条件**（不通过=不是杯柄突破）和**可关开关**（默认开启）。没有评分。

### 3.1 前置条件（全部必要条件）

```
条件 0.1: 前置上涨验证
          在前高确定后，在前高之前 120 个交易日内找最低点作为"前低"
          要求 (前高 - 前低) / 前低 ≥ min_prior_advance（默认 0.30）
          → 不满足？不是杯柄形态（可能是熊市反弹）

条件 0.2: 趋势确认
          检测日收盘 > SMA50

条件 0.3: 数据充分
          至少需要 lookback + 50 个交易日的日 K 线数据

条件 0.4: 市值过滤（可关开关）
          流通市值 < min_market_cap（默认 0 = 关闭）
```

### 3.2 杯身识别（核心算法）

#### 步骤 1：定位前高和杯底 —— 必要条件

```
1.1 在检测日 t 之前 lookback（默认 120）天内，找到最高收盘价 = 前高
    记录: 前高日期 prior_high_date, 前高价格 prior_high_price

    ⚠️ 简单取最高收盘价可能误抓旧周期的高点。
    例: 旧周期反弹峰 80 元在 lookback 窗口内，杯体左沿实际是 65 元。
    取 80 会算出回调 (80-50)/80=37.5%>33%，排除掉一个标准杯柄。

    解决: prior_high_mode（默认 "simple"）可用两种模式:
      - "simple": 直接取 lookback 内最高收盘价（快速，适合大多数场景）
      - "cutting": 形态切割法——先找杯底，从杯底逆向爬坡找前高，
        遇 ≥20% 大波动即切割（与碟形基部 v3.0 的切割法一致）。
        切换方法: prior_high_mode = "cutting"

1.2 在前高之后找到最低收盘价 = 杯底
    记录: 杯底日期 bottom_date, 杯底价格 bottom_price

    必要条件:
      前高距检测日 ≥ cup_min_age（默认 35 天，约 7 周）
      前高距检测日 ≤ cup_max_age（默认 325 天，约 65 周）— O'Neil 原文上限
      杯底在前高之后: bottom_date > prior_high_date
      杯底距前高 ≥ min_descent_days（默认 10 天）— 不是尖顶一日游
      杯底距检测日 ≥ min_ascent_days（默认 10 天）— 反弹已确认

1.3 验证回调深度:
      drawdown = (prior_high_price - bottom_price) / prior_high_price
      要求 ∈ [cup_drawdown_min, cup_drawdown_max]（默认 [0.12, 0.33]）
      → 太浅（< 12%）不够清洗浮筹，太深（> 33%）是崩盘不是基部

1.4 验证杯底不是尖底:
      杯底周围 ±5 天振幅 ≤ cup_bottom_flatness（默认 0.08，即 8%）
      → 放宽于碟形（碟形要求 5%）——杯柄的杯底可以有弧度但不应是 V 形尖底
      → 可关开关（cup_bottom_check，默认 true）
```

#### 步骤 2：验证杯身回升 —— 必要条件

```
2.1 杯底反弹之后，收盘价必须回升到接近前高:
      杯底到检测日之间的最高收盘价 ≥ 前高 × cup_recovery（默认 0.90）
      → 回升到前高 90% 以上才可能形成杯口（O'Neil 的杯口在"前高附近"）

2.2 回升路径验证:
      回升段线性回归斜率 > 0（确认上升趋势）
      回升段不存在急剧拉升（单周涨幅 > 20% 的次数 = 0）
      → 杯身右侧应是温和的回升，不是 V 形反转

2.3 【左右侧速度对比】（可关开关，默认开启）
      回升段周数 ≥ 下行段周数 × ascent_descent_ratio（默认 0.50）
      → 右侧回升时间不少于左侧下降时间的一半
      → 区分 U 形杯体（右侧 ≥ 左侧 × 0.5）和 V 形反转（右侧远短于左侧）
      → 可关开关: ascent_descent_check = false 即关闭
```

#### 步骤 3：验证成交量 —— 必要条件

```
条件 3.1: 杯底区间的日均成交量 < 50 日均量 × vol_bottom_max（默认 0.60）
          → 杯底萎缩（杯柄的杯底量要求比碟形宽松——碟形 0.50，杯柄 0.60）

条件 3.2: 杯身下行段后半日均量 < 下行段前半日均量 × vol_contraction（默认 0.65）
          → 越跌量越小

条件 3.3: 回升段日均量 ≥ 下行段后半日均量
          → 右侧放量推升
```

### 3.3 柄部（Handle）检测 —— 必要条件

杯柄形态的柄部是**必要组件**（极罕见的无柄杯柄可通过 `handle_required = false` 放行）。

```
步骤 H1: 在杯底反弹后、检测日之前，找到回升段的最高收盘价
         要求该最高价 ≥ 前高 × 0.85（回升到杯口附近）
         该日期 = 杯口日期

步骤 H2: 在杯口日期之后、检测日之前，找到最低收盘价 = 柄部低点
         记录: handle_low_date, handle_low_price

    必要条件:
      柄部高点到柄部低点 ≥ handle_min_days（默认 5 天，约 1 周）
      柄部高点到柄部低点 ≤ handle_max_days（默认 30 天，约 6 周）
      柄部回调: (handle_high_price - handle_low_price) / handle_high_price ≤ handle_max_drawdown（默认 0.12）
      → 柄部回调不超过 12%（O'Neil 标准 ≤ 8%-12%）

步骤 H3: 验证柄部位置 —— 必须在上半部
      handle_low_price ≥ bottom_price + 0.50 × (prior_high_price - bottom_price)
      → 柄部低点必须高于杯身中点

步骤 H4: 验证柄部成交量
      柄部期间日均量 < 杯身整体日均量 × handle_vol_ratio（默认 0.50）
      → 柄部的量不到杯身的一半（最后的清洗，极度安静）

步骤 H5: 验证柄部形态
      柄部区间线性回归斜率 ≤ 0（向下或水平，不应向上倾斜）
      → 柄部是向下漂洗，不是提前突破

步骤 H6: 柄部低点 ≥ SMA50
      → 均线支撑
```

### 3.4 突破日验证 —— 必要条件

```
条件 B1: 价格突破
          有柄: 检测日收盘 ≥ 柄部高点 + breakout_buffer（默认 0.01 元）
          无柄: 检测日收盘 ≥ 前高 + breakout_buffer

条件 B2: 成交量放大
          检测日成交量 ≥ 50 日均量 × breakout_vol_ratio（默认 1.4）
          注意: 50 日均量取 [t-50, t-1]，不含今天

条件 B3: 阳线（可关开关）
          检测日收盘 > 开盘

条件 B4: 收盘在高位
          收盘位置 ≥ close_position_min（默认 0.50）

条件 B5: 均线多头
          收盘 > SMA50 且 > SMA150
```

### 3.5 虚假突破排除 —— 必要条件

- 检测日或次日盘中最低价 < 买点（以最低价计）
- 检测日成交量 < 50 日均量
- 检测日前 5 个交易日内存在放量下跌
- 柄部形成期间成交量放大（而非萎缩）

### 3.6 RS 强度验证 —— 可关开关

```
默认关闭。杯柄形态突破时 RS 通常较高（≥ 80），但不应因 RS 数据缺失而阻断检测。

条件: rs_20 ≥ rs_threshold 或 rs_60 ≥ rs_threshold 或 rs_250 ≥ rs_threshold
      默认 rs_threshold = 80, rs_required = false
```

## 4. 参数调优指南

| 想达到的效果 | 改什么参数 | 怎么改 |
|---|---|---|
| 只抓深回调杯柄（教科书级） | `cup_drawdown_min` | 从 0.12 → 0.15 |
| 容忍浅回调的杯柄 | `cup_drawdown_min` | 从 0.12 → 0.08 |
| 排除深度崩盘后杯柄 | `cup_drawdown_max` | 从 0.33 → 0.25 |
| 只抓有柄杯柄（标准） | `handle_required` | 保持 true |
| 放开无柄杯柄 | `handle_required` | true → false |
| 柄部更严格（回调更小） | `handle_max_drawdown` | 从 0.12 → 0.08 |
| 信号更少但更准 | `breakout_vol_ratio` | 从 1.4 → 1.6 |
| 要求 RS 强度 | `rs_required` | false → true |
| A 股波动大放宽 | `cup_drawdown_max` + `vol_bottom_max` | 调到 0.35 / 0.70 |

## 5. 信号输出格式

| 字段 | 类型 | 说明 |
|---|---|---|
| `signal_date` | date | 突破信号日期 |
| `stock_code` | str | 股票代码 |

| `descent_days` | int | 前高到杯底下行天数 |

| `handle_high_price` | float | 柄部高点价格 |
| `handle_low_date` | date | 柄部低点日期 |
| `handle_low_price` | float | 柄部低点价格 |
| `handle_drawdown_pct` | float | 柄部回调 % |
| `handle_vol_ratio` | float | 柄部日均量 / 杯身日均量 |
| `buy_point` | float | 买点 = 柄部高点 + 0.01（或无柄前高+0.01） |
| `breakout_close` | float | 突破日收盘价 |
| `breakout_vol_ratio` | float | 突破日量 / 50 日均量 |
| `breakout_chg_pct` | float | 突破日涨幅 % |
| `close_position` | float | 收盘位置 |
| `prior_advance_pct` | float | 前置涨幅 % |
| `rps_20` | int | RS 20 日 |
| `rps_60` | int | RS 60 日 |
| `rps_250` | int | RS 250 日 |
| `market_cap` | float | 流通市值（亿） |
| `close_vs_sma50` | float | 收盘 / SMA50 |

**接口**：
- 单股票：`GET /api/cup-handle?stock=600519&date=2026-05-16`
- 批量扫描：`GET /api/cup-handle/scan?date=2026-05-16&pool=hs300`
- 看板：`/cup-handle-backtest/`

## 6. 参数配置表

| 参数 | 默认值 | 可调范围 | 类型 | 说明 |
|---|---|---|---|---|
| `lookback` | 120 | 60~250 | 必要 | 寻找前高的回溯天数 |
| `prior_high_mode` | simple | simple/cutting | 必要 | 前高定位模式：simple=取最高收盘，cutting=形态切割法 |
| `min_prior_advance` | 0.30 | 0.20~0.50 | 必要 | 前置最小涨幅 |
| `cup_min_age` | 35 | 20~60 | 必要 | 前高距检测日最少天数（≈7 周） |
| `cup_max_age` | 325 | 100~400 | 必要 | 前高距检测日最多天数（≈65 周），O'Neil 上限 |
| `min_descent_days` | 10 | 5~20 | 必要 | 前高到杯底最少下行天数 |
| `min_ascent_days` | 10 | 5~20 | 必要 | 杯底到检测日最少回升天数 |
| `cup_drawdown_min` | 0.12 | 0.08~0.20 | 必要 | 杯身最小回调深度 |
| `cup_drawdown_max` | 0.33 | 0.25~0.40 | 必要 | 杯身最大回调深度 |
| `cup_bottom_check` | true | — | 开关 | 是否检查杯底不是尖底 |
| `cup_bottom_flatness` | 0.08 | 0.05~0.12 | 必要 | 杯底 ±5 天最大振幅 |
| `cup_recovery` | 0.90 | 0.80~0.95 | 必要 | 回升到前高比例下限 |
| `ascent_descent_check` | true | — | 开关 | 是否检查右侧回升时间 ≥ 左侧 × 0.5 |
| `ascent_descent_ratio` | 0.50 | 0.30~0.80 | 必要 | 回升周数/下行周数最小比 |
| `vol_bottom_max` | 0.60 | 0.40~0.80 | 必要 | 杯底量 / 50 日均量上限 |
| `vol_contraction` | 0.65 | 0.50~0.85 | 必要 | 下行后半量 / 下行前半量上限 |
| `handle_required` | true | — | 开关 | 无柄是否不输出信号 |
| `handle_min_days` | 5 | 3~10 | 必要 | 柄部最少天数（≈1 周） |
| `handle_max_days` | 30 | 15~40 | 必要 | 柄部最多天数（≈6 周） |
| `handle_max_drawdown` | 0.12 | 0.08~0.15 | 必要 | 柄部最大回调 |
| `handle_position_ratio` | 0.50 | 0.40~0.60 | 必要 | 柄部在上半部：低点 ≥ 杯底 + N × 杯高 |
| `handle_vol_ratio` | 0.50 | 0.30~0.70 | 必要 | 柄部量 / 杯身均量上限 |
| `breakout_buffer` | 0.01 | 0.00~0.05 | 必要 | 突破买点缓冲（元） |
| `breakout_vol_ratio` | 1.4 | 1.1~1.8 | 必要 | 突破日量 / 50 日均量下限 |
| `require_green` | true | — | 开关 | 是否要求阳线 |
| `close_position_min` | 0.50 | 0.40~0.70 | 必要 | 收盘在当日区间最小分位 |
| `rs_required` | false | — | 开关 | RS 不达标是否排除 |
| `rs_threshold` | 80 | 70~95 | 必要 | RS 达标阈值 |
| `min_market_cap` | 0 | 0~500 | 必要 | 市值过滤（亿），0=关闭 |
| `fake_breakout_lookback` | 5 | 1~10 | 必要 | 假突破检查：前 N 日无放量下跌 |

## 7. 回测与评估

| 指标 | 目标值 | 说明 |
|---|---|---|
| 信号准确率 | ≥ 60% | 突破后 20 日内涨幅 ≥ 5%（杯柄胜率通常高于标准突破） |
| 平均收益 | ≥ 15% | 突破后 60 日平均最大收益 |
| 盈亏比 | ≥ 3:1 | 平均盈利 / 平均亏损 |
| 误报率 | ≤ 20% | 突破后 5 日内跌回买点以下 |
