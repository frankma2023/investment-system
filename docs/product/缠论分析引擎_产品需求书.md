# 缠论分析引擎 · 产品需求书

> 版本：v2.0 | 日期：2026-05-28 | 状态：P0/P1/P2 核心功能已交付

---

## 1. 产品概述

### 1.1 定位
在现有欧奈尔 CAN SLIM 体系之外，增加缠中说禅（缠论）技术分析引擎，作为**平行分析维度**。两者互补：欧奈尔侧重基本面+动量选股，缠论侧重 K 线形态结构+买卖时机。

### 1.2 核心价值
- 自动识别分型→笔→中枢→背驰→买卖信号，无需人工画线
- 多周期联立（日/周/月同屏），共振评分定位高置信度信号
- 周/月 K 线从日线自动合成，图表密度随周期自然变化
- 与现有 20+ 回测看板共享同一 hanako-glass 设计系统

## 2. 技术架构

### 2.1 依赖
- **CZSC**（waditu/czsc v0.10.12）：Rust 核心的缠论分型/笔计算库
- **ECharts 5.5**：K 线图渲染（已有本地副本）
- **SQLite**：笔/分型/中枢数据持久化（已有 lixinger.db）
- **Pandas**：周/月 K 线合成（`resample`）

### 2.2 数据流
```
SQLite (daily_kline / index_daily_kline)
  → pd.Timestamp RawBar 列表（直接传，不用 .to_pydatetime()）
  → CZSC(bars, freq) 计算分型/笔
  → 自研：compute_zhongshu() 滑动窗口检测中枢
  → 自研：detect_divergence() 趋势背驰 + 盘整背驰
  → 自研：generate_trade_signals() 一买/一卖/三买/三卖
  → 自研：multi_period_analyze() 日/周/月联立 + 共振评分
  → JSON API
  → 前端 ECharts 渲染 + 数据面板
```

### 2.3 文件清单

| 文件 | 作用 |
|------|------|
| `src/scanners/chanlun.py` | 引擎：分析、中枢/背驰/信号、多周期、K线合成、ECharts配置 |
| `web/chanlun-backtest/index.html` | 看板：单周期+多周期双模式、7个数据面板 |
| `src/server.py` | 3个API端点（analyze/echarts/multi-period） |
| 数据库表 `chanlun_bi` | 笔记录（方向、高低点、力度、斜率、角度） |
| 数据库表 `chanlun_fx` | 分型记录（顶/底分型） |
| 数据库表 `chanlun_zs` | 中枢记录（ZG/ZD/ZZ、起止日期、笔数） |

### 2.4 关键设计决策

**dt 传入**：CZSC Rust 核心将 Python `datetime.datetime`（naive）当作 UTC 处理，导致北京时间回退 8h 出现休息日日期。修复为直接传入 `pd.Timestamp`。

**中枢算法**：CZSC 不提供 `zs_list`，自研滑动窗口检测连续 3 笔重叠，含相邻中枢合并延伸逻辑。

**周/月 K 线**：ECharts 渲染时从日线 `resample` 合成（`W-FRI` / `ME`），笔标记通过 `_build_period_index_map` 映射到合成后索引。

## 4. API 端点

| 端点 | 方法 | 参数 | 返回 |
|------|------|------|------|
| `/api/chanlun/analyze` | GET | `code`, `freq`(D/W/M), `limit` | 笔/分型/中枢/背驰/买卖信号/未完成笔 |
| `/api/chanlun/echarts` | GET | `code`, `freq`, `limit`, `theme` | ECharts option（K线+成交量+中枢markArea+ZG/ZD线+买卖scatter） |
| `/api/chanlun/multi-period` | GET | `code`, `limit` | 日/周/月三周期完整分析 + 共振信号 |

### 4.1 自动 limit 放大

为防止周/月线合成后 K 线数量不足，后端自动放大读取量：

| 请求 freq | 前端传 limit | 实际读取日线量 |
|-----------|-------------|---------------|
| D | 400 | max(400, 750) ≈ 750 |
| W | 400 | max(400×3, 900) = 1200 |
| M | 400 | max(400×3, 900) = 1200 |

## 5. 中枢算法

### 5.1 基本检测
```
输入: 笔列表 [BI_0, BI_1, ..., BI_n]
输出: 中枢列表 [{start_dt, end_dt, zg, zd, zz, width}]

for i in 0..n-3:
    ZG = min(BI_i.high, BI_{i+1}.high, BI_{i+2}.high)
    ZD = max(BI_i.low, BI_{i+1}.low, BI_{i+2}.low)
    if ZG > ZD → 构成中枢
```

### 5.2 延伸合并
相邻中枢在日期上连续、价格重叠>50%时合并为一个延伸中枢，ZG取min、ZD取max，日期扩展至最后笔的终点。

## 6. 背驰算法

### 6.1 趋势背驰
```
比较间隔同向笔: BI_i vs BI_{i+2}（中间隔一个反向笔）
power_ratio = BI_{i+2}.power / BI_i.power
if power_ratio < 0.6 → 背驰
  向上笔力度衰减 → 顶背驰
  向下笔力度衰减 → 底背驰
severity: ratio<0.3→强, ratio<0.45→中, else→弱
```

### 6.2 盘整背驰
```
对每个中枢，比较进入笔(首) vs 离开笔(末)
if 同向 and power_ratio < 0.6 → 盘整背驰
```

## 7. 买卖信号逻辑

| 信号 | 触发条件 | 方向 | 置信度来源 |
|------|---------|------|-----------|
| 一买 | 底背驰（下跌力度衰减） | buy | 背驰 severity |
| 一卖 | 顶背驰（上涨力度衰减） | sell | 背驰 severity |
| 三买 | 向上离开中枢后回踩 low>ZG | buy | 高 |
| 三卖 | 向下离开中枢后回抽 high<ZD | sell | 高 |

## 9. 已知限制

1. CZSC Rust 版 `to_echarts()` 返回错误，需手动构建 ECharts option
2. 周/月线 K 线从日线合成（非交易所原始周/月线数据），成交量是日线合计
3. 线段（至少3笔构成）尚未实现，目前用笔直接替代
4. 背驰阈值 0.6 为经验值，硬编码，不同品种可能需调整
5. 仅支持单只股票/指数分析，无全市场批量扫描
6. CZSC 的笔计算在周/月模式下仍基于日线 RawBar（freq 仅标注级别），非真正周/月线分型
