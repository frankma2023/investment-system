# 缠论分析引擎 · 产品需求书

> 版本：v1.0 | 日期：2026-05-28 | 状态：MVP 已上线

---

## 1. 产品概述

### 1.1 定位
在现有欧奈尔 CAN SLIM 体系之外，增加缠中说禅（缠论）技术分析引擎，作为**平行分析维度**。两者互补：欧奈尔侧重基本面+动量选股，缠论侧重 K 线形态结构+买卖时机。

### 1.2 核心价值
- 自动识别分型→笔→中枢，无需人工画线
- 多周期联立（日/周/月），区间套定位买卖点
- 与现有 20+ 回测看板共享同一 hanako-glass 设计系统

---

## 2. 技术架构

### 2.1 依赖
- **CZSC**（waditu/czsc v0.10.12）：Rust 核心的缠论计算库，pip 安装
- **ECharts 5.5**：K 线图渲染（已有）
- **SQLite**：笔/分型数据持久化（已有 lixinger.db）

### 2.2 数据流
```
SQLite (index_daily_kline)
  → RawBar 列表
  → CZSC(bars) Rust 计算
  → 分型 / 笔 / 中枢 / 信号
  → JSON API
  → 前端 ECharts 渲染
```

### 2.3 新增文件
| 文件 | 作用 |
|------|------|
| `src/scanners/chanlun.py` | 引擎：分析、持久化、ECharts 配置生成 |
| `web/chanlun-backtest/index.html` | 看板：K 线图 + 笔列表 + 信号面板 |
| 数据库表 `chanlun_bi` | 笔记录（方向、高低点、力度、斜率） |
| 数据库表 `chanlun_fx` | 分型记录（顶/底分型） |

## 3. 功能清单

### 3.1 已实现（MVP）

| 功能 | 说明 |
|------|------|
| 个股/指数切换 | 支持股票和指数两种模式，默认中证全指 |
| 日/周/月周期 | 按钮组切换，位于 K 线图上方 |
| K 线图 | ECharts candlestick，hanako-glass 配色（深灰底+红涨绿跌） |
| 成交量副图 | 红涨绿跌柱体，匹配 K 线方向 |
| 笔标记 | 顶底分型在 K 线上标记为▼/▲ |
| 统计卡片 | 笔数、分型数、当前方向、笔力度 |
| 笔列表 | 最近 20 笔表格（方向/起止日期/高低点/力度） |
| 信号面板 | CZSC 输出的买卖信号展示 |
| 深浅色切换 | toggleTheme 联动，图表自动重绘 |

## 5. 设计规范

### 5.1 遵循 hanako-glass 设计系统
- 共享 CSS：`web/shared/css/hanako-glass.css`
- 设计令牌：`web/shared/design-model.yaml`
- 导航栏：`web/shared/js/nav.js`（已添加「缠论分析」链接）
- 统计卡片：`.stat-card`（与 stock-valuation 一致）
- 表格：`.mini-table`（与 index-scan 一致）
- 面板：`.xhs-card` + `.xhs-card-header` + `.xhs-card-label`

### 5.2 配色
- K 线背景：`rgba(26,26,31,.6)` 深色 / `rgba(255,255,255,.75)` 浅色
- 轴颜色：`rgba(200,200,200,0.3)` 深色 / `rgba(128,128,128,0.3)` 浅色
- 涨跌：红涨 `#ef4444` / 绿跌 `#10b981`
- Tooltip：玻璃背景 + 细边框，中文标签

## 6. 已知限制

1. CZSC 0.10.12 的 `to_echarts()` 在 Rust 版未实现，需手动构建 ECharts option
2. 周线/月线 K 线数据从日线合成（CZSC 需要对应周期的原始 K 线）
3. 当前仅支持单只股票/指数分析，无批量扫描
4. 中枢渲染尚未实现（需 CZSC 输出中枢坐标）
