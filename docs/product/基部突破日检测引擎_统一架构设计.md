# 基部突破日检测引擎 — 统一架构设计 v1.0

> **化繁为简**：1 个核心引擎（Layer 1）找所有突破日 + N 个特征标注器（Layer 2）标注形态类型 + 用户自主过滤（Layer 3）。

---

## 1. 为什么重构

### 现状：10 个独立引擎，上百个参数，零信号

| 引擎 | 参数量 | 问题 |
|---|---|---|
| 杯柄形态 | 27 | 27 条件串联通过率 ≈ 0.24%，实测零信号 |
| 碟形基部 | 31 | 切割法 + 双层窗口 + 弧线圆润 + 对称性 + 成交量 → 太紧 |
| VCP | 24 | |
| 高紧旗 | 16 | 前置暴涨 ≥ 100% 几乎排除了所有 A 股标的 |
| 箱体 | 8 | |
| 一底更比一底高 | 8 | |
| 递升底部 | 9 | |
| 双重底 | (已有) | |
| 扁平基部 | (已有) | |
| 标准突破日 | 21 | |

**根本问题**：每个形态引擎都把"检测是否是突破日"和"判断是什么形态"混在一起做。形态判断的严格参数把突破日信号也过滤掉了。结果：引擎越多，信号越少，最终归零。

### 新架构：检测与分类分离

```
旧: K线 → 杯柄引擎(27参数) → 杯柄信号(几乎为0)
         → 碟形引擎(31参数) → 碟形信号(几乎为0)
         → VCP引擎(24参数)  → VCP信号(几乎为0)
         ...

新: K线 → Layer1: 核心突破引擎(≤10参数) → 全部突破信号
              │
              ├→ Layer2: 杯柄特征标注器 → 杯柄得分 0.85
              ├→ Layer2: 碟形特征标注器 → 碟形得分 0.30
              ├→ Layer2: VCP特征标注器  → VCP得分 0.65
              └→ Layer2: ...其他标注器...
```

**核心原则**：
1. Layer 1 负责**择时**——只回答"这是不是突破日"。参数少，通过率可控。
2. Layer 2 负责**标注**——回答"这个突破日有什么形态特征"。不参与过滤，只提供信息。
3. Layer 3 由用户**自主过滤**——想只要杯柄？在结果上按杯柄得分筛。

## 2. Layer 1：基部突破日核心引擎

### 设计理念

所有 O'Neil 八种基部形态共享同一个底层结构：**上涨 → 见顶 → 回调/整理 → 反弹 → 突破**。Layer 1 只验证这个底层结构是否存在，不关心具体形状。

### 2.1 核心问题

> 今天，某只股票是否出现了一个**有效的基部突破信号**？

### 2.2 形态结构（最低公分母）

```
前高 ───────────────────────── 突破买点
   \                            /
    \                          /
     \     (任何形态的回调)     /
      \    - 杯柄: 12-33%     /
       \   - 碟形: 8-20%     /
        \  - 箱体: 8-15%     /
         \ - VCP: 逐次收缩   /
          \                  /
           \                /
            谷（最低点）───┘
```

### 2.3 量化计算规则

#### 步骤 1：定位前高和谷

```
1.1 在检测日 t 之前 lookback（默认 120）天内，找到最高收盘价 = 前高

1.2 在前高之后找到最低收盘价 = 谷

    必要条件:
      前高在谷之前（时序上更早）
      前高距今 ≥ min_base_days（默认 20 天）—— 基部至少 4 周
      谷距今 ≥ min_recovery_days（默认 5 天）—— 反弹已确认

1.3 验证回调深度:
      drawdown = (前高 - 谷) / 前高
      要求 ∈ [drawdown_min, drawdown_max]（默认 [0.05, 0.40]）
      → 极宽范围：只要不是噪音波动（<5%）也不是崩盘（>40%）
```

#### 步骤 2：验证回升

```
2.1 谷之后存在反弹:
      谷到检测日之间的最高收盘价 ≥ 前高 × min_recovery（默认 0.75）
      → 回升到前高 75% 以上即可——不要求必须到杯口（90%），那是 Layer2 的事

2.2 回升段线性回归斜率 > 0（确认上升趋势）
```

#### 步骤 3：前置上涨验证

```
3.1 在前高之前 lookback 天内找到最低收盘价 = 前低
    要求 (前高 - 前低) / 前低 ≥ min_prior_advance（默认 0.30）
    → 前高之前有过 ≥30% 的上涨（排除熊市反弹）
```

#### 步骤 4：突破日验证

```
B1: 价格突破
     检测日收盘 ≥ 前高 + breakout_buffer（默认 0.01）

B2: 成交量放大
     检测日成交量 ≥ 50 日均量 × breakout_vol_ratio（默认 1.3）
     注意：50 日均量取 [t-50, t-1]，不含今天

B3: 阳线（可关开关 require_green，默认 true）

B4: 收盘在高位
     收盘位置 ≥ close_position_min（默认 0.50）

B5: 趋势确认
     收盘 > SMA50（可关开关 sma50_check，默认 true）
```

#### 步骤 5：虚假突破排除

```
- 检测日或次日盘中最低价 < 前高（以最低价计）
- 检测日成交量 < 50 日均量
- 检测日前 3 个交易日内存在放量下跌
```

### 2.4 参数配置（≤ 10 个）

| 参数 | 默认值 | 可调范围 | 说明 |
|---|---|---|---|
| `lookback` | 120 | 60~250 | 寻找前高的回溯天数 |
| `min_prior_advance` | 0.30 | 0.20~0.50 | 前置最小涨幅 |
| `drawdown_min` | 0.05 | 0.03~0.12 | 最小回调深度 |
| `drawdown_max` | 0.40 | 0.25~0.50 | 最大回调深度 |
| `min_base_days` | 20 | 10~40 | 前高距今最少天数 |
| `min_recovery_days` | 5 | 3~15 | 谷距今最少天数 |
| `min_recovery` | 0.75 | 0.65~0.90 | 回升到前高比例 |
| `breakout_vol_ratio` | 1.3 | 1.1~1.8 | 突破量比 |
| `require_green` | true | — | 是否要求阳线 |
| `close_position_min` | 0.50 | 0.40~0.70 | 收盘位置 |
| `sma50_check` | true | — | 是否要求收盘>SMA50 |

### 2.5 信号量估算

以 `drawdown_min=0.05, drawdown_max=0.40, min_recovery=0.75` 的宽松默认值：

```
单条件通过率估算:
  - 前高在谷前: 95%
  - 前高距今≥20天: 70%
  - 回调∈[5%,40%]: 85%
  - 回升≥75%: 60%
  - 前置上涨≥30%: 40%
  - 突破量比≥1.3: 50%
  - 阳线: 55%
  - 收盘高位≥50%: 65%
  - 收盘>SMA50: 55%

串联通过率 ≈ 0.95×0.70×0.85×0.60×0.40×0.50×0.55×0.65×0.55 ≈ 1.4%
```

每只股票每年约 250 个交易日，期望信号数 ≈ 250 × 0.014 ≈ **3.5 个/年**。5000 只 A 股 ≈ **17500 个信号/年**。信号量充足，再通过 Layer 2 标注 + Layer 3 过滤来筛选。

### 2.6 信号输出格式

| 字段 | 类型 | 说明 |
|---|---|---|
| `signal_date` | date | 突破日期 |
| `stock_code` | str | 股票代码 |
| `stock_name` | str | 股票名称 |
| `prior_high_date` | date | 前高日期 |
| `prior_high_price` | float | 前高价格 |
| `trough_date` | date | 谷日期 |
| `trough_price` | float | 谷价格 |
| `drawdown_pct` | float | 回调深度 % |
| `recovery_pct` | float | 回升到前高比例 % |
| `base_days` | int | 基部持续天数 |
| `buy_point` | float | 买点 = 前高 + 0.01 |
| `breakout_close` | float | 突破日收盘价 |
| `breakout_vol_ratio` | float | 突破量比 |
| `breakout_chg_pct` | float | 突破日涨幅 % |
| `prior_advance_pct` | float | 前置涨幅 % |
| `tags` | dict | **Layer 2 标注结果** |

## 3. Layer 2：形态特征标注器

### 设计理念

Layer 2 不参与过滤。对每个 Layer 1 信号，每个标注器计算一个 **0~1 的匹配得分**和**关键特征的量化值**。标注器之间相互独立，可以按需启用。

### 3.1 标注器列表

| 标注器 | 计算内容 | 主要得分 | 次要特征 |
|---|---|---|---|
| **杯柄标注器** | 杯身 U 形 + 柄部存在 | `cup_handle_score` | 柄部回撤%、柄部位置 |
| **碟形标注器** | 弧线圆润 + 碟底平坦 | `saucer_score` | 碟底振幅、下行段标准差 |
| **双重底标注器** | W 形双底 + 右底成交量 | `double_bottom_score` | 两底价差、中峰位置 |
| **扁平基部标注器** | 窄幅横盘 + 水平箱体 | `flat_base_score` | 基部振幅、持续天数 |
| **箱体标注器** | 矩形箱体 + 上下沿平行 | `box_score` | 箱体振幅、平行度 |
| **高紧旗标注器** | 前置暴涨 + 极窄旗形 | `htf_score` | 前置涨幅、旗形振幅 |
| **VCP 标注器** | 波动率逐次收缩 | `vcp_score` | 收缩次数、最后振幅 |
| **一底更比一底高** | 前一个基部存在 | `botb_score` | 前基部位置 |
| **递升底部** | 三底逐级抬高 | `ascending_score` | 谷底数、逐次抬高比 |

### 3.2 标注器接口

每个标注器是一个独立函数：

```python
def tag_cup_handle(
    daily: List[Dict],      # K线数据（到检测日为止）
    signal: Dict,           # Layer 1 信号
    params: Dict,           # 该标注器的参数（从 config 加载）
) -> Dict:
    """
    返回:
    {
        "cup_handle_score": 0.85,       # 0~1 综合得分
        "handle_found": True,           # 是否有柄
        "handle_drawdown_pct": 8.2,     # 柄部回调%
        "handle_position_ratio": 0.62,  # 柄部在上半部比例
        "cup_bottom_flatness": 0.07,    # 杯底振幅
        "arc_smoothness_std": 0.006,    # 弧线圆润度
    }
    """
```

### 3.3 标注器输出合并到 Layer 1 信号

```json
{
  "signal_date": "2026-05-17",
  "stock_code": "600519",
  "buy_point": 1850.01,
  "tags": {
    "cup_handle": {
      "score": 0.85,
      "handle_found": true,
      "handle_drawdown_pct": 8.2,
      "cup_bottom_flatness": 0.07
    },
    "saucer": {
      "score": 0.30,
      "bottom_flatness": 0.12
    },
    "vcp": {
      "score": 0.65,
      "contractions": 3,
      "final_amp": 0.04
    }
  }
}
```

### 3.4 标注器复用现有代码

标注器可以直接复用现有引擎的核心检测函数（去掉最后的"是否输出"判断，只返回特征值）：

| 现有引擎 | 复用为标注器的方式 |
|---|---|
| `cup_handle.py` → `_find_handle()` | 返回 handle_found + handle_drawdown |
| `saucer_base.py` → 碟底平坦性 + 弧线圆润性 | 返回 amplitude + std_dev |
| `vcp.py` → 收缩序列识别 | 返回 contraction_count + final_amp |
| `high_tight_flag.py` → 旗形识别 | 返回 flag_amp + surge_pct |

## 4. Layer 3：用户自主过滤

### 设计理念

引擎不再替用户做"是否值得买"的判断。Layer 1 给出信号 + Layer 2 给出标签，用户自己决定过滤条件。

### 前端界面

```
┌─────────────────────────────────────────────┐
│ 📊 基部突破日扫描                            │
│                                              │
│ Layer 1 核心参数（10个）:                     │
│  [回调深度: 5% ~ 40%]  [回升比例: ≥75%]      │
│  [突破量比: ≥1.3]      [趋势: SMA50之上]     │
│                                              │
│ Layer 3 自定义过滤:                          │
│  ☑ 杯柄得分 ≥ 0.7    ☐ 碟形得分 ≥ 0.6       │
│  ☐ VCP得分 ≥ 0.6    ☐ 带柄                   │
│  ☐ 回调深度 12%~25%                          │
│                                              │
│ [🔍 扫描]                                     │
│                                              │
│ 信号列表（17500 → 筛选后 320 条）:             │
│ ┌──────────────────────────────────────────┐ │
│ │ 600519 2026-05-17 回调=15% 杯柄=0.85 ...│ │
│ │ 000858 2026-05-16 回调=22% 杯柄=0.72 ...│ │
│ └──────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

### 实现方式

Layer 3 是纯前端过滤——Layer 1 返回全部信号（一次性），前端按用户选中的条件筛。不需要重跑后端。
