# 欧奈尔每日精选 — 评分引擎产品需求书 v1.0

## 一、目的

为 `src/discipline/screener.py` 六层筛选引擎的第六层（综合评分）制定精确的 O'Neil 得分计算公式，替代当前临时加权逻辑。

## 二、O'Neil 综合得分公式

```
O'Neil = CANSLIM × 0.30 + RPS250 × 0.30 + 形态信号 × 0.25 + 行业共振 × 0.15
```

| 组件 | 满分 | 输入来源 |
|------|------|----------|
| CANSLIM | 30 | `canslim_scores.score`（最新），0-100 线性映射 |
| RPS250 | 30 | `stock_rs_daily.rps_250`，0-100 线性映射 |
| 形态信号 | 25 | 见第三节 |
| 行业共振 | 15-18.75 | 精确命中 ×1.25=18.75 / 模糊命中 ×1.15=17.25 / 无=15 |

## 三、形态信号评分细则（25%）

### 3.1 通过条件

必须满足以下条件之一，否则不进入综合评分：

- 近 20 日内出现 ≥1 个 **`base_breakout`** 信号(src=`base_breakout`)
- 近 20 日内出现 ≥1 个 **`pocket_pivot`** 信号(src=`pocket_pivot`)

> 注意：`cdl_engine`、`talib_engine` 的信号是**补充加分项**，不能单独满足通过条件。

### 3.2 计算公式

```
信号分 = 最佳主信号分 + min(额外基部加分, 20) + min(额外形态加分, 10)
信号分封顶 = 100
O'Neil 贡献 = 信号分 / 100 × 25
```

**最佳主信号分**：所有 `base_breakout` / `pocket_pivot` 信号中，取 `基础分 × 时间衰减` 的最大值。

**基础分**：`base_breakout` = `pocket_pivot` = **70**

**时间衰减**：

| 距今天数 | 衰减因子 |
|----------|----------|
| 0-5 日 | ×1.0 |
| 6-10 日 | ×0.7 |
| 11-20 日 | ×0.4 |
| >20 日 | ×0（不计入） |

**额外基部加分**：除最佳信号外，每个额外的 `base_breakout` / `pocket_pivot`（20 日内）加 **5 分**，封顶 **20 分**。

**额外形态加分**：每个 `cdl` 系列看涨信号 或 `talib` 系列看涨信号（20 日内，`type='bullish'`）加 **2 分**，封顶 **10 分**。

### 3.3 计算示例

| 场景 | 最佳主信号 | 额外基部 | 额外形态 | 信号分 | O'Neil |
|------|-----------|---------|---------|--------|--------|
| 1×5日基部突破 | 70×1.0=70 | 0 | 0 | 70 | 17.5 |
| 1×5日口袋支点 | 70×1.0=70 | 0 | 0 | 70 | 17.5 |
| 基部突破+口袋支点(均5日) | 70 | 5 | 0 | 75 | 18.75 |
| 1×10日口袋支点+3×cdl | 49 | 0 | 6 | 55 | 13.75 |
| 1×15日基部突破+2×talib | 28 | 0 | 4 | 32 | 8.0 |
| 3×基部突破(5/8/12日) | 70 | 5+5=10 | 0 | 80 | 20.0 |

## 四、相关信号源

| 信号源 | 引擎文件 | 信号类型 | 在公式中角色 |
|--------|---------|---------|-------------|
| `base_breakout` | `src/scanners/base_breakout.py` | 基部突破检测 | **主信号**（通过条件 + 评分） |
| `pocket_pivot` | `src/scanners/pocket_pivot.py` | 口袋支点 | **主信号**（通过条件 + 评分） |
| `cdl` / `cdl_*` | `src/scanners/cdl_engine.py` | K线形态看涨 | 补充加分 |
| `talib` / `talib_*` | `src/scanners/talib_engine.py` | TA-Lib 信号看涨 | 补充加分 |
| 其他引擎 | 各 `src/scanners/*.py` | 各异 | **不计入评分**（仅用于一票否决检查） |

## 五、实现约束

1. `buY_SIGNALS` 仅包含 `base_breakout` + `pocket_pivot`
2. 信号时间窗口：标准股 = 20 日；长牛回调股（RPS250≥90 且 RPS20<85）= 10 日
3. 一票否决信号（`top_pattern` / `climax_top` / `breakout_failure` / `distribution_day`）20 日内出现直接淘汰
4. 数据来源：
   - 主信号 + cdl/talib：`run_all_engines(klines, indicators)` 实时计算
   - 引擎不可用时回退 `discipline_observation_pool.signals_json`
5. CANSLIM 评分优先读 `canslim_scores`（`MAX(date)`），无则回退 `discipline_observation_pool`

## 六、验收标准

1. `python src/discipline/screener.py` 输出 TOP 20，得分分布合理
2. 一个 5 日内的基部突破贡献 ≈ 17.5 分（满分 25）
3. 基站突破和口袋支点具有相同的信号分贡献
4. cdl/talib 信号仅作为加分项，不能单独满足 Layer 4 通过条件
