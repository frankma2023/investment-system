## Bug #2（高影响）：成交量均线把当天量算进去了

```python
# 第 103-105 行
vs = [klines[j]['volume'] for j in range(i - 49, i + 1)]  # ← i+1 包含了今天
ma50_vols.append(sum(vs) / len(vs))
```

然后第 123 行：
```python
vol_ratio = today['volume'] / ma50_vols[i]
```

今天的大量同时出现在**分子和分母**里。假设前 49 天日均量 1000 万，今天放量到 2000 万：`ma50 = (49×1000 + 2000)/50 = 1020`，`vol_ratio = 2000/1020 = 1.96`。如果分开算：`vol_ratio = 2000/1000 = 2.0`。看起来差别不大？但如果今天量不大：今天 1100 万，`ma50 = (49×1000+1100)/50 = 1002`，`vol_ratio = 1100/1002 = 1.098`——明明放了 10% 的量，ratio 却几乎没变化。阈值设在 1.4，大量正常的温和放量突破会被漏掉。

**修复：**
```python
vs = [klines[j]['volume'] for j in range(i - 50, i)]  # 不包含今天
```

## Bug #3（高影响）：`min_range_pct` 的公式和你想要的不一致

```python
# 第 161 行
if (hh_val - ll_val) / ll_val < MIN_R:  # MIN_R = 30%
    return 0
```

你注释写的逻辑是「120日内存在峰→谷结构(**涨幅**>30%)」。但 `(peak - trough) / trough` 计算的是"峰比谷高多少"——而谷在峰**之后**，这意味着：**回撤越浅的基部，越难通过这个检查。**

举个例子：

```
股票 A: 30 → 50 (涨幅 +67%) → 回调到 45 → 盘整
        (50 - 45) / 45 = 11%  ❌ 不通过 30% 阈值

股票 B: 30 → 50 (涨幅 +67%) → 暴跌到 30 → 盘整  
        (50 - 30) / 30 = 67%  ✅ 通过
```

股票 A 的回撤只有 10%，形态更健康，但被过滤掉了。股票 B 暴跌回原点，反而通过了。这和欧奈尔的理念完全相反——**好的基部回调浅**。

**修复：** 把 `MIN_R` 的判断改为检查**前置上涨**（峰之前的低点 → 峰），或者改为检查整个窗口的总振幅：

```python
# 方案 1：检查窗口总振幅（峰到谷的距离）
total_range = (hh_val - ll_val) / hh_val  # 定义为回撤百分比
# 这个你已经在 drawdown 那里做了，所以这个条件可以大幅降低或直接删除

# 方案 2：真正查找前置上涨
# 在峰之前的窗口部分找最低点
pre_peak_lows = [(k['low'], j) for j, k in enumerate(window[:hh_idx]) if k['low'] is not None]
if pre_peak_lows:
    pl_val, _ = min(pre_peak_lows, key=lambda x: x[0])
    prior_advance = (hh_val - pl_val) / pl_val
    if prior_advance < MIN_R:
        return 0
```

## Bug #4（中影响）：多条件串联通过率接近 0

把所有条件展开来看（假定 RS 表有数据的前提下，单日通过概率估算）：

| 条件 | 阈值 | 单条件通过率 |
|---|---|---|
| 峰在谷前 | — | ~95% |
| (峰-谷)/谷 ≥ 30% | 30% | ~20% |
| 回撤 12%-33% | [12%, 33%] | ~50% |
| 反弹 ≥ 60% | 60% | ~40% |
| 反弹距今 5-20 天 | [5, 20] | ~60% |
| 峰龄 ≥ 30 天 | 30 | ~50% |
| RS ≥ 87 | 87 | ~15% |
| 量比 ≥ 1.4 | 1.4 | ~50% |
| 阳线 | — | ~55% |
| 收盘位置 ≥ 60% | 60% | ~50% |

**串联通过率 ≈ 0.95 × 0.20 × 0.50 × 0.40 × 0.60 × 0.50 × 0.15 × 0.50 × 0.55 × 0.50 = 0.004%**

每只股票每年约 250 个交易日，期望信号数 = 250 × 0.00004 = **0.01 个/年**。100 只股票才有 1 个信号/年。这还没算 Bug #1 的 RS 表为空的情况。

**这不是 bug，是参数太紧。** 建议先在贵州茅台上用 `--stock 600519` 跑一把，把每个中间条件的结果打印出来，看哪些条件挡掉了最多信号，再调参。
