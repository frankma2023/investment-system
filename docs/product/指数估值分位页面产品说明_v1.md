# 指数估值分位页面产品说明 v1.0

> 最后更新：2026-05-09  
> 数据来源：`index_fundamental_daily` 表 + 理杏仁 `/index/fs/hybrid` 财报 API  
> 页面路径：`web/index-valuation/`

## 1. 产品定位

为用户提供任意指数的 **PE-TTM / PB / 股息率** 历史走势及 **十年分位** 可视化查询。帮助判断当前估值在历史中的相对位置——贵了还是便宜了，辅助择时和仓位决策。

## 2. 页面布局

顶部导航栏 + 搜索选择器 + 三张时序图表。

### 2.1 导航栏

与其他页面统一：🏠 看板 | 回测 ▾ | 指数扫描 | 估值分位（高亮）| 个股财务 | 大盘扫描 | 仓位管理

### 2.2 搜索选择器

- **搜索框**：输入代码或名称实时筛选下拉列表
- **下拉选择器**：408 个指数的 `代码 + 名称`
- **起始年份**：2016 / 2018 / 2020 / 2022 / 2024
- **截止年份**：2023~至今

- 近十年均值、最高值、最低值

## 3. 技术实现

### 3.1 数据流

**估值数据（PE/PB/股息率）**：
```
选择指数 + 年份范围 → GET /api/valuation?code=xxx&start=...&end=...
                     → 查询 index_fundamental_daily 表
                     → 返回 {dates, pe, pe_pct, pb, pb_pct, dyr, dyr_pct}
                     → ECharts 折线图
```

**财务指标数据（ROE/EPS/营收/利润等）**：
```
选择指数 → GET /api/valuation/fs?code=xxx
         → 调用理杏仁 /index/fs/hybrid 实时拉取
         → 返回 {dates, eps, roe, revenue, op_profit, op_margin, net_profit, net_margin, dividend, tax}
         → ECharts 柱状图
```

### 3.2 API 端点

**估值端点**：`GET /api/valuation?code=000300&start=2016-01-01&end=2026-05-07`

返回格式：
```json
{
  "code": "000300",
  "dates": ["2016-01-04", ...],
  "pe": [13.5, ...],
  "pe_pct": [0.45, ...],
  "pb": [1.62, ...],
  "pb_pct": [0.38, ...],
  "dyr": [0.025, ...],
  "dyr_pct": [0.55, ...]
}
```

**财务指标端点**：`GET /api/valuation/fs?code=000300`

返回格式：
```json
{
  "code": "000300",
  "dates": ["2016", "2017", ...],
  "eps": [0.85, ...],
  "roe": [0.12, ...],
  "revenue": [3.2e13, ...],
  "op_profit": [5.8e12, ...],
  "op_margin": [0.15, ...],
  "net_profit": [4.9e12, ...],
  "net_margin": [0.12, ...],
  "dividend": [1.4e12, ...],
  "tax": [2.5e12, ...],
  "peg": [null, ...]
}
```

### 3.3 十年分位说明

十年分位 = 当前估值在过去 10 年所有交易日中的百分位。由理杏仁 API 原生计算（`pe_ttm.y10.mcw.cvpos`），非本地计算。取值 0~1，1=历史最高位，0=历史最低位。

## 4. 使用场景

- 选指数前快速看一眼估值水位，避免在高位接盘
- 对比不同指数的分位，找估值洼地（如全指消费 PE 分位 85% vs 全指医药 PE 分位 30%）
- 季度复盘时查看主要持仓指数的估值变化趋势
- 结合拥挤度指标：高拥挤+高估值=双杀信号
- 结合 RS 强度：低估值+RS 拐头=潜在买点
