> 版本：v1.1

## 1. 产品定义

### 1.1 一句话描述

输入股票或指数代码，自动调用 TA-Lib + 4个自研引擎 + TA-Lib K线形态识别，共6个独立引擎全面扫描，输出K线图与中文分析建议的综合看板。

### 1.2 解决的问题

当前系统有口袋支点、双重底、标准突破、扁平基部4个回测看板，但它们是**原子化的**——用户需要分别进入每个看板、输入代码、查看结果。没有一个地方可以一次性看到"这只股票到底有哪些信号"。

形态识别看板要做的：**一站式形态与K线信号全扫描**。

### 1.3 核心原则

- **引擎平级，互不替代**：每一个引擎各自独立运行、独立输出、标注来源，不存在谁为谁服务
- **建议层是裁判，不是选手**：读取全部信号后，用规则引擎生成中文建议，不修改任何引擎输出
- **引擎代码零侵入**：已开发引擎不做任何修改，对新引擎的唯一要求是实现统一接口契约
- **插件化架构**：引擎通过约定优于配置自动发现。新引擎放入 `src/scanners/` 目录即自动生效，API层、前端层、统计层全部自动适配，零硬编码

## 2. 用户场景

### 2.1 典型工作流

1. 用户在导航栏点击「形态识别」（或输入URL）
2. 页面加载，显示输入区
3. 输入股票代码（如 `600519`），可选调整日期范围
4. 点击「扫描」
5. 页面展示：
   - 顶部：**中文分析建议**（3~5条要点）
   - 中部：**K线图**（含MA均线、BBANDS布林带、6类信号标注点）
   - 下部：**成交量副图**
   - 右侧：**信号时间线面板**（按日期逆序列出全部信号）
6. 用户可点击信号时间线中的某条信号，K线图自动滚动到对应日期

### 2.2 与现有回测看板的关系

| 看板 | 定位 |
|------|------|
| 各类回测看板 | 参数调优 + 历史回测 → 验证单一策略有效性 |
| **形态识别看板（本看板）** | 单只股票/指数的**全引擎实时扫描** → 辅助交易决策 |

回测看板回答"这个策略历史上好不好"，形态识别看板回答"现在这只股票有什么信号"。

### 2.3 与未来引擎的关系

形态识别看板是**引擎消费平台**，不是引擎本身。未来开发的任何新形态识别引擎，只要遵守统一接口契约，放入 `src/scanners/` 目录，就会被自动发现并纳入扫描——无需修改看板任何代码。引擎开发和看板开发彻底解耦。

### 3.2 建议合成层

规则引擎读取全部6个引擎的信号 + TA-Lib指标数据，生成中文分析建议。

**输出格式示例：**

```
【走势判断】价格位于SMA50和SMA200上方，中期趋势向上。BBANDS带宽处于正常范围，未出现极端收缩或扩张。

【信号汇总】近3个月内共检测到4个信号：
  • 2025-03-15 口袋支点（来源：pocket_pivot），突破日成交量放大2.35倍
  • 2025-04-02 标准突破（来源：breakout），突破前期高点42.50
  • 2025-04-20 看涨吞没形态（来源：cdl），位于SMA50附近
  • 2025-04-28 双重底突破确认（来源：double_bottom），颈线突破

【综合评估】
  • 多个买入信号共振，形态层与K线层互相确认
  • RSI 62 未超买，仍有上行空间
  • ATR处于正常水平，波动风险可控
  • 建议：可纳入观察名单，等待回踩SMA50确认后介入
```

**规则引擎逻辑框架：**

```
趋势判断 = f(SMA5 vs SMA20, SMA50 vs SMA200, 价格 vs SMA50)
波动率判断 = f(ATR当前值 vs ATR历史分位, BBANDS带宽)
超买超卖 = f(RSI, 阈值 70/30)
成交量判断 = f(当日成交量 vs VOL_MA50)
信号共振 = f(同日期附近多个引擎是否同时发出信号)
蜡烛图确认 = f(看涨CDL信号是否出现在形态信号附近)
综合建议 = 组装上述判断为自然语言文本
```

### 3.4 插件化引擎架构

#### 3.4.1 设计目标

- **加引擎不改代码**：新引擎文件放入 `src/scanners/` 目录，API层自动发现，前端自动适配
- **引擎和看板解耦**：引擎只关心形态识别逻辑，看板只关心数据展示，中间通过统一接口契约连接
- **零配置**：不维护引擎注册表、不写配置文件、不硬编码引擎列表

#### 3.4.2 实现原理

```
约定优于配置：
  src/scanners/ 目录下的每个 .py 文件，只要实现了 detect() 函数
  并声明了 ENGINE_META 字典，即是合法引擎 → API层遍历发现 → 注入 source → 返回前端

前端：
  从 API 响应中的 engines 字段获取本次扫描使用的全部引擎元信息
  按 meta.category 分组展示，颜色从调色板轮转分配（不硬编码映射）
```

#### 3.4.3 开发者体验

开发一个新引擎（以碟形基部为例），只需要：


#### 3.5.2 信号标注颜色方案（动态分配）

前端不硬编码引擎→颜色映射。改为：从API响应的 `engines` 字段获取引擎列表，按预设调色板轮转分配。

**预设调色板（按 category 分组）：**

| category | 调色板 | symbol范围 |
|---|---|---|
| `pattern`（自研形态） | `#E53935` `#FF9800` `#4CAF50` `#2196F3` `#9C27B0` `#FF5722` `#009688` `#673AB7` … | `triangle` `diamond` `pin` `rect` `arrow` … |

| `candlestick`（K线形态） | 看涨 `#9C27B0` / 看跌 `#607D8B` | `emptyCircle` |

前端伪代码：

```javascript
const palette = ['#E53935','#FF9800','#4CAF50','#2196F3','#9C27B0','#FF5722','#009688','#673AB7'];
const symbols = ['triangle','diamond','pin','rect','arrow','circle','roundRect','emptyCircle'];

const colorMap = {};
engines.forEach((eng, i) => {
    colorMap[eng.name] = {
        color: palette[i % palette.length],
        symbol: `path://${symbols[i % symbols.length]}`,
        size: eng.category === 'pattern' ? 14 : 10
    };
});
```

这样新引擎加入后，前端自动获得颜色和形状，无需修改代码。

## 4. 技术架构

### 4.1 统一引擎接口契约

每个引擎文件必须遵守以下契约。满足即插即用。

#### 4.1.1 ENGINE_META（模块级字典）

```python
ENGINE_META = {
    "name": "double_bottom",         # 唯一标识（字母+下划线）
    "display_name": "双重底",         # 前端展示名
    "category": "pattern",           # pattern / indicator / candlestick / risk
    "version": "1.0",
    "description": "识别W形双重底形态及其颈线突破"
}
```

#### 4.1.2 detect() 函数签名

```python
def detect(klines: pd.DataFrame, indicators: dict) -> list:
    """
    参数:
        klines: 列 [date, open, high, low, close, volume]，按日期升序
        indicators: TA-Lib指标字典（sma50, rsi14, atr14, bb_upper 等）,
                    由框架统一计算后传入，引擎无需自己算指标
    返回:
        List[dict] — 信号列表，每条信号格式如下：
    """
    return [
        {
            "type": "bullish",           # bullish / bearish
            "date": "2025-03-15",        # YYYY-MM-DD
            "close": 42.50,              # 当日收盘价
            "pivot": 41.00,              # 关键价格，无则 null
            "confidence": "high",        # high / medium / low
            "details": { ... }           # 引擎专属字段，自由扩展
        }
    ]
```

**注意：** 引擎返回的信号**不包含 `source` 字段**——该字段由框架在聚合时自动注入（取 `ENGINE_META["name"]`）。引擎开发者无需关心。

### 4.2 TA-Lib 指标判断引擎（`talib_engine.py`）

**职责：** 从技术指标角度输出独立判断信号，不依赖自研引擎。

**计算的指标：**
- `SMA(5, 10, 20, 50, 120, 250)` — 多周期趋势
- `BBANDS(20, 2)` — 布林带
- `ATR(14)` — 波动率
- `RSI(14)` — 相对强弱
- `MACD(12, 26, 9)` — 趋势强度
- `VOL_MA(50)` — 成交量基准

**输出的信号类型：**

| 信号条件 | type | 说明 |
|----------|:---:|------|
| 价格上穿SMA50 / SMA50上穿SMA200 | `bullish` | 金叉/趋势走强 |
| 价格下穿SMA50 / SMA50下穿SMA200 | `bearish` | 死叉/趋势走弱 |
| RSI > 70 | `bearish` | 超买警告 |
| RSI < 30 | `bullish` | 超卖机会 |
| 成交量 > VOL_MA50 × 2 | `bullish` | 放量（配合其他信号看） |
| BBANDS带宽 < 历史10%分位 | 纯数据 | 带宽收窄（不判多空，留给建议层） |
| MACD金叉/死叉 | `bullish`/`bearish` | 趋势变化 |

**调用方式：**

```python
from scanners.talib_engine import scan_talib

signals = scan_talib(df_klines)  # 返回 List[SignalDict]
```

### 4.4 引擎自动发现机制

API层启动时（或首次请求时）扫描 `src/scanners/` 目录，自动发现所有合法引擎。

```python
import importlib
import pkgutil

def discover_engines():
    """
    扫描 src/scanners/ 下所有模块，收集实现了 detect() 且声明了 ENGINE_META 的引擎。
    返回: dict[name] = { 'module', 'meta', 'detect' }
    """
    import src.scanners as pkg
    engines = {}

    for _, name, _ in pkgutil.iter_modules(pkg.__path__):
        # 跳过私有模块和辅助模块
        if name.startswith('_') or name in ('recommend',):
            continue

        mod = importlib.import_module(f'src.scanners.{name}')

        # 检查引擎契约
        if not hasattr(mod, 'detect'):
            continue
        meta = getattr(mod, 'ENGINE_META', {
            'name': name,
            'display_name': name,
            'category': 'misc'
        })

        engines[meta['name']] = {
            'module': mod,
            'meta': meta,
            'detect': mod.detect
        }

    return engines
```

**发现结果缓存：** 引擎列表在进程生命周期内不变，首次发现后缓存，后续请求直接使用。

**已开发引擎适配：** 现有4个引擎（double_bottom / breakout_scanner / pocket_pivot）需要各加一个 `ENGINE_META` 字典，不修改其内部逻辑。后续引擎从创建时就包含。

**调用流程：**

```python
engines = discover_engines()
all_signals = []

for name, engine in engines.items():
    raw_signals = engine['detect'](klines=df, indicators=indicators)
    for sig in raw_signals:
        sig['source'] = name  # 框架自动注入 source
        all_signals.append(sig)

# all_signals 已包含所有引擎的输出，source 字段已填充
```

### 4.6 建议合成层（`recommend.py`）

**输入：**
- 全部 `signals[]`（含6个来源）
- TA-Lib 指标数组（SMA/RSI/ATR/BBANDS/MACD）

**输出：**
- 结构化文本对象，包含 `trend` / `signals_summary` / `assessment` 三个段落

**规则概要：**

```
trend:
  if 价格 > SMA50 > SMA200 → "中期上升趋势"
  if 价格在SMA50附近震荡 → "区间整理"
  if 价格 < SMA50 < SMA200 → "中期下降趋势"
  if SMA50上穿SMA200近5日 → "金叉信号，趋势可能转强"

signals_summary:
  按时间逆序列出6个月内全部信号，标注来源和多信号共振

assessment:
  if RSI > 70 → "超买区域，注意回调风险"
  if RSI < 30 → "超卖区域，可能存在反弹机会"
  if 看涨信号 ≥ 3 AND 无看跌信号 → "多信号共振，形态层与K线层互相确认"
  if 看涨信号附近有看跌CDL → "存在矛盾信号，建议等待更明确确认"
  if BBANDS带宽 < 历史10%分位 → "布林带收窄，可能酝酿大幅波动"
  综合以上 → 输出一段建议文本
```
