# 欧奈尔 CAN SLIM A 股量化投资系统：从零到一的工程实践

> 一篇写给自己的项目总结，也是一份完整的功能地图。

---

## 缘起

每一笔亏损的交易背后，都有一个没有被系统捕捉到的信号。

2025 年初，我开始将威廉·欧奈尔的 CAN SLIM 投资体系系统化。不是写几篇笔记、画几个流程图，而是真正把每一条规则翻译成可执行的代码，跑在真实的 A 股数据上。一年后回头看，这个系统已经有了 70+ 个 API 端点、40 多个功能模块、超过 50 张数据库表，每天自动更新近 5000 只股票和 400 多个指数的全量数据。

这篇文章记录它是什么、能做什么、以及为什么这样设计。

---

## 一、核心理念：把投资纪律交给机器

欧奈尔体系本质上是一套**规则驱动的投资纪律**。它在每个环节都给出了明确的量化标准：什么时候买（突破买点 + 成交量验证），买什么（CAN SLIM 七维评分），什么时候卖（7% 止损、20% 止盈、形态见顶、基本面恶化、大盘环境转向）。

这些规则如果只靠人脑记忆和手动翻看 K 线图，不仅低效，而且容易在执行时被情绪干扰。把纪律交给机器的意义在于：**你在深夜刷到的恐慌消息不会影响它的判断，你在连续盈利后的过度自信也不会让它放宽标准**。

但这并不意味着机器替代人做决策。我们的设计原则一直是：**引擎负责检测和打分，人负责最终判断**。引擎不说「你应该买」，它说「这里出现了突破信号，RS 评分 85，基本面恶化 0 项红灯，大盘健康度 72」。决策权始终在你手里。

## 二、技术架构：一个 Python 单体 + SQLite 的务实选择

系统采用 Python Flask 作为 API 服务，SQLite 作为数据库，前端用原生 JavaScript + ECharts。这个技术选型看起来「朴素」，但它是基于一个现实约束做出的最优解：**这是一个个人投资者在 Windows 笔记本上运行的本地系统，不需要分布式、不需要微服务、不需要云数据库**。

### 2.1 数据层

数据源是[理杏仁](https://www.lixinger.com)开放 API，覆盖 A 股全量股票和指数的日 K 线、财务数据、估值指标、机构持股、融资融券等 30 多类数据。通过 `scripts/` 目录下的 20 多个采集脚本，每天自动拉取增量数据存入 SQLite。

数据库目前约 20GB，核心表包括：

| 表 | 行数 | 用途 |
|---|---|---|
| `daily_kline` | 1846 万 | 个股日 K 线 |
| `index_daily_kline` | — | 指数日 K 线 |
| `fundamental_indicator` | 2.7 亿 | 估值指标（PE/PB/PS 等） |
| `stock_financials_quarterly` | — | 季度财报 |
| `stock_financials_annual` | — | 年度财报 |
| `stock_rs_daily` | — | 个股 RS 强度日数据 |
| `index_rs_daily` | — | 指数 RS 强度日数据 |
| `chanlun_bi` / `chanlun_fx` / `chanlun_zs` | — | 缠论笔、分型、中枢持久化 |

还有一个 `engine_registry.py` 模块，实现了**引擎自动发现机制**：`src/scanners/` 目录下任何实现了 `detect()` 函数且声明了 `ENGINE_META` 字典的 Python 文件，都会被自动识别并纳入形态扫描体系，无需手动注册。

### 2.2 服务层

两个本地服务：

- **Flask API**（端口 8788）：74 个 REST 端点，覆盖指数分析、个股扫描、形态识别、回测、基本面检测、缠论分析、知行系统等
- **静态文件服务**（端口 8772）：Python `http.server` 直接挂载 `web/` 目录

### 2.3 前端层

纯原生 HTML/CSS/JavaScript，全站 41 个页面统一使用 **hanako-glass 玻璃拟态设计系统**——深色炭黑底 + 玻璃模糊卡片 + 金色强调色，深浅双模式一键切换。核心基础设施全部共享：

- `hanako-glass.css`：全站唯一样式真相源，30+ CSS 变量，深度适配深浅色
- `nav.js`：全站 41 页面统一导航栏，增删页面只需改一处数组
- `kline-chart.js`：通用 K 线图组件，一行引入一行渲染
- `components.css`：通用 UI 组件样式（参数卡片、滑块、开关、数据表格）
- `template-backtest.html`：回测看板模板，复制 → 改 8 处标记 → 完成

全站 CSS 铁律：`hanako-glass.css` 是唯一真相源，页面级 CSS 只为特有组件服务，不覆盖全局布局（`.app-container`、导航栏、卡片变量）。

## 三、大盘环境：判断能不能买的第一道闸门

欧奈尔体系最重要的原则之一：**75% 的个股走势与大盘方向一致**。因此，系统首先判断大盘环境。

### 3.1 抛盘日与追盘日

抛盘日（Distribution Day）是机构派发行为的量化指标。系统对沪深 300、中证全指等关键指数实时追踪抛盘日数量——当 25 个交易日内累积 5 个以上抛盘日时发出警告，6 个以上直接触发清仓信号。

追盘日（Follow-Through Day）是下降趋势转为上升趋势的确认信号。系统在指数暴跌后的第 4~7 天检测放量反弹，确认新一轮上升趋势的开始。

这两个指标不仅是独立的检测引擎，还配有完整的回测看板——你可以调整参数（如抛盘日跌幅阈值、成交量倍数）来观察不同参数组合的历史表现。

### 3.2 大盘健康度评分

除了抛盘日和追盘日，系统还综合了 MA50 上方占比、新高新低比、涨跌家数比、融资余额变化等多项指标，输出一个 0~100 的大盘健康度评分。这个分数不是用来做精确择时的，而是帮你回答一个简单问题：**现在适合重仓吗？**

### 3.3 大盘卖出评分

「什么时候该减仓」和「能不能买」同样重要。大盘卖出评分从 12 个维度综合计算（满分 400），包括：

- **抛盘日累积**：过去 25 个交易日抛盘日数量（4个=10分，5个=20分，≥6个=40分），指数处高位时分数翻倍
- **追盘日失效**：追盘日后 10 日内跌破追盘日最低价 → +30 分
- **龙头股见顶**：市场领头羊开始大幅回调 → +25 分
- **垃圾股补涨**：低价小盘股加速上涨 → +25 分
- **指数背离**：指数创新高但指标背驰 → +20 分，多指数共振最高 60 分
- **均线破位**：跌破 50 日线 5 日未收复（+30 分）、跌破 200 日线（+50 分）、MA50 下穿 MA200（+50 分）
- **AD/NHNL 比率**：AD 比率持续恶化（+20~40 分）、NH/NL 比率恶化（+30 分）
- **上涨无量**：连续上涨但成交量萎缩（+20 分）

熔断规则：任意一个「清仓级」信号触发，仓位上限 ≤ 25%；两个叠加强制清仓。

分数实时展示在 `market-scan` 大盘健康度看板上，和健康度评分卡并列——一个告诉你「能不能买」，一个告诉你「该不该卖」。

## 四、指数扫描：找到资金流向的方向

A 股有超过 400 个指数——从沪深 300 到半导体、从白酒到新能源。在 5000 只个股中选股之前，先要看资金在往哪个板块流。

### 4.1 指数 RS 相对强度

模仿欧奈尔的个股 RS 算法，系统对 29 个全市场指数、11 个一级行业、45 个二级行业、229 个主题指数、93 个策略指数分别计算 RS_20、RS_60、RS_120、RS_250 四个周期的相对强度。

然后自动过滤出三个层级的强势池：

- **L1 绝对强势池**：RS_120 ≥ 90 且 RS_250 ≥ 85 且 RS_60 ≥ 80——长中短期全面领先
- **L2 短期爆发池**：RS_20 ≥ 95——近期动量最强
- **L3 潜在共振池**：RS_60 ≥ 85 且 RS_120 ≥ 80 但 RS_20 < 90——中长强势但短期回调

每个池子都可独立开关、独立调整阈值。

### 4.2 指数拥挤度

一个指数被所有人看好时，恰恰是最危险的时候。系统从交易热度、资金流向、估值水位、交易结构、机构行为五个维度计算复合拥挤度得分，当拥挤度过高时提示风险。

### 4.3 机构吸筹/出货评级

基于日级别资金流量和累积 A/D 线，对每个指数给出 A~E 的机构行为评级。A 表示机构在积极吸筹，E 表示机构在出货。

## 五、个股基本面：不只看数字，更看趋势

CAN SLIM 的 C 和 A 两项要求当期和年度盈利强劲增长。但「增长」不是一个数字，而是一条曲线。

### 5.1 个股全维度分析

在 `stock-valuation` 页面，输入任意一只 A 股代码，你可以看到：

- **估值仪表**：PE/PB/PS/股息率/市值的十年历史分位
- **盈利质量**：ROE、EPS、净利增速、营收增速、毛利率、自由现金流的年度趋势
- **财务健康**：资产负债率、有息负债率、负债结构、流动比率、速动比率
- **市场强度**：RS 评级、双强股判定、机构持股变化
- **财务评估**：DCF 估值 + 可比公司分析 + 三表联动模型

每个图表都是一张柱状图（或折线图），下方有自动生成的文字分析，告诉你这个指标的趋势、当前值与历史均值的关系、以及需要注意的异常波动。

### 5.2 基本面恶化检测

这是最新完成的核心模块。它不是看 K 线形态，而是读财报数据——当一家公司的 EPS 增速连续两个季度同比下降、营收同步放缓、净利率和毛利率连续下滑、ROE 跌破 17% 时，无论技术面多好看，都该认真考虑卖出。

系统对每只股票做 6 项检查（5 项参与计分，1 项仅展示），3 项以上红灯直接建议卖出。

### 5.3 CAN SLIM 评分

七个维度（C/A/N/S/L/I/M）加权评分 v3.0，满分 100。每天自动计算全市场 5300+ 只股票的分数，在看板上按分数排序。配有独立的评分卡看板和全市场评分查询页面。

## 六、缠论分析：欧奈尔体系的平行技术维度

在 CAN SLIM 基本面+动量的框架之外，系统引入了**缠中说禅（缠论）技术分析引擎**作为独立的平行分析维度。两者互补：欧奈尔解决「买什么」和「什么时候该卖」，缠论解决「什么时候买在精确位置」。

### 6.1 核心计算链路

缠论的分析层次是一个严格的递推关系：

```
K线 → 分型 → 笔 → 中枢 → 背驰 → 买卖信号
```

- **分型与笔**：基于 CZSC（Rust 实现的高性能缠论库）自动计算，500 根日线可生成约 27 笔、91 个分型
- **中枢**：自研滑动窗口算法，检测连续三笔重叠区间，含延伸合并逻辑
- **背驰**：趋势背驰（间隔同向笔力度衰减 < 0.6）+ 盘整背驰（中枢内力度衰减）
- **买卖信号**：一买/一卖（背驰驱动）+ 三买/三卖（中枢突破确认）

### 6.2 多周期联立

日/周/月三周期并行分析，自动检测跨周期信号共振。三周期笔方向一致、中枢重叠、背驰信号共振、买卖点共振——四项指标综合评分，输出「强/中/弱」三级共振强度。

周线和月线 K 线图从日线自动合成（`resample`），而非简单复用日线——这根周 K 线的开盘价是周一开盘、收盘价是周五收盘，和你在任何软件上看到的周线完全一致。

### 6.3 可视化

K 线图上同时展示：中枢矩形（金色半透明框）+ ZG/ZD 水平虚线 + 笔顶底标记（红箭头）+ 买卖点标记（红买绿卖）。数据面板包含笔列表、中枢列表、买卖信号详情卡片、背驰信号详情卡片、多周期共振面板。

## 七、买入形态：从八种基部到一个统一引擎

### 7.1 Layer 1 核心突破引擎

欧奈尔定义了八种经典基部形态：杯柄、碟形、双重底、扁平基部、高紧旗、一底比一底高、递升底部、箱体。最初系统为每种形态写了独立引擎——参数越来越精细，信号却越来越少，最终几乎归零。

重新审视后，我们发现所有八种形态共享同一个底层结构：**上涨 → 前高 → 回调/整理 → 谷 → 回升 → 突破**。区别只在于回调的形状、深度、谷的形态——这些不改变「是否突破」的判断。

于是我们把「检测突破」和「判断形态」分离：

- **Layer 1**：核心突破引擎（10 个参数），只回答「今天是不是一个有效的基部突破日」
- **Layer 2**：形态标注器（9 个），对每个突破信号给出 0~1 的形态匹配得分
- **Layer 3**：用户自主按形态得分过滤

这个架构让信号量从原本的几乎为零恢复到每只股票每年约 3.5 个，同时保留了形态信息。

### 7.2 形态识别看板

在 `pattern-scan` 页面，输入股票代码和日期范围，系统同时运行所有已注册的形态引擎（包括基部突破、口袋支点、双重底、扁平基部、头部形态、铁轨线、高潮见顶、量价背离等），在 K 线图上标注所有信号点，右侧显示信号时间线。是买入信号还是卖出信号一目了然。

### 7.3 全市场形态扫描

`daily-pattern-scan` 页面内嵌 4000+ 只股票的全量扫描结果（RS 强度 + 信号摘要），支持客户端分页、筛选（双强/稳健龙头/加速爆发/有信号/看涨/看跌/标准突破/口袋支点/双重底/扁平基部）、列排序。一次生成 HTML，无需实时查 API，打开即用。

## 八、卖出信号：比买入更重要的能力

「会买的是徒弟，会卖的才是师傅」。系统的卖出信号分为三大类：

### 8.1 见顶信号

- **高潮见顶**：连续急涨后出现放量长上影线或高位滞涨，是机构在高位派发
- **铁轨线**：两根几乎等高的放量 K 线，表明多空在关键价位激烈博弈后空方获胜
- **头部形态**：头肩顶、双重顶、三重顶的识别

### 8.2 突破失败

基于 Layer 1 核心突破引擎的输出，系统对每一个突破信号进行 10 日监控。7 条规则（跌回枢轴点、跌破突破日最低价、跌破谷底支撑、放量长阴砸穿、连续放量溃败、抛盘日叠加、窗口结束机构出货）独立触发，三级严重级别（mild/confirmed/severe）。每个规则可独立开关、独立调整参数。

### 8.3 基本面恶化

如第五节所述，不属于形态引擎体系，而是独立的财务分析模块。

## 九、回测体系：用历史数据验证你的参数

系统的每一个检测引擎都配有独立的回测看板，采用统一的「左参数卡 + 右信号表」布局。你可以在左侧调整参数（滑块、开关），点击「回测」后右侧显示在选定时段内触发的所有信号明细，下方还有 K 线图标注每个信号点。

更重要的是，**所有参数都可以保存到 YAML 配置文件**，下次打开页面自动恢复。整个回测到实盘的参数流通路径是：

```
YAML 配置文件 ↔ API ↔ 前端控件 ↔ 检测引擎
```

目前有 21 个回测看板，覆盖抛盘日、追盘日、吸筹日、指数 RS、拥挤度、机构吸筹出货、指数背离、个股 RS、杯柄、碟形、基部突破、口袋支点、高潮见顶、铁轨线、头部形态、量价背离、突破失败、股票精选回测、指数精选回测等。

## 十、知行系统：从分析到执行的最后一公里

分析得再好，不执行也等于零。「知行系统」是整个投资系统的执行层，负责每日精选、观察池、自选池、交易记录、持仓监控、买入前检查清单和回测验证。

### 10.1 每日精选

系统每天自动运行六层筛选引擎，从全市场 5000+ 只股票中筛选出满足 CAN SLIM 标准的高质量标的。筛选结果自动存入快照表，次日访问秒出。

### 10.2 精选回测

精选结果不是「选了就完了」——股票精选回测和指数精选回测两个看板允许你回溯任意历史日期的精选快照，查看当时入选的股票/指数在 5 日、10 日、20 日后的实际表现。这是整个系统中最诚实的检验：**如果精选池的历史表现不好，说明筛选标准需要调整**。

指数精选回测还支持点选任意指数查看六层得分拆解（L1 大盘闸门 → L2 RS 强度 → L3 趋势健康 → L4 技术信号 → 最终得分）。

### 10.3 观察池 → 自选池 → 交易

系统自动从全市场筛选出值得关注的股票进入「观察池」。用户从观察池中选择高度关注的加入「自选池」，也可以手动添加。当决定买入时，系统会运行买入前 6 项规则检查（市场阶段是否合适、RS 是否达标、是否在买点附近、仓位是否超限、止损位是否合理、是否有明确买入理由），全部通过方可录入交易。

### 10.4 持仓监控

已经持有的股票，系统每天自动刷新现价、计算盈亏、监控是否触发卖出信号（如跌破止损价、基本面恶化、大盘环境转向等）。告警灯分为绿/黄/红三级。

## 十一、工程哲学：几个贯穿始终的设计原则

### 11.1 检测与分类分离

从 Layer 1 基部突破引擎到突破失败引擎，从基本面恶化检测到缠论中枢与背驰，系统中凡是「检测某个现象是否存在」和「判断这个现象属于什么类型」的场景，都遵循**检测与分类分离**的原则。检测负责择时（不漏报），分类负责信息（不误报）。两者解耦后，用户可以根据自己的风险偏好自由组合。

### 11.2 参数即标准，回测来校准

系统中每个引擎的阈值都不是「拍脑袋写死的」。它们默认有一个合理的初始值，但所有参数都暴露在 YAML 配置文件和回测看板中。你可以收紧参数让信号更精准（牺牲数量），也可以放宽让信号更敏感（牺牲质量）。哪种更好？回测说了算。

### 11.3 共享基础设施，消灭重复代码

从 26 个页面各自手写导航栏，到 `nav.js` 一改全改。从每个页面复制粘贴 K 线图代码，到 `kline-chart.js` 一行引入。从满天飞的 `<style>` 块，到 `hanako-glass.css` 统一管理——CSS 是唯一真相源，页面不覆盖全局布局。从 42 个页面加载但无人使用 Tailwind CDN 到全站清理。每消除一次重复，就少一个未来会忘记同步的地方。

### 11.4 先对齐再动手

这是踩了无数次坑之后沉淀下来的工作习惯。产品需求书不是「写出来就完了」，而是在 grilling session 中逐条追问——「连续下滑的具体含义是什么」「环比还是同比」「数据不足时怎么办」——把所有边界条件澄清后再写代码。

### 11.5 数据铁律

系统中有一条不可违反的铁律：**严禁使用任何兜底、估算或默认值进行财务分析**。所有缺失数据必须通过理杏仁 API 拉取真实数据。DCF、Comps 等模块中如果出现简化假设，那只是临时占位，必须逐一替换为真实数据。

## 后记

刚开始写第一行代码时，我并不知道这个系统会变成今天这个样子。它不是按照一份完整的蓝图「自上而下」建造的，而是在使用中不断发现问题、不断重构、不断抽象的过程中「自下而上」生长出来的。

从最初一个跑得通但零信号的手写杯柄引擎，到 Layer 1/2/3 三层架构的自动发现体系；从每个页面复制粘贴的 K 线图代码，到一行引入的精调组件；从手忙脚乱的「这个页面更新了那个忘了」，到 `nav.js` 一改全改；从头疼「到底该不该卖」到基本面恶化、突破失败、缠论背驰三大卖出维度——每一步都在解决一个真实而具体的问题。

风格和审美也不断进化。从 Bloomberg 终端风到 NEO 全息风，从小红书手账到 hanako-glass 玻璃拟态——现在全站 41 个页面统一在同一个设计系统下，深浅双模一键切换。每一个回测看板的布局、每一个参数滑块的颜色、每一张 K 线图的均线配色，都经过反复精调。

这个系统不会完美，也不需要完美。它只需要比昨天更准确、更完整、更易用。它是我作为一个个人投资者的脚手架，也是我与市场对话的工具。希望它也能帮助到你。
