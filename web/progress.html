📋 本次开发经验教训
1. API 返回 JSON 时前端不要用 YAML 解析
load_config 返回 dict，前端却检查 data.ok && data.yaml 等不到，白费半天排查。
2. 预计算存表，多页面共享
index_rs_daily 每天算一次，最强指数和指数强度两页面都读它。避免重复计算和结果分歧。
3. 先对齐算法再写代码
RS 公式 (*99 vs *100)、AD 斜率 (百分比 vs 首尾对比) 反复改了 4 轮才对齐。动手前先读产品文档中的公式。
4. YAML 文件名与代码中 signal_type 严格一致
accumulation vs accumulation_day、theme vs thematic，名不对读不到。
5. 大池子 SQL 占位符不要嵌套子查询
229 个占位符导致主题池静默失败。拆成两步：先取最新日期，再按日期查。
6. 引擎 CLI 打印了结果 ≠ 写入了数据库
stock_rs.py 控制台打印双强股票但没 INSERT，快照读到全是 0。
7. RS 值不能一查到底贴到所有信号上
get_rs 只取最新快照，所有历史信号共用同一个 RS 值。需逐日查库。
8. 前低搜索窗口不设限会追溯到上市首日
双重底前低搜索了全历史数据，紫金矿业回溯到 2014 年。加 200 天窗口限制。
9. YAML 和引擎默认参数必须完整同步
YAML 缺键→前端 gp() 传不全→API merge 后缺参数→引擎 KeyError。
10. 回测页面需要统一的开发标准
口袋支点 v2 成为事实标准后写了 docs/dev/回测看板开发标准.md，后续页面直接套模板。
11. 排查工具必须覆盖引擎所有条件
扁平基部排查漏了斜率、前涨幅、突破条件，导致"全通过但不检出"的困惑。排查项=引擎条件全集。
12. 前低搜索窗口要足够大
默认 200 天不够——前高恰好在窗口边界时前低被截断。扩到 400 天。诊断工具和引擎要同步。
13. 连续突破信号需去重
同一基部突破后连日收盘都在高点之上，每天都被标为突破。加"昨日收盘<基部高点"约束只保留首日。
14. 数组截取方向错误导致全指标偏移
_sanitize_indicators 用 arr[:target_len] 取头部而非 arr[-target_len:] 取尾部，所有 BB/SMA/RSI 值偏移 600+ 天。需用外部软件交叉验证数值。
15. BB 百分位比较方向反转是系统性陷阱
talib_engine 和 flat_base 两处同时犯了 h>current 当 h
16. 插件化架构的核心是函数签名兼容
用 inspect.signature 动态适配不同引擎的 detect() 参数差异（indicators/params/rs_info），避免改旧代码。约定优于配置。
17. conda 环境与系统 Python 的包隔离
daily_update 用 sys.executable 引入 conda 环境但 talib 装在系统 Python 下。固定 PYTHON_EXE 路径避坑。
18. 全市场扫描需控制 HTML 体积
4333只股票的39K行HTML用客户端分页+筛选处理，后端只生成一次。JSON 存档、HTML 是分发格式，不实时查API。
19. 表格对齐不要用 table-layout:fixed
fixed + 固定 width 导致列头与内容错位。让浏览器 auto 分配列宽，th/td 统一 text-align:center，仅 th 加 nowrap。数字用 tabular-nums 等宽。参考 canslim-scores 的实现。
20. 理杏仁API proportion 单位不统一
fund-shareholders 的 proportionOfOutstandingSharesA 对大盘股返小数（0.0115）、小盘股返百分比（68.0）。入库前统一归一化：>1.0 除以100。
21. 双源数据取 max 不叠加
机构持股理杏仁+东方财富数据来源同一季报，占比取 max(props) 防止双重计数。最终砍掉东方财富，只保留理杏仁三API。
22. INSERT OR REPLACE 断送历史对比
机构持股每次 INSERT OR REPLACE 覆盖上季数据，机构数量变化无法计算。需保留多季数据（PK含report_date），或独立回补脚本拉8季历史。
23. YAML 被 JSON 覆盖
前端 saveConfig 发 JSON 到 /api/config，直接覆写 YAML 文件导致 yaml.safe_load 失败。修复：API 层检测 JSON 自动转 YAML，前端不包 wrapper。
24. CAGR 负数底数导致复数
3年EPS CAGR 公式 (latest/old)**(1/3)，old 为负时产生复数 → score_a 崩溃。加 old>0 and lat>0 前置守卫。
25. 评分维度子项必须设 fallback
inst_change/excess/buyback 等子项在数据缺失或负值时漏设 bd key，前端显示空白。所有子项必须有 else 分支显式设 0 或 fallback 值。
26. overflow-x:auto 会暗中强制 overflow-y:auto
CSS 规范：一个轴的 overflow 设为非 visible 时，另一轴未显式设置则强制变为 auto。.nav-links 设了 overflow-x:auto 导致 overflow-y 也变 auto，将绝对定位的下拉菜单裁剪掉。导航栏父容器不要设 overflow。
27. href="#" 点击跳顶 + 多个Python进程占端口
导航栏占位链接用 javascript:void(0) 而非 #。端口8788累积多个僵尸Flask进程导致旧代码一直运行，需 taskkill /F /IM python.exe 全杀。
28. 日期选择触发 run() + 周期切换有竞态条件
4个页面同时出现：改起始日期→图表不更新、切周期无效。根因：runId 计数器 + await 后检查 myId，旧请求结果直接丢弃。
29. setTimeout(run) 自动渲染会吃掉按钮文字
按钮初始"🔍 回测"在自动加载后被"🔄"覆盖。修复：给 run() 加 auto 参数，自动模式不改 btn.textContent。
30. KlineChart.update() 在 chart 被 dispose 后静默失效
主题切换时 chart.dispose()，后续 update 不走 _render。修复：update 内部检测 isDisposed 时自动 re-init。同时 axisPointer.link 从 [{xAxisIndex:'all'}] 改为 {xAxisIndex:'all'}。
31. 后端硬截断日线数据导致历史回测失效
result['daily'][-600:] 硬截到 600 条。选 2021 年起始只看到 2023 年。改为按起止日期动态过滤 + 2000 条安全帽（~10年才触发）。
32. 回测看板标准化模板一劳永逸
制作 shared/template-backtest.html，固定 4CSS+6JS+通用函数。新页面复制→改8处🔧标记→完成。nav.js 统一管理导航，新增页面只需加一行到 BACKTEST_ITEMS。chart-legend 迁入 components.css。
33. nav.js 共享导航栏 — 一改全改
26个页面导航栏全部替换为 `
` + `Nav.init({...})`。增删页面只需改 shared/js/nav.js 中的 BACKTEST_ITEMS/MAIN_ITEMS 数组，所有页面自动同步。
34. YAML嵌套键 vs 引擎平铺读取 — 静默用默认值
前端 saveCfg 存 `composite: {threshold: 75}`（嵌套），引擎 load_params 平铺出 `volume_divergence_composite_threshold=75`。引擎读 `params.get('composite',{}).get('threshold',40)` 找不到→用默认40。修复：load_params 先解包顶层键，composite 保持嵌套。
35. catch(e){} 静默吞错 + svp 函数未定义
loadCfg 中 catch(e){} 吞掉 ReferenceError:svp is not defined，整段参数恢复静默失败。前端辅助函数(svp/svn/svs/svs2)必须全部定义后再用。catch 块至少加 console.error。
36. 突破失败检测架构：统一引擎 + 三级严重级别 + 7条独立规则
突破失败无需按8种基部形态分拆——和Layer 1统一突破检测同理，所有基部形态的突破失败共享同一个骨架：突破后N日收盘<关键价位+成交量验证。7条规则(B~H)每条可独立开关，三级严重级别(mild/confirmed/severe)，每次触发都独立产出信号。诊断端点用 need/actual 对比格式一目了然。K线图用五角星(SVG path)三级颜色，位于收盘价下方 range*0.75 偏移堆叠。回测看板三参数卡+信号表(11列)+单日排查(方案B:突破日+检查日双选)。
37. KlineChart.create() 只建容器不渲染，首次加载空白图表
KlineChart 构造函数不调 _render()，原代码把 kc.update() 放在 else 分支，导致首次 create 后无数据渲染。6个页面(breakout-failure/volume-divergence/climax-top/railroad-tracks/top-pattern/template)全部中招。修复：create 后无条件调 update()，消除 if-else 分支。
38. ROE检测必须用年度数据而非季度，避免季节性Q1陷阱
基本面恶化引擎最初用季度roe_single做ROE判定，宁德时代Q1 ROE=5.94%被误判为红灯（<12%阈值）。但制造业Q1天然是淡季，ROE全年最低。改为年度ROE（stock_financials_annual）后，宁德时代23.83%正确显示为绿色。教训：财务比率判读必须考虑行业季节性，同比跨年对比而非环比。阈值从12%调至17%。
39. sqlite3.Row 不支持 .get()，必须用索引访问
screener.py 多次出现 `row.get('field')` → AttributeError。sqlite3.Row 是只读映射对象，只支持 `row['field']` 索引。同样 `dict()` 包装可安全转换。教训：所有数据库读取若传给字典操作，先 `dict(row)` 转换。
40. run_all_engines indicators=None 导致 8 个引擎静默崩溃
canslim_score.py 和 screener.py 多处调用 run_all_engines(klines=klines, indicators=None)。引擎内部访问 indicators['sma5'] 遇到 NoneType 直接崩溃，被 try/except 静默吞掉，信号检测不全。修复：每个调用点先 compute_indicators(klines) 计算 SMA/BB/ATR 再传入。教训：框架函数的可选参数 ≠ 可用默认值——需确认下游是否真的容忍 None。
41. .pyc 缓存导致修改后代码不生效
trades_api.py 修改后 Flask 仍跑旧代码，报错行号与实际文件不符。`__pycache__` 中的 .pyc 文件比 .py 源文件更新时，Python 优先加载缓存。修复：每次代码部署后执行 `del /s /q __pycache__` 清除所有缓存。教训：代码修改后看不到效果，第一反应应该是清除缓存而非怀疑逻辑。
42. K线图信号颜色/样式必须与 pattern-scan 完全一致
review-index.html 最初用了自创的颜色方案（PALETTE 轮询），导致指数复核页与股票复核页信号标记颜色完全不同。用户要求必须"完全保持一致"。修复：将 ENGINE_STYLES、getSignalStyle、maColors、tooltip 格式、信号位置（蜡烛下方而非上方）、dataZoom 滑块全部从 review.html 逐行复制。教训：复用已有页面的可视化组件时，不要重新发明颜色/样式——直接复制源文件的完整渲染逻辑。
43. base_breakout 引擎的 buy_point ≠ 信号日收盘价
base_breakout 返回 buy_point = prior_high + 0.01（基部前高突破价），而非信号日的 close。301377 买点 137.71 vs 实际收盘 253.68 差 85%。回测页显示买点价应优先取信号日 close（实际可成交价），取值链调整为 close → breakout_close → breakout_price → buy_point。教训：引擎输出的价格字段语义不同，直接展示前需确认其含义。
44. 精选 API 每次刷新重跑完整筛选——极慢，应优先读快照表
api_screening 最初直接调用 screener.run()，每次刷新页面都重跑六层筛选（加载 1500+ 股票的 K 线和引擎），耗时数十秒。修复：优先从 discipline_screening_daily 快照表读取（毫秒级），仅当日无数据时才触发实时计算。daily_update.py 已在盘后自动预存快照，次日访问始终秒出。教训：高频访问的 API 前端页面，后端务必加缓存层——预计算落盘 > 每次实时计算。
38. SQLite AVG() 聚合函数忽略 LIMIT，导致 S 维度量比恒为 1.0
CAN SLIM v3.0 中 S 维度计算 5 日均量 / 50 日均量。原 SQL 为 `SELECT AVG(volume) FROM t ORDER BY date DESC LIMIT N`，但 AVG() 是聚合函数，先算全表均值再应用 LIMIT（对单行结果无意义），两个查询返回相同的全历史均值，比值恒为 1.0。修复：改为子查询 `SELECT AVG(volume) FROM (SELECT volume FROM t ORDER BY date DESC LIMIT N)`。同理 base_breakout 的 `_sma` 等函数遇到 NULL close/volume 时 `sum()` 崩掉——需全部加 None 过滤。教训：聚合+排序+LIMIT 的组合必须用子查询隔离；任何假定"数据不会为 NULL"的金融计算必须有守卫。
