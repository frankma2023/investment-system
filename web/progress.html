📋 本次开发经验教训
1. API 返回 JSON 时前端不要用 YAML 解析
load_config 返回 dict，前端却检查 data.ok && data.yaml 等不到，白费半天排查。
2. 预计算存表，多页面共享
index_rs_daily 每天算一次，最强指数和指数强度两页面都读它。避免重复计算和结果分歧。
3. 先对齐算法再写代码
RS 公式 (*99 vs *100)、AD 斜率 (百分比 vs 首尾对比) 反复改了 4 轮才对齐。动手前先读产品文档中的公式。
4. YAML 文件名与代码中 signal_type 严格一致
accumulation vs accumulation_day、theme vs thematic，名不对读不到。
5. 大池子 SQL 占位符不要嵌套子查询
229 个占位符导致主题池静默失败。拆成两步：先取最新日期，再按日期查。
6. 引擎 CLI 打印了结果 ≠ 写入了数据库
stock_rs.py 控制台打印双强股票但没 INSERT，快照读到全是 0。
7. RS 值不能一查到底贴到所有信号上
get_rs 只取最新快照，所有历史信号共用同一个 RS 值。需逐日查库。
8. 前低搜索窗口不设限会追溯到上市首日
双重底前低搜索了全历史数据，紫金矿业回溯到 2014 年。加 200 天窗口限制。
9. YAML 和引擎默认参数必须完整同步
YAML 缺键→前端 gp() 传不全→API merge 后缺参数→引擎 KeyError。
10. 回测页面需要统一的开发标准
口袋支点 v2 成为事实标准后写了 docs/dev/回测看板开发标准.md，后续页面直接套模板。
11. 排查工具必须覆盖引擎所有条件
扁平基部排查漏了斜率、前涨幅、突破条件，导致"全通过但不检出"的困惑。排查项=引擎条件全集。
12. 前低搜索窗口要足够大
默认 200 天不够——前高恰好在窗口边界时前低被截断。扩到 400 天。诊断工具和引擎要同步。
13. 连续突破信号需去重
同一基部突破后连日收盘都在高点之上，每天都被标为突破。加"昨日收盘<基部高点"约束只保留首日。
14. 数组截取方向错误导致全指标偏移
_sanitize_indicators 用 arr[:target_len] 取头部而非 arr[-target_len:] 取尾部，所有 BB/SMA/RSI 值偏移 600+ 天。需用外部软件交叉验证数值。
15. BB 百分位比较方向反转是系统性陷阱
talib_engine 和 flat_base 两处同时犯了 h>current 当 h
16. 插件化架构的核心是函数签名兼容
用 inspect.signature 动态适配不同引擎的 detect() 参数差异（indicators/params/rs_info），避免改旧代码。约定优于配置。
17. conda 环境与系统 Python 的包隔离
daily_update 用 sys.executable 引入 conda 环境但 talib 装在系统 Python 下。固定 PYTHON_EXE 路径避坑。
18. 全市场扫描需控制 HTML 体积
4333只股票的39K行HTML用客户端分页+筛选处理，后端只生成一次。JSON 存档、HTML 是分发格式，不实时查API。
19. 表格对齐不要用 table-layout:fixed
fixed + 固定 width 导致列头与内容错位。让浏览器 auto 分配列宽，th/td 统一 text-align:center，仅 th 加 nowrap。数字用 tabular-nums 等宽。参考 canslim-scores 的实现。
20. 理杏仁API proportion 单位不统一
fund-shareholders 的 proportionOfOutstandingSharesA 对大盘股返小数（0.0115）、小盘股返百分比（68.0）。入库前统一归一化：>1.0 除以100。
21. 双源数据取 max 不叠加
机构持股理杏仁+东方财富数据来源同一季报，占比取 max(props) 防止双重计数。最终砍掉东方财富，只保留理杏仁三API。
22. INSERT OR REPLACE 断送历史对比
机构持股每次 INSERT OR REPLACE 覆盖上季数据，机构数量变化无法计算。需保留多季数据（PK含report_date），或独立回补脚本拉8季历史。
23. YAML 被 JSON 覆盖
前端 saveConfig 发 JSON 到 /api/config，直接覆写 YAML 文件导致 yaml.safe_load 失败。修复：API 层检测 JSON 自动转 YAML，前端不包 wrapper。
24. CAGR 负数底数导致复数
3年EPS CAGR 公式 (latest/old)**(1/3)，old 为负时产生复数 → score_a 崩溃。加 old>0 and lat>0 前置守卫。
25. 评分维度子项必须设 fallback
inst_change/excess/buyback 等子项在数据缺失或负值时漏设 bd key，前端显示空白。所有子项必须有 else 分支显式设 0 或 fallback 值。
